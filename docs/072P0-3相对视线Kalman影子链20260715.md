# P0-3相对视线Kalman影子链

## 背景与目标

P0-2 已把 raw pivot 转为完整源画面坐标下的 yaw/pitch，并使用实际成功发送 counts 与固定 `commandToFrameDelayMs` 得到观测时刻的稳定视线角。本阶段在该时间语义之上建立二维独立常速度 Kalman shadow 链，输出角度、相对视线角速度、协方差、innovation、innovation variance、NIS 和前馈可信度。

本阶段仍不改变 r30 正式控制，不向设备队列写入新命令；`active` 请求继续安全降级为 `shadow`。One Euro 与 Alpha-Beta 不串联进运行链，仅保留为后续回放对照候选。

## 状态与时间更新

X/Y 两轴分别维护相同的二阶线性状态：

```text
x = [losAngleDegrees, relativeLosRateDegreesPerSecond]ᵀ
F(dt) = [[1, dt], [0, 1]]
z = losAngleDegrees
```

每次观测先在 `observationTime` 预测并用稳定视线角校正，再预测到 `controlTime` 输出 shadow 快照。`dt` 使用 P0-1 的 `steady_clock` 时间，限制在 `0.0001–0.5 s`；乱序或未知时间不伪造角速度。

过程噪声使用常加速度离散模型，当前加速度标准差为 `180 deg/s²`。测量噪声由检测置信度贯通：

```text
R = (0.15° / max(confidence, 0.05))²
```

置信度越低，测量方差越大，Kalman 更依赖已有状态。首个观测直接初始化角度、角速度置零，并保留初始协方差；不会把第一帧位移误当成速度。

## 统计量与安全语义

每轴校正计算：

```text
innovation = z − Hx̂
S = HP̂Hᵀ + R
NIS = innovation² / S
```

CSV 中的 `covariance` 是控制时刻角度方差，`innovationVariance` 是校正时的 `S`，二者不能混用。`measurementConfidence` 原样记录有效检测置信度；`feedforwardConfidence` 为置信度平方乘以 NIS 衰减：

```text
feedforwardConfidence = confidence² × exp(−min(max(NISx, NISy), 20) / 2)
```

该值只作后续速度前馈的 shadow 可信度，不直接门控 r30，也不因一次高 NIS 丢弃高置信度真实反转或 jump。

## 数据流与重置

- `AimbotTarget` 贯通检测置信度；无法提供置信度的旧排序路径使用安全默认 `1.0`。
- `MouseThread::moveMousePivot()` 将 P0-2 的稳定视线角和目标置信度交给 `AimPipelineRuntime::setViewMotionDiagnostics()`。
- 运行时在目标 `trackId` 改变时清空 Kalman 状态、重置观测序号并递增 reset generation；目标丢失仍沿用已有的整链 reset。
- `observe()` 每帧清空上一帧估计，shadow 未生成有效角度诊断时不会沿用旧 Kalman 数据。
- 估计结构只嵌入同一帧 `AimPipelineFrameState`，不持有设备队列、不调用鼠标后端。

## CSV 字段

新增 `AimPipeline` 字段包括：

- `AimPipelineAngleX/Y`、`AimPipelineRateX/Y`；
- `AimPipelineCovarianceX/Y`、`AimPipelineInnovationVarianceX/Y`；
- `AimPipelineInnovationX/Y`、`AimPipelineNisX/Y`；
- `AimPipelineMeasurementConfidence`、`AimPipelineFeedforwardConfidence`。

这些字段与 P0-2 的稳定视线角和 r30 的 `FinalMx/FinalMy` 同帧写入，BuildIdentity、控制器 r30 和旧输出均保持不变。

## 验证结果

- DML Release 构建成功，CTest `8/8` 通过。
- CUDA Release 构建成功，CTest `8/8` 通过。
- 单元测试覆盖首帧角度初始化、二维常速度收敛、正协方差和 innovation variance、低置信度增大测量不确定性、前馈可信度降权、目标模式 shadow 及 CSV 字段。
- 构建过程中仅出现既有 OpenCV 模板警告；无新增编译错误或测试失败。

## 边界与后续

本阶段不做自适应 Q/R、不做 IMM、不做 One Euro/Alpha-Beta 串联，不启用角度 P 反馈或速度前馈。下一阶段 P0-4A 才在 shadow 中拆分角误差 P、相对视线角速度前馈和抗饱和 PI，并继续以 `feedforwardConfidence` 作为可审计输入；任一新控制语义正式接管前必须重新评估控制器修订号。
