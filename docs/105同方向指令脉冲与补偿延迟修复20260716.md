# 同方向指令脉冲与补偿延迟修复

## 背景

r43 使用相同 DML/NDI 配置完成 `horizontal_left/right` 各三轮复测。用户观察大部分时间已经平稳，仍存在轻微震荡。本轮必须区分方向反转、预测开关和同方向幅值调制，不能继续叠加 PI 或预测滤波。

数据身份为 `DML / 37e31b7e0350 / r43`，配置保持响应 120 ms、最大速度 3200 counts/s、积分 500 ms、预测 50/50/1.0 和 KMBOX_NET。

## 排查过程与数据

正式移动分析器确认：

- 六轮 X/Y 的 `OutputSideFlipCount` 均为 0，Y 轴保持锁存且没有输出换向；
- `PredictionStationarySuppressedPct` 均为 0，预测方向翻转为 0；
- 预测提前量变化 P95 为 1.03~1.66 px，预测中断 left 合计 3 次、right 合计 1 次；
- X 真实稳态 P95 为 left 12.50 px、right 10.17 px。

因此 r43 已消除短平台停止门控脉冲，剩余震荡不是方向翻转或预测不连续。逐帧检查发现最终设备命令反复出现约 `-1/-8/-1` 的同方向强弱交替，并与 `ViewMotionX` 约 `-3.6~-4.7 px` 的完整命令阶跃同步。

分析器新增以下指标后，六轮旧数据得到：

| 场景 | 同方向步进P95 | 脉冲次数 | 脉冲频率 |
|---|---:|---:|---:|
| left 三轮 | 7~8 counts | 126~157/轮 | 28.79~33.90 Hz |
| right 三轮 | 7~8 counts | 142~165/轮 | 28.16~34.05 Hz |

`OutputSameSidePulseCount` 使用三个连续非零同号命令，在中点为局部极值且左右幅值差均不小于 3 counts 时计数；它不会把零输出或真正换向混入同方向脉冲。

## 根因

旧 `ViewMotionHistory` 在设备确认发送成功时，立即把完整 counts 转为像素并加入正式预测的稳定坐标系；真实命令还要经过设备、游戏渲染、NDI 和采集链路后才出现在观测画面。控制器因此会在画面尚未响应时提前补偿一整个命令，下一帧又随真实画面阶跃回摆，形成不换方向的幅值锯齿。

按 `PassiveProfileCalibrator` 的相同时间契约，使用 `CommandDeviceSendNs`、`BackendReceiveNs`、`CommandAppliedCountsX` 和 `RawPivotX` 对六轮逐轮扫描 0~250 ms、步长 2 ms，并同时回归恒定目标漂移。最优延迟为：

- left：12/12/12 ms；
- right：10/12/12 ms。

六轮集中在 10~12 ms，生产默认取 12 ms。单向目标运动使 counts 比例和目标漂移高度相关，因此本轮只采用跨六轮稳定的延迟位置，不自动采用该数据中的 px/count 回归值。

## 设计与实现

1. 新增根配置 `aim_motion_compensation_delay_ms=12`，限制为 0~250 ms。它只控制 legacy 正式自运动补偿，不改变 shadow 的 60 ms 独立候选。
2. `ViewMotionHistory` 仍只接收底层确认成功的命令，但样本时间改为 `sendTime + delay`；延迟前查询保持原累计基线，到生效时间才加入完整像素步进。
3. 运行时修改延迟会清空旧累计时间线，防止同一坐标系混用两种时基。设备发送失败、目标丢失、换向、预测和 PI 语义均不改变。
4. 控制器身份提升到 r44。移动分析器新增同方向步进 P95、脉冲次数和频率，作为本轮复测的直接验收指标。

## 测试与验收

单元测试覆盖延迟前不可见、生效时完整加入、旧样本裁剪后累计不归零、重配置清空、配置默认值和上限、同方向脉冲识别及旧 CSV 兼容。正式交付还必须完成 DML/CUDA Release 构建、CTEST、UI/CSV/产物身份核对和干净提交重建。

r44 使用相同 left/right 各三轮复测，要求：

- `OutputSideFlipCount` 继续为 0，Y 轴继续无输出换向；
- `OutputSameSideStepP95Counts` 相比 r43 的 7~8 counts 明显下降；
- `OutputSameSidePulseRateHz` 相比 r43 的约 28~34 Hz 明显下降；
- X 真实稳态 P95 不比 r43 的 12.50/10.17 px 回退超过 10%；
- 预测提前量变化、停止抑制和中断不回退。

## 后续建议

若 r44 脉冲下降但仍有固定频率残差，先在 10/12/14 ms 三点做 A/B，不调整 PI、预测或分轴锁存。只有延迟三点均不能改善时，才评估一阶连续相机响应；避免在固定延迟已经足够时增加不可辨识的时间常数。

r44 实测同方向步进 P95 仍为 7~8 counts，脉冲频率未下降，固定延迟方案未通过。后续已转为六轮共同支持的 24 ms 连续响应，详见 `docs/106固定延迟复测与连续响应修复20260716.md`。
