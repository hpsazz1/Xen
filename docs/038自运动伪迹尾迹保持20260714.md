# 自运动伪迹尾迹保持

## 背景与版本

2026-07-14使用 `BuildRevision=4130c6f71321`、`ControllerRevision=11`、DML+NDI、3200 counts/s和预测50/50/1.0完成static九段复测，全程未使用手动鼠标辅助。CSV版本、控制器修订号和参数身份一致。

## 数据结论

r11门控已产生明确收益：static X/Y预测生效率从r10的15.3%/19.9%降至4.7%/4.9%，自运动抑制比例均为7.6%，X/Y P95误差为4.61/5.02 px。说明“三帧屏幕收敛、自身视角移动、预测速度方向”联合判定能够识别主动拉回静止目标产生的补偿残差，不需要回退3200 counts/s或削弱50/50/1.0预测参数。

剩余晃动集中在门控命中后的旧速度恢复。trial 8在第22帧命中后，第23至26帧立即恢复8.1~12.0 px提前；trial 9在第22至24帧连续命中后，第25至26帧恢复4.0~6.4 px提前。trial 8后半段也在连续抑制结束后恢复两帧。门控只撤销命中当帧，但预测器为保持真实移动连续性仍保留最后可靠速度，因此控制响应与检测框收敛尾迹会再次使用旧方向。

## r12设计

1. 联合判定条件保持不变；只有r11已确认的自运动伪迹才能启动保持，避免扩大误判入口。
2. 命中当帧及其后四个有效观测帧统一撤销预测偏移。在约100 FPS DML推理下对应约40 ms后续保持，完整覆盖本轮2~4帧恢复尾迹。
3. 保持期间不清空回归窗口、最后可靠速度和方向锁定。保持结束后真实移动可直接恢复，不重新引入r8的大量预测中断。
4. `PredictionSelfMotionSuppressed` 在命中帧和保持帧均记录为1，使CSV直接反映实际被撤销的控制帧。
5. 目标丢失、身份切换或瞄准状态变化时，`resetTracking()`通过预测器重置同步清空保持计数，禁止跨目标继承。

控制器修订号升级为12。单元测试覆盖命中当帧撤销、四个后续观测持续撤销、第五帧恢复以及跟踪重置清空保持。

## 下一轮测试

参数保持 `move_max_speed_cps=3200`、预测 `50/50/1.0`，禁止手动鼠标：

- static九段优先复测，要求X/Y预测生效率不高于3%，trial 8/9不再出现门控后2~4帧旧提前量恢复；重点看整体P95误差和准星肉眼稳定性，不再把仅对少量有效预测帧计算的提前P95单独作为否决项。
- static通过后采集left/right各4轮，要求X轴P95不高于r10的22.26/19.97 px，预测生效率至少95%，自运动抑制低于5%。
- jump连续20秒，要求单个有效试次至少15秒、X轴P95不高于60 px、自运动抑制低于10%，确认40 ms保持未破坏高速追踪。
- reverse和 `Target_static_Role_reverse` 各20秒，核对方向翻转与角色移动时没有新增长时间预测空窗。
- 目标飞出、目标死亡、角色死亡各10次，确认丢失后无输出且新目标不继承保持状态。
