# 反转与跳跃提前量渐进释放

## 背景

r20已经通过static与持续left/right验收，但本轮`horizontal_reverse.csv`表现为准星无法稳定落在目标附近而来回摆动，`horizontal_jump.csv`则明显跟不上。两份数据来自DML构建`7abe8c58d732`、控制器r20，预测参数为`50/50/1.0`，最大设备速度3200 counts/s。

## 数据结果

### reverse

- 连续采集19.51秒，检测目标未丢失。
- X/Y P95误差为17.81/2.95 px。
- 识别22次反转，全部恢复；平均恢复43.3 ms，P95为66 ms。
- 预测只在17.5%的X轴样本生效，中位连续长度仅2帧。
- 预测方向换边36次、中断83次，自运动抑制率21.9%。
- 预测提前P50/P95为10.22/15.31 px。

反转识别本身并不慢，主要问题是每次短暂满足方向条件后立即释放全额提前量，随后又被停止、反向或自运动门控撤销，形成短脉冲式左右摆动。

### horizontal_jump

- 连续采集21.40秒，检测目标未丢失。
- 预测点X/Y P95误差为118.61/36.62 px；去除预测偏移后的目标框中心P95仍为51.81/28.54 px。
- 设备限速比例10.9%，X轴闭环滞后估算51.9 ms。
- X轴预测中位连续长度仅3帧，中断66次。
- 预测提前P95达到112 px，即320检测跨度的35%，显著大于目标框宽度中位数47 px。

jump既有真实高速跟随不足，也有预测点过远放大控制误差的问题。直接继续提高设备速度会同时放大反转过冲，不能作为首选方案。

## 设计

1. 保留r20的四次可靠方向确认，避免static误预测回归。
2. 方向正式建立后，提前量按连续有效预测帧数在七帧内从`1/7`线性增长到全额。
3. 停止、窗口退化、反向或自运动抑制会沿用现有逻辑清零连续计数；下一段重新渐进释放。
4. 最大预测距离从检测跨度35%降为20%，320区域由112 px降为64 px。
5. 普通left/right提前P95约20 px，不触发64 px上限；连续约70 ms后仍恢复用户设置的完整提前强度。
6. 不修改基础控制器响应、积分、稳定半径、设备速度、目标选择和丢失规则。

## 离线收益估算

使用本轮CSV保留真实观测和视角轨迹，仅替换预测偏移进行反事实计算：

- reverse X轴P95预计从17.81 px降至约12.63 px。
- jump X轴P95预计从118.61 px降至约87.45 px，Y轴从36.62 px降至约34.30 px。
- jump预测距离P95由112 px限制到64 px。

该估算不能代替闭环实机结果，因为实际视角运动会随新输出变化；它只用于证明修改方向与现有数据一致，并筛除明显无收益方案。

## 验证方案

- 单元测试覆盖首个正式预测帧小于成熟提前量、七个稳定帧后恢复全额、异常jump不超过64 px。
- DML与CUDA执行Release构建和完整CTest，UI、CSV、EXE统一显示r21与同一干净提交。
- 首轮实机只复测reverse和horizontal_jump各4轮。
- reverse要求X轴P95低于17.81 px、预测方向摆动明显减少，且恢复P95不高于66 ms。
- jump要求X轴P95和限速比例同时下降，完整跟踪期间不得新增目标丢失。
- 专项通过后回归static九段与left/right各4轮；static预测生效率保持不高于0.5%，left/right预测生效率不低于94%，X轴P95不明显回退于7.62/8.49 px。
