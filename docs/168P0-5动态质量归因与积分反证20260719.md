# P0-5 动态质量归因与积分反证

## 背景

`docs/167P0-5物理集合与绝对门禁契约20260719.md`冻结14/0、14/2、15/0、15/2四个实测物理端点，并确认当前候选同时存在三类阻断：left/right场景平均P95回退，jump最差域在整步端点越过检测裁剪内接角半径约2%，static新增输出翻转。本轮不修改门禁、active、r64实机shadow或legacy输出，只定位动态回退能否由一个现有成熟控制分量共同解决。

## 归因复核

15/2的既有归因矩阵中，left/right共324域：

| 分类 | 数量 |
|---|---:|
| `ACTUATOR_LIMITED` | 256 |
| `EITHER_INTERVENTION_RESCUES` | 58 |
| `ESTIMATOR_LIMITED` | 2 |
| `COUPLED_OR_GATE_LIMITED` | 1 |
| 原门禁通过 | 7 |

解除限速反事实把候选上限提高到100000 counts/s，只能证明误差受执行能力约束，不能证明真实设备应突破1440 counts/s。完整调用链复核确认`BasicAimController`和`LosAimController`都按二维合量执行同一个每周期预算，不存在legacy按轴、候选按合量的语义错配。

四端点下，left/right候选垂直P95平均约0.112～0.126°，legacy约0.166～0.192°，候选反而改善约33%；平均控制中断率与legacy差异不足0.2个百分点。因此单向二维P95回退主要来自水平持续跟随相位，不是Y轴抢预算或停发。

## 速度前馈有界反证

时间轴修复前的前馈结果建立在约一半真实速率的错误估计上，不能复用。时间轴修复后的历史60/0矩阵中，0.05前馈已从660降至645；考虑实测15 ms物理相位不同，本轮仍只补做一次0.05反证，不继续扫描。

## 候选角积分审计入口

`LosAimController`已有带I-zone、二维25%周期预算、饱和冻结和反向解卷绕的角积分，但跨域CLI此前没有候选专用入口。本轮新增：

- `--candidate-integral-ms`：0关闭，非零限制为50～2000 ms；
- `--candidate-integral-zone-deg`：限制为0～10°，默认1°；
- decision与summary记录`CandidateIntegralTimeMs/CandidateIntegralZoneDeg`；
- 单元测试验证候选请求发生变化、legacy P95逐位不变，并核对summary身份字段。

入口默认关闭，只接入跨域离线候选。它不修改`aim_shadow_integral_time_ms=0`、UI默认值、运行时控制器、active能力或控制器修订号。

## 500 ms/10°积分反证

1°默认I-zone覆盖不了约6°单向稳态误差，因此使用既有配置允许的上界10°，积分时间采用legacy长期实测使用的500 ms，只做一次结构反证：

| 场景 | 基线通过 | 积分通过 | 基线候选平均P95 | 积分平均P95 | 基线/积分翻转总数 |
|---|---:|---:|---:|---:|---:|
| jump | 162 | 162 | 6.976° | 6.476° | 1175/2351 |
| left | 27 | 0 | 6.323° | 6.438° | 1161/1504 |
| reverse | 162 | 162 | 1.069° | 0.870° | 11048/12982 |
| right | 25 | 0 | 6.202° | 6.331° | 1538/1908 |
| static | 148 | 24 | 0.085° | 0.072° | 18/184 |

总通过由524降至348。积分虽然降低jump、reverse和static平均误差，却在持续单向中累积错误相位，并把static翻转扩大一个数量级；不能用平均误差改善覆盖设备方向翻转。正式否决角积分方向，不继续扫描积分时间或I-zone。

summary SHA-256为`85579C02A43E99B91F40B67B5908DA1889D69C4544BD7555C999C549454512AD`。

## 结论与下一步

1. 保持前馈0、角积分0、候选响应80 ms、最大1440 counts/s和全部r64参数不变；继续`HOLD_SHADOW`。
2. 现有相对LOS速度直接前馈和误差积分都不能分离“持续单向需要较弱P”与“jump/reverse需要较强瞬态P”，不再扫描相邻数值。
3. 下一步只增加诊断：常速度/常加速度模型角度与速率差、物理速率证据、限速前请求和实际限速比例。先验证100 ms响应在持续单向改善、却破坏jump/reverse的状态边界是否能由真实模型状态稳定分离，再决定是否提出响应调度候选。
4. 在上述诊断形成跨FOV/FPS/延迟/速度证据前，不实现新状态机、不提高设备上限、不修改active或门禁。
