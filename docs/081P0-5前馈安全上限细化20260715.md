# P0-5前馈安全上限细化

## 背景

主场景矩阵采用相对LOS速度前馈0.15后达到733/810。剩余static失败复核表明，29组输出翻转和8组P95回退仍来自已知的静止估计抖动与量化脉冲；此前0.05°静止阈值、1.5°/s速率阈值和80 ms低速反向确认已经完成反证。继续叠加static门控会重复局部补丁，因此本轮不改控制器，只细化0.15与0.20之间的统一前馈安全上限。

## 选择原则

用户当前关注目标运动/角色静止及双方静止。选择不能只最大化总通过数，必须满足：

- left/right继续各162/162；
- 相对当前0.15候选，jump、reverse、static任一场景不得减少通过数；
- 不修改Kalman、P响应、垂直追赶、静止状态机或轨迹执行层；
- 不继续扫描更多小数位拟合当前视频的整数counts门槛。

## 结论

离线主场景候选前馈从0.15细化为0.16。该变化只增加2组reverse通过，其余四场景保持不变，是当前证据下的安全上限。运行时`aim_shadow_feedforward_gain`默认继续为0，不能把离线候选直接视为实机默认；P0-6仍不得开始。
