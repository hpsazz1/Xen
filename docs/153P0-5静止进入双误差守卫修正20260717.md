# P0-5静止进入双误差守卫修正

## 背景

`docs/152P0-5在途命令静止进入保护20260717.md`记录的冻结目标guard达到753/810，但仍损失5个原通过static域。本轮只对这5域采集baseline/guard逐帧明细，目标是定位首次新增翻转，不扫描horizon或放宽门禁。

为避免对同一视频重复解码10次，`--cross-domain-detail-variant`扩展为逗号分隔的精确域列表。每项仍必须属于162项标准矩阵，重复项自动去重，非法项在模型初始化前失败；CSV继续用`Variant`标识来源，原单域命令完全兼容。

## 首次新增翻转

代表损失域`FOV120/1920x1080/144 FPS/40 ms/1.5x`中，旧baseline在0.345556 s才满足当前误差安静并进入settled。原guard却在0.290000 s进入settled：当时当前Y误差仍为0.431088°，承诺终点Y误差仅0.030688°。

这证明原实现把承诺终点误差替换了当前误差，而不是作为附加守卫。随后候选在0.296944、0.317778 s两次释放并重入，改变了后续相机轨迹；0.567778 s再次释放，0.574722 s首次产生新增反向输出。该链路与整数余量无关，根因是当前大误差被错误终点提前判静止。

## 双条件修正

正确进入条件必须同时满足：

1. 当前二维误差不超过0.08°；
2. 已承诺终点二维误差不超过0.08°；
3. 二维相对LOS速率不超过1.2°/s；
4. 连续两个观测。

单元测试新增“当前误差0.20°、终点误差0.05°仍不得累计quiet sample”，防止以后再次把guard误写成替换值。

完整矩阵结果由错误的753/810回落到正确的728/810：dynamic四场景通过数仍为156/145/155/147，static由118升到125。相对721基线救回9项、损失2项；static P95均值基本不变（0.08159°到0.08158°），总翻转344到342。

剩余两个损失域为：

- `domain_fov106_1920x1080_144fps_40ms_0.75x`：翻转3到4，P95从0.085562°改善到0.069436°；
- `domain_fov120_2560x1440_94fps_40ms_1.5x`：翻转2到10，P95从0.076522°回退到0.101973°。

## 结论

753/810不能再作为候选依据，它的大部分收益来自错误的提前settle。双条件guard结构正确且有净收益，但仍违反原721项零损失门禁，因此继续默认关闭，不接入配置、UI、正式shadow或active，控制器身份保持r63。

下一步若继续，只允许对上述两个正确guard损失域采集逐帧明细，首先检查第二域为何在双条件下仍形成10次翻转；不得恢复替换语义、扫描horizon或用总净收益覆盖局部回退。
