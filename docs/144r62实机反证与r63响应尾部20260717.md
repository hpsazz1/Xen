# r62实机反证与r63响应尾部

## 背景与数据身份

用户上传 `C:\Users\16143\Desktop\Xen\DML\ndi\static_r62.csv`，用于验证 r62 的命令响应观测不确定性。文件共 1839 帧，身份为 `DML / 412d46fc5e7a / r62`；逐帧配置为 shadow、20/20 ms、`maneuver_gated_ca + 8000/12/120`、不确定度增益 1.25，Profile 标定关闭。

CSV 包含九个运行段，合计 831 个运行态观测和 1008 个暂停态观测。构建身份、模式降级、时序、估计有限性、20/20 响应、1.25 增益乘积、暂停零命令、暂停连续观测及选择计数均为 0 违例。

## r62结果

r62 static 硬门槛失败：运行态有 13 个机动启用帧，其中 6 帧已进入 settled，集中在第 5 个点的单一连续段；其余 8 个点均为 0。暂停态有 252 个机动启用帧，发生在松键换位期间，不计入 static 运行态门禁但继续保留为诊断。

第 5 点在恢复后约 82 ms 首次触发。触发前相机垂直响应率曾达到 61.60~95.48 度/秒，随后响应区间结束、瞬时不确定度归零；常速度 Kalman 的门控速率在约 15~31 ms 内发生反向过冲，证据峰值 14.44 度/秒并越过 12 度/秒门槛。首帧触发时相机响应率和不确定度均为 0，因此继续增大 1.25 倍增益无法解决；后续 12 帧主要是 120 ms 保持，不是首次触发原因。

正式离线复放以 13 运行帧、6 settled 帧、1 段精确复现 r62，证明问题来自模型结构而非运行时与复放分叉。

## 尾部设计

只对“命令可解释速率上界”增加短尾部：每条成功命令在线性响应结束后继续保留原响应率一段固定时间，再按轴进行有符号合成并乘 1.25。相机累计角度仍严格使用原 20/20 线性响应，不延长画面补偿；Kalman 测量、状态、CA 状态、12 度/秒门槛和 120 ms 保持均不变。

有界搜索固定增益 1.25，只比较尾部 0/5/10/15/20/25/30/40/50 ms：

| 尾部 | 四份static运行启用/settled/段 | jump启用帧/覆盖 | fast reverse启用帧/settled/覆盖 |
|---:|---:|---:|---:|
| 0 ms | 13 / 6 / 1 | 977 / 5/5 | 1440 / 463 / 4/4 |
| 5 ms | 12 / 6 / 1 | 701 / 5/5 | 1273 / 439 / 4/4 |
| 10 ms | 12 / 6 / 1 | 385 / 5/5 | 1165 / 431 / 4/4 |
| 15 ms | 12 / 6 / 1 | 308 / 5/5 | 1163 / 431 / 4/4 |
| 20 ms | 0 / 0 / 0 | 249 / 5/5 | 1151 / 431 / 4/4 |

20 ms 是首个通过所有硬门槛的最小值；更长尾部没有必要，拒绝扩大抑制范围。慢速 reverse 仍低于物理门槛，不要求激活。

## r63实现

r63 在 `AppliedViewMotionModel` 中增加带尾部的不确定度速率查询，固定 DML shadow 为 1.25 倍、20 ms 尾部。CSV 分别记录真实相机响应率、含尾部的不确定度响应率、增益、尾部、逐轴不确定度和最终门控证据，避免把尾部误写成真实画面运动。分析器校验 20 ms 身份、有限值、乘积关系及运行链传递一致性。

CUDA 继续强制 0 ms 阶跃、常速度 Kalman和 0 不确定度增益；active 仍降级为 shadow，设备输出仍由 legacy 产生。控制器身份升级为 r63。
