# 局域网 Web 控制台

## 背景

桌面叠加层适合本机调参，但实机测试时频繁切回桌面会打断捕获与输入链路。局域网 Web 控制台用于在手机或另一台电脑上查看运行状态，并执行暂停、恢复、配置重载和有限参数修改。

## 设计

控制台直接嵌入 Xen 进程，使用固定版本的单头文件 `cpp-httplib v0.50.1` 提供 HTTP 服务，不依赖 Node.js 或外部服务。页面由程序内嵌并通过同源 API 通信，支持手机与桌面尺寸。

配置项：

- `lan_console_enabled`：默认 `false`，只有显式启用才启动服务。
- `lan_console_bind_address`：默认 `0.0.0.0`，监听本机全部 IPv4 网卡。
- `lan_console_port`：默认 `17888`，限制在 `1024..65535`。

启动后控制台打印六位配对码。浏览器使用配对码换取进程生命周期内有效的随机会话令牌，后续 API 必须在 `X-Xen-Session` 请求头中携带该令牌。进程重启后配对码和令牌全部失效。

## 控制范围

运行控制只开放暂停、恢复和重载。重载请求通过 `remoteReloadRequested` 原子标志交给键盘线程，复用 F4 的配置加载、差异检测和模块通知链路。

参数接口只允许修改以下白名单：置信度、NMS、移动响应时间、最大移动速度、预测提前量、捕获帧率、自动瞄准、预测开关，以及身体/头部类别 ID 映射。页面预置 `0=警身`、`1=警头`、`2=匪身`、`3=匪头`，类别 ID 经过 `0..255` 范围校验且身体与头部不能相同。所有数值经过有限值与上下界校验，保存仍调用项目统一的 `Config::saveConfig()`。接口不提供退出、文件访问、路径修改、任意配置键或系统命令执行能力。

## 使用

在 `config.ini` 中启用：

```ini
lan_console_enabled = true
lan_console_bind_address = 0.0.0.0
lan_console_port = 17888
```

启动 Xen 后，在手机浏览器访问 `http://主机局域网IP:17888`，输入程序控制台显示的六位配对码。Windows 防火墙若阻止入站连接，需要仅对专用网络放行该端口。

## 安全边界

当前版本使用明文 HTTP，只适用于可信家庭或测试局域网，不应映射到公网、访客 Wi-Fi 或跨互联网隧道。需要跨网访问时，应在独立反向代理上配置 TLS、访问控制和审计，不应直接暴露 Xen 监听端口。

## 验证

`tests/lan_control_tests.ps1` 固定检查鉴权、API 路由、默认关闭、配置保存和禁止系统命令执行等契约。DML/CUDA Release 构建用于验证共享服务代码在两个后端均可链接。
