# 反转阈值与最新积分复测

## 背景

- `C:\Users\16143\Desktop\XenMovingPI240`

六份 CSV 均为 `BuildRevision=58e2dc0cf024`、`ControllerRevision=4`，UI、CSV 和程序身份一致且无 `-dirty`。两组保持 `move_response_ms=80`、`move_max_speed_cps=1440`，唯一控制变量是320或240 ms积分时间。

## 反转漏检根因

240 ms高频文件肉眼包含大量左右换向，但原默认±12 px误差侧阈值只识别6次。阈值扫描结果如下：

| 误差侧阈值 | 320 ms反转 | 240 ms反转 |
|---:|---:|---:|
| ±8 px | 29 | 23 |
| ±10 px | 27 | 23 |
| ±12 px | 19 | 6 |

240 ms高频误差P95仅12.85 px，±12 px已经贴近信号上沿，导致大多数真实换向没有连续3帧越过阈值。±10 px仍高于8 px恢复半径，且连续3帧确认能够抑制中心噪声，因此将分析器默认值由12 px调整为10 px。

## 最新对照结果

使用默认±10 px阈值重新分析：

| 场景与指标 | 320 ms | 240 ms | 240相对变化 |
|---|---:|---:|---:|
| 左移末尾1秒P95 | 3.58 px | 3.18 px | 改善11.2% |
| 右移末尾1秒P95 | 3.62 px | 3.08 px | 改善14.9% |
| 高频误差P95 | 15.34 px | 12.85 px | 改善16.2% |
| 高频恢复均值 | 60.8 ms | 67.0 ms | 回退10.2% |
| 高频恢复P95 | 76 ms | 90 ms | 回退18.4% |
| 反转次数/时长 | 27/10.72 s | 23/8.94 s | 2.52/2.57次每秒 |

两组高频反转速率相差约2%，具备直接比较价值。240 ms在持续跟随和高频误差幅度上均有改善，但恢复尾延迟明显变差，不满足“一个核心指标至少改善10%，另一核心指标不得回退超过10%”的验收门槛。

单向试次的起始误差仍不统一，范围约0～151 px，因此整段P95继续只作辅助指标；末尾1秒稳态指标可以用于判断持续跟随。

## 优化结论

1. 当前现场推荐继续保持320 ms，不直接采用240 ms。
2. 分析器默认反转阈值改为10 px，连续确认帧保持3帧，恢复半径保持8 px。
3. 下一轮只测试280 ms中间档与320 ms基线，判断能否保留240 ms的大部分稳态收益，同时把恢复P95控制在320 ms的10%回退范围内。
4. 不修改80 ms响应、1440 counts/s上限、滤波、NDI采集或预测模块，避免多变量混合。

## 下一轮测试方案

1. 使用同一干净DML构建和同一OBS NDI场景，目录分别为 `XenMovingPI320/DML/ndi` 与 `XenMovingPI280/DML/ndi`。
2. 按 `320→280→280→320` 执行ABBA；每档左右单向各3轮，每轮4秒，末尾稳定移动至少1.5秒。
3. 单向试次起始水平误差固定为80±5 px；超出范围立即重录，不进入汇总。
4. 高频反转每轮10秒，目标反转速率2.5±0.15次/秒；使用±10 px、连续3帧分析，要求两档反转速率差不超过5%。
5. 280 ms通过条件：左右末尾P95平均至少改善8%，任一方向不得回退超过10%；高频误差P95至少改善8%，恢复P95不得高于320基线10%；全部反转必须恢复，队列深度不超过1且限速为0。
6. 移动通过后补做上中、左下、右下静止各3轮，要求稳定后最大误差不超过8 px、稳定态退出为0、无持续反向积分。
