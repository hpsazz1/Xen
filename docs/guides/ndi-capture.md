# NDI 网络采集

NDI 采集通过网络从另一台 PC 或同一台 PC 上的 NDI 源接收视频帧。发送端（如 OBS、vMix 等）通过 NDI 协议流式传输视频；本应用在接收端上接收这些帧。

## NDI 采集特点

- **零配置发现** — NDI 使用 mDNS 自动发现局域网中的所有 NDI 源，无需手动输入 IP 地址。
- **高画质** — NDI 使用高质量视频编码，画质优于 MJPEG 的 UDP 传输。
- **带宽较高** — 全屏 1080p60 的 NDI 传输约需 100-200 Mbps 带宽。千兆局域网可以轻松应对。

## 推荐配置

```ini
capture_method = ndi
ndi_source_name = HPSAZZ (OBS PGM)
ndi_source_width = 2560
ndi_source_height = 1440
detection_resolution = 320
capture_fps = 0
```

- `ndi_source_name` — 必须填写发现列表显示的完整名称。例如 `HPSAZZ (OBS PGM)` 中主机名和括号内节目名都属于名称，不能只填其中一段。
- `ndi_source_width/height` — OBS 发送 1:1 中心预裁剪 ROI 时，填写游戏完整 FOV 尺寸。收到合法 Xen 帧元数据时元数据优先；设为 `0/0` 时回退为 NDI 视频帧尺寸。
- `detection_resolution` — 从 NDI 完整帧中心按 1:1 像素裁出的正方形检测区域，不会拉伸完整画面。
- `capture_fps` — 辅机处理帧率上限；控制器实际使用相邻有效观测的时间间隔。

## OBS 低延迟 320×320 方案

完整 2560×1440 NDI 通常需要显著更高的编码、内存复制和网络带宽，链路接近带宽或编解码能力上限时会增加排队和延迟。延迟是否增加取决于 OBS 输出帧率、NDI 模式、网卡和接收端性能，不能只按像素数断定；应以发送时间码和接收统计实测。

当前推荐保持 320×320 传输，但 OBS 场景必须满足以下约束：

1. OBS 的 320×320 输出是主画面中心的 1:1 像素裁剪，不是把完整 2560×1440 缩放或挤压成正方形。
2. 接收端设置 `ndi_source_width = 2560`、`ndi_source_height = 1440`。这两个值只影响 FOV/鼠标换算，不会放大网络图像或增加 NDI 带宽。
3. CSV 验收时 `Resolution` 应为 320，`SourceWidth/SourceHeight` 应为 2560/1440。

接收端还支持视频帧 XML 元数据：

```xml
<xen source_width="2560" source_height="1440"
     roi_x="1120" roi_y="560" roi_width="320" roi_height="320" />
```

现有 OBS NDI 插件通常不会自动注入这段 Xen 自定义元数据，因此本配置采用宽高字段回退。以后更换可注入元数据的发送器后，无需删除配置；合法帧元数据始终优先。

## 接收端设置

1. 确保发送端正在输出 NDI 信号（如 OBS 开启 NDI 输出）。
2. 设置采集配置如上。
3. 启动应用，界面中选择 NDI 源。

## 网络负载说明

NDI 的带宽消耗取决于传输分辨率：

| 分辨率/帧率 | 大致带宽 | 适用场景 |
|---|---|---|
| 640×640 @ 60fps | ~30-50 Mbps | 千兆网络下轻松应对 |
| 1920×1080 @ 60fps | ~100-200 Mbps | 建议千兆网络 |
| 3840×2160 @ 30fps | ~200-400 Mbps | 建议万兆网络 |

**注意：** NDI 是局域网协议，不适合互联网传输。如果网络负载过高，可在发送端降低输出分辨率或帧率。

## NDI 源找不到

1. 确认发送端正在输出 NDI 信号。
2. 确认两台电脑在同一个局域网/子网。
3. Windows 防火墙可能阻止 NDI 发现。NDI 使用 UDP 端口 5959 进行发现，TCP 端口 5960 进行数据传输。
4. 在界面中点击"刷新 NDI 源"按钮重新扫描。
5. 尝试将 `ndi_source_name` 设为 `Auto`，让其自动连接第一个发现的源。

## 全屏传输的建议

对于高分辨率全屏传输（如 1920×1080），建议：

- 使用**千兆网络**（1000 Mbps），这是基本要求。
- 在发送端（如 OBS）降低输出分辨率，或降低帧率到 30fps。
- NDI 接收端读取视频帧自身的分辨率，并从中心裁出检测区域。辅机本地显示器分辨率不参与 FOV 或鼠标换算。
