# 预测设置入口修复

## 背景

连续观测预测模块完成后，现场准备按关闭、20 ms、35 ms、关闭的顺序测试，但UI中找不到预测配置。配置文件和运行时预测参数已经接通，问题仅出现在界面入口，导致用户无法可靠执行单变量A/B。

## 原因与排查

预测绘制函数仍保留基础阶段使用的 `false &&` 编译期屏蔽条件，因此即使调用该函数也不会创建“预判参数”区域。同时，新版侧栏定义表只注册了移动参数、射击辅助和灵敏度配置，没有注册 `draw_mouse_prediction`。两个条件叠加后，预测控件在任何页面都不可见。

## 解决方案

移除预测页面专属绘制条件前的编译期屏蔽，并在“瞄准”分组中新增“预测参数”入口。页面只显示实际参与当前控制链的三个设置：启用预测、额外前瞻和速度平滑。旧的“自动优化参数”复用了跟踪器开关且不会自动推导新预测参数，容易造成错误理解，因此不再显示。

新增正式UI可见性测试，静态核对以下契约：预测页面不能被 `false &&` 隐藏；页面必须存在有效绘制条件；三个关键控件必须存在；侧栏必须把“预测参数”绑定到 `draw_mouse_prediction`。测试已注册到双后端CTest，后续界面重构若破坏入口会直接失败。

## 验证与后续

现场测试入口固定为“瞄准 → 预测参数”。关闭预测时两个数值滑块置灰但保持可见；开启后可分别设置20 ms或35 ms。提交前DML Release与CUDA Release均构建成功，两套完整CTest均为8/8通过，其中包含新增的预测UI可见性测试。提交后仍需重新生成干净版本并核对UI、CSV和可执行文件身份；测试矩阵继续以 `docs/todolist.md` 为准。
