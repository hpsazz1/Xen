# 短暂停追赶补偿尾迹误门控修复

## 背景

最新复测数据位于 `C:\Users\16143\Desktop\Xen\DML\ndi`，包括左移 1232 行和右移 1123 行。两份 CSV 均来自 DML、程序提交 `21576a29e955`、构建时间 `20260716T111510Z`、控制器修订号 r52。

r52 为同方向短暂停恢复增加了最多 100 ms、120 ms 收紧到 100 ms 的大误差追赶窗口。本轮验证其实际收益及是否引入预测连续性回退。

## 数据分析

共识别 14 次不超过 350 ms 的短暂停恢复和 4 次 1.1～1.45 s 的长暂停：

- 所有短暂停的重按连续前三帧均保持同方向非零提前量，r51 的次帧空洞没有复发；
- `ResponseSeconds=0.100` 实际持续 31～84 ms，并在误差低于 32 px 后提前恢复 0.120，r52 窗口条件符合设计；
- 35～60 px 中小原始积压追到 16 px 内为 55～70 ms，相比 r51 的 62～95 ms 明确改善；
- 67～98.5 px 大积压受 3200 counts/s 限制，追到 16 px 内为 78～105 ms，个别段未在 300 ms 内稳定进入；
- 5 个恢复段在重按后 57～128 ms 出现预测提前量撤销，其中两段先命中 5 帧高速瞬态抑制，随后转入运动证据重建；另外三段直接进入运动证据重建；
- 典型失败段的预测回归速度仍保持成熟方向和 484～744 px/s，但补偿后的单帧观测速度交替达到约 `-16250～+2887 px/s`。这与 r52 快速相机追赶的控制响应时间一致，不是真实目标反向。

因此 r52 的追赶收益成立，但预测器把快速控制产生的补偿尾迹误当成目标高速加速或反向回归，形成“先追上、提前量再归零、随后重新建立”的新复位。

## 设计与实现

新增仅由同方向短恢复触发的成熟方向保护：

1. 保护期覆盖 100 ms 快速追赶窗口，并额外覆盖 50 ms 命令到画面的响应尾迹，总计最多 150 ms；
2. 保护期内不因同方向高速加速度触发 `highSpeedSuppressed`；
3. 若回归速度突然与已经持续确认的方向相反，不覆盖最后可信速度，也不累计不可靠/反向建向证据；
4. 仅对 `sustainedMotionConfirmed`、`predictionEstablished` 且方向已锁定的成熟预测生效；
5. 静止检测仍优先执行，长暂停、目标切换/丢失仍完整重置，预测距离仍受 24 px 上限约束；
6. 保护期结束后，真实反向仍在首个可靠反向回归撤销旧提前量，并按原三次可靠回归规则换边。

控制器修订号由 r52 升级为 r53。r52 的 100 ms/100 ms 追赶响应保持不变，本轮只修复预测与控制响应尾迹的衔接。

## 单元测试

- 相同成熟高速加速度序列：普通预测器仍命中既有高速安全门控，保护期预测器保持非零成熟提前量；
- 连续三个补偿反向回归：无保护预测器撤销旧方向，保护期预测器保留原方向与最后可信正速度；
- 既有真实反向、静止、高速、长暂停、短暂停连续恢复和四样本接管测试保持通过；
- 流水线 CSV 与 UI 身份断言升级为 r53。

## 复测建议

使用 r53 重复左右移动中的松开/重按：

- r52 中 57～128 ms 出现的 `PredictionHighSpeedSuppressed` 和 `PredictionMotionEvidenceSuppressed` 应为 0；
- 短暂停后至少前 150 ms 的 `PredictionOffsetX` 应连续保持同方向非零；
- 中小积压仍应保留 r52 的 55～70 ms 追赶收益；
- 长暂停继续冷启动，真实停止应在 30 ms 后撤销提前量；
- 若刻意在重按后立即真实反向，最多 150 ms 后必须恢复普通反向确认，不得长期沿用旧方向。
