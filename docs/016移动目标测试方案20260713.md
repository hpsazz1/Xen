# 移动目标测试方案

## 背景与静止基线归档

静止目标调参阶段已完成并归档。最终基线为320×320检测区域、2560×1440完整FOV、`move_response_ms=80`、`move_max_speed_cps=1440`、5 px进入稳定半径和8 px释放半径。1440复测的36段均无稳定态退出，四链路平均首次稳定为299.1～323.3 ms，远距限速为61.2%～66.7%，当前不继续调整静止目标参数。

静止九宫格只证明目标停止时能够稳定到达中心，不能证明对持续横移和方向反转目标的跟随误差。移动测试阶段必须保持静止基线参数不变，先建立数据基线，再决定是否需要速度、响应、滤波或未来预测；禁止同时修改多个变量。

## 诊断缺口与实现

原流水线只导出 `ObservedSpeed` 模长，无法区分向左、向右、向上和向下，也无法可靠识别方向反转。本轮增加：

- `ObservedVelocityX`：相邻原始观测的水平速度，单位 px/s，正值向右。
- `ObservedVelocityY`：相邻原始观测的垂直速度，单位 px/s，正值向下。
- `ObservedSpeed` 继续保留，并严格等于两个有符号分量的模长。
- `tools/analyze_moving_target.ps1`：按采集空档拆分试次，统计轴向跟随误差、二维误差、相对观测速度、滤波残差、限速、观测年龄、队列和反转恢复。

有符号速度是目标在检测画面中的相对观测速度，包含目标自身运动与自瞄转动造成的画面变化，不等同于世界坐标真实速度。因此测试源必须使用固定、可重复的移动脚本；速度字段主要用于识别相对方向和审计帧间变化，不能脱离场景直接推导目标世界速度。

## 第一轮最小测试闭环

只使用上一轮确定的日常主环境：CUDA 下分别测试 NDI 和 UDP。DML不参与第一轮完整移动基线，但仍完成Release构建和单元测试。

每条传输采集三个文件：

1. `horizontal_right.csv`：目标持续向右横移。
2. `horizontal_left.csv`：目标持续向左横移。
3. `horizontal_reverse.csv`：目标左右往返，至少包含5次明确反转。

每个单向文件包含3轮，每轮保持目标连续移动至少2秒；轮次之间松开瞄准至少300 ms，使分析器按时间空档自动分段。反转文件连续采集至少6秒。两种传输必须使用相同目标脚本、速度、起点、程序版本和配置，文件放置结构如下：

```text
XenMoving/
  CUDA/
    ndi/
      horizontal_right.csv
      horizontal_left.csv
      horizontal_reverse.csv
    udp/
      horizontal_right.csv
      horizontal_left.csv
      horizontal_reverse.csv
```

流水线追踪容量1000帧足以保存约6秒、140 FPS附近的检测发布结果；如果单文件超过1000条，应缩短单次采集或将 `pipeline_tracer_max_frames` 临时提高到2000，并在文件配置旁记录该变化。

## 分析命令与指标

使用新CUDA构建采集后执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools/analyze_moving_target.ps1 `
  -DataRoot C:\Users\16143\Desktop\XenMoving -Axis X `
  -OutputCsv C:\Users\16143\Desktop\XenMoving\moving_summary.csv
```

首轮重点比较：

- `ObservedP95AbsAxisErrorPx`：准星相对原始检测支点的轴向屏幕误差P95，是实际跟踪质量的主要指标；`ObservedTrackingAvailable=1`时才有效。
- `ObservedP95DistancePx`：准星相对原始检测支点的二维屏幕误差P95。
- `ObservedInsideTargetPct`：准星中心位于检测框内的样本比例。
- `P95AbsAxisErrorPx/P95ErrorDistancePx`：预测后控制目标相对准星的误差，包含有意加入的预测前瞻，只用于解释控制请求，不能再当作实际跟踪误差。
- `OutputSideFlipCount/OutputSideFlipRateHz`：忽略零输出后，最终设备整数命令的换向次数和频率；与预测方向翻转分开审计。
- `SpeedLimitedPct`：持续移动阶段的软件速度上限占比。
- `ObservationAgeAvgMs/P95Ms`：传输和推理延迟背景。
- `MaxQueuedMoves`：必须继续不超过1；超过1先排查输入发送积压。
- `ReversalCount/RecoveredReversals/RecoveryMeanMs`：误差从一侧持续穿过中心进入另一侧的次数、回到±8 px并连续保持3帧的次数和平均恢复时间。
- `P95FilterResidualPx`：基础滤波对移动观测造成的滞后量。

## 首轮决策规则

第一轮用于建立真实基线，不预设脱离场景的绝对P95阈值，也不提前修改1440 counts/s。采集后按以下顺序决策：

1. 若队列深度大于1或观测年龄明显异常，先修复链路，不调整控制器。
2. 若 `SpeedLimitedPct` 高且误差随目标速度持续扩大，再单独评估最大设备速度；CUDA+UDP 静止数据已出现5.06 px保持误差，提速前必须确认反转不恶化。
3. 若不限速但 `ObservedP95AbsAxisErrorPx` 仍大，单独评估响应时间或基础滤波时间常数，不同时修改两者；不得根据含预测前瞻的 `P95AbsAxisErrorPx` 直接增加前瞻。
4. 若单向跟随可接受但反转恢复慢，先检查滤波残差和误差侧翻转，不直接接入预测。
5. NDI与UDP不得使用不同控制参数；差异必须连续三轮可重复，才能归因到传输链路。

## 后续范围

水平固定速度和反转测试通过后，再增加垂直、斜向和不同速度档位。基础移动目标闭环通过前，不评审未来预测和轨迹整形。静止九宫格不再作为日常主线任务；只有移动调参导致稳定性回归、修改坐标/控制器或准备正式发布时才恢复。
