# 固定延迟复测与连续响应修复

## 背景

r44 将正式 `ViewMotionHistory` 从发送时刻立即生效改为固定 12 ms 后完整生效。用户使用 `DML / 7453fb2433ad / r44` 和相同 120 ms 响应、3200 counts/s、500 ms 积分、50/50/1.0 预测配置完成 left/right 各三轮，肉眼仍可见轻微震荡。

## r44 复测结果

| 指标 | left | right |
|---|---:|---:|
| X真实稳态P95 | 12.00 px | 12.50 px |
| X输出换向 | 2 | 0 |
| 同方向步进P95 | 7.33 counts | 7.00 counts |
| 同方向脉冲 | 537 | 501 |
| 同方向脉冲频率 | 36.66 Hz | 34.05 Hz |
| 预测停止抑制 | 0% | 0% |
| 预测方向翻转 | 0 | 0 |
| Y输出换向 | 0 | 0 |

r43 的同方向步进 P95 为 7~8 counts、脉冲频率约 28~34 Hz。r44 没有降低幅值步进，left 频率反而升高；right 真实稳态 P95 由 10.17 px 回退到 12.50 px。固定延迟方案未通过，不能继续只调 10/12/14 ms。

## 原因与响应形状拟合

固定延迟只改变完整命令阶跃的相位，没有改变每次约 0.519 px/count 的整块注入。NDI 观测时间、游戏渲染边界和设备发送相位存在帧间抖动，即使平均延迟正确，单点阶跃仍会在相邻控制帧间形成补偿突变。

使用六轮 r44 的 `CommandDeviceSendNs`、`CommandAppliedCountsX`、`BackendReceiveNs` 和 `RawPivotX`，扫描以下模型：

```text
画面位移 = counts比例 × 区间响应counts + 目标恒定漂移 × dt
区间响应counts = 每条命令按 delayStart 后、riseMs 内线性从0增长到完整counts
```

扫描范围为起始延迟 0~30 ms、步长 2 ms，线性宽度 0~40 ms、步长 4 ms。六轮最优候选集中在：

- 起始延迟 0~8 ms；
- 线性响应宽度 20~28 ms；
- 多数轮次的最优宽度为 24 ms；
- 响应中心约 12~16 ms，与 r44 固定 12 ms 的中心位置一致。

这证明 r44 失败的主要原因是响应核形状，而不是中心延迟数量级错误。单向运动中目标漂移与命令长期同向，px/count 和漂移项仍有共线性，因此继续使用游戏 Profile 的完整补偿总量，只采用跨六轮稳定的响应时间形状。

## r45 设计

1. `aim_motion_compensation_delay_ms=12` 改为表示响应中心。
2. 新增 `aim_motion_compensation_response_ms=24`，表示完整命令线性出现的宽度；默认组合从发送时刻开始，在 24 ms 完成，12 ms 时应用一半。
3. `response_ms=0` 保留固定延迟阶跃兼容路径。响应宽度限制为 0~100 ms，中心限制保持 0~250 ms。
4. 每条命令的最终像素补偿总量不变，不修改 PI、预测、分轴锁存、限速、目标丢失和设备发送失败语义。
5. 历史裁剪只移除已经完成响应的旧样本，并把完整增量折叠进累计基线；部分响应样本不能提前折叠。
6. CSV 新增 `ViewMotionCompensationDelayMs` 和 `ViewMotionCompensationResponseMs`，复测文件自身必须证明实际使用 12/24 ms。
7. 控制器身份提升到 r45。

## 测试与验收

单元测试覆盖固定阶跃兼容、发送前零响应、响应中心一半、结束时完整响应、区间部分响应、裁剪累计、重配置清空、配置默认/保存/上限和 CSV 列一致性。

r45 使用同一 DML/NDI 配置完成 left/right 各三轮：

- CSV 每轮必须为 `ViewMotionCompensationDelayMs=12`、`ViewMotionCompensationResponseMs=24`；
- 同方向步进 P95 必须低于 r44 的 7.33/7.00 counts；
- 同方向脉冲频率必须低于 r44 的 36.66/34.05 Hz；
- X 真实稳态 P95 不高于 r44 的 12.00/12.50 px，输出换向不高于 2/0；
- Y 输出换向继续为 0，预测停止抑制和方向翻转继续为 0。

若连续响应只降低 `ViewMotionX` 阶跃而设备脉冲无改善，应停止修改自运动模型，转而检查控制器请求到整数余数之间的二帧极限环；不得继续扩大响应宽度掩盖其他环节。
