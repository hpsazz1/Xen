# P0-5静止真值与在途命令时轴

## 背景

r63门控常加速度候选的跨域矩阵为`721/810`，static仍有P95和输出翻转失败。逐帧明细显示，代表域在进入settled后没有新发送命令，但此前已发送的命令在`commandToFrameDelay`到达画面，导致物理目标相对误差再次增大。该现象不能通过清空pending命令解释：真实设备已经收到命令，清队列只会制造不可能的离线反事实。

## 本轮实现

跨域回放新增默认关闭的`--candidate-view-motion-compensation 1`。开启后，候选链路使用与正式`AppliedViewMotionModel`相同的时间语义：

1. 每个候选成功发送的counts按`degrees/count`和发送时刻加入视角历史。
2. 检测测量先形成相对画面角，再加回观测时刻的已发送视角，供Kalman估计。
3. 控制误差在控制时刻减去已发送视角，避免把在途命令误认为目标运动。
4. 目标丢失时不撤销已经提交的候选命令；默认关闭时保留旧回放行为，保证历史矩阵可复现。

逐帧`cross_domain_frames.csv`新增测量相对角、观测/控制时刻视角字段；summary和decision记录开关身份。`--static-fixed-truth 1`仍仅用于静止物理真值敏感性分析，默认关闭。

第二个默认关闭入口为`--candidate-command-commit-horizon-ms 60`。它把候选目标角按估计速率投影60 ms，同时查询已发送命令在该时刻必然兑现的累计视角，再把两者差值交给原控制器。该入口用于检验Smith型“已承诺终点”是否能消除settled后迟到位移，不修改Kalman、P响应、settle阈值、限速或门禁。

实现过程中发现一个离线时间轴陷阱：当观测延迟大于帧间隔时，上一帧`controlTime`可能晚于下一帧`observationTime`。可变`Camera`已前推后不能回退，因此观测相对角必须使用`AppliedViewMotionModel.at(observationTime)`历史查询；直接读取当前Camera会错误扣除未来相机角，并把矩阵压到396/810。该结果作废，只保留为排查记录。

## 验证

- 固定真值：总通过718/810，static 115/162；candidate P95均值由0.08159°降到0.05309°，但flips不变，净损失3项。
- 正式观测/控制时轴：总通过466/810；static仍为118/162、reverse仍为155/162、jump为157/162，但left/right由145/147降到15/21，失败均为单向P95回退。
- 60 ms提交时域：总通过431/810；jump/reverse均162/162，static P95均值降到0.0675°，但static因输出翻转增加只剩80/162，left/right因P95回退只剩10/17。
- 三组结果均为完整810项真实视频回放，输出分别保存在桌面Video目录的`analysis_p0_5_static_fixed_truth_on`、`analysis_p0_5_viewmotion_on_r63`和`analysis_p0_5_commit60_r63`。

## 结论与下一步

三项候选均正式否决并保持默认关闭。固定真值证明旧逐帧真值高估绝对误差，但不能绕过相对门禁；观测/控制时轴证明当前世界角回放在连续检测static中已与正式补偿代数等价；提交时域证明在途命令确实是静止尾部误差来源，但全时60 ms投影会把近中心量化方向和单向跟踪一起改变，不能晋级，也不继续扫描horizon。

active继续强制降级为shadow，运行控制器、配置和r63身份均不变。下一步只能设计“仅影响settle进入判定、不改变运动态P控制参考”的在途命令保护，并先证明static翻转不增加；在此之前不得把完整Smith投影接入运行时。
