# 短暂停恢复次帧预测空洞修复

## 背景

最新 DML NDI 复测数据位于 `C:\Users\16143\Desktop\Xen\DML\ndi`，包含左右水平移动过程中“松开瞄准、按住重新追踪”的操作。两份 CSV 均来自 DML、程序提交 `e05b8b6d7f49`、控制器修订号 r50。

用户观测为重新按住后不能第一时间稳定预测，准星位于移动目标身后。本轮需区分三个可能原因：预测状态未恢复、恢复后预测被安全门控撤销、松开期间积累的位置误差需要控制器追赶。

## 数据分析与原因

左右数据共识别出 16 次 350 ms 内的短暂停恢复，以及 3 次超过 1.1 s 的长暂停恢复：

- 16 次短暂停的重按首帧均已有同方向提前量，说明 r50 的同目标快速恢复入口生效；
- 16 次短暂停全部稳定复现“首帧有提前量、第二帧 `PredictionVelocityX=0` 且 `PredictionOffsetX=0`、第三帧恢复”；
- 第二帧的自运动、运动证据、静止、高速和振荡门控标志均为 0，排除安全门控误判；
- 重新按住时水平误差为 38.4～137.8 px，基础控制器的 `HorizontalCatchUp=1` 已正常进入追赶。最大误差样本同时出现速度限幅，继续全局缩短响应时间会增加饱和与闭环伪运动风险。

根因位于 `TargetPredictor` 的稀疏恢复生命周期。调用方仅在重按首帧把最大观测间隔临时放宽到 350 ms；预测器首帧会保留成熟速度，但第二帧调用恢复默认 100 ms 规则。此时回归窗口只有三个样本，旧实现直接把速度清零，第三帧凑齐四样本后才重新估速，形成确定性的一帧预测空洞。

## 设计与实现

预测器新增内部 `sparseResumeFramesRemaining_` 锁存：

1. 仅当首个跨暂停观测速度达到激活阈值的一半、方向与成熟方向夹角一致度不低于 0.50 时启动；
2. 在四样本回归接管前，最多保留两帧最后可信速度，覆盖首帧和随后的一个稀疏退化帧；
3. 第二帧允许检测量化造成的零位移，不再清空成熟速度与提前量；
4. 一旦观测表现为达到阈值的明确反向运动，立即取消锁存并清空旧速度；
5. 四样本最小二乘回归开始计算后立即结束锁存；完整重置和运动证据丢弃路径同步清零锁存。

本轮不修改 120 ms 基础响应、45～60 ms 水平追赶响应和 3200 counts/s 限速。这样只消除预测链路的固定空洞，不用更激进的控制输出掩盖问题。

控制器修订号由 r50 升级为 r51。

## 后续建议

使用 r51 重复左右水平移动和松开/重按操作。CSV 中短暂停恢复后的连续三帧应保持同方向非零 `PredictionOffsetX`；长暂停、目标切换和明确反向仍应重新建向。若准星仍落后，应单独按恢复时的初始误差分桶评估追赶时间，避免把暂停期间自然积累的位置差误判为预测冷启动。
