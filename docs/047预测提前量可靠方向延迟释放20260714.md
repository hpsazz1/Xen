# 预测提前量可靠方向延迟释放

## 背景与版本

2026-07-14使用 `BuildRevision=83059b23b01b`、`ControllerRevision=18`、DML+NDI、3200 counts/s和预测50/50/1.0完成static九段复测。NDI接收平均238.6 FPS，DML推理96.1 FPS，CSV包含 `FilterTrendActive`。

## r18结果

r18按启用时机限制趋势滤波后，static X/Y预测生效率仍为2.1%，平均P95误差为4.28/5.56 px，未恢复r14的0.4%。trial 8预测生效率为11.6%，其余有效试次为1.7%~4.2%。

`FilterTrendActivePct`与预测生效率逐试次完全一致，证明趋势滤波只在上一帧已有预测后开启，没有在初始收敛阶段提前介入。趋势滤波启用时机修正本身工作正确，但不是全部根因。

## 剩余根因

逐帧检查显示所有有效预测段只有1~3帧，随后即被自运动门控或停止证据撤销。预测器当前在稳健回归窗口连续两次同向后锁定方向，并立即输出提前量；内部已有 `predictionEstablished_` 状态，会继续要求三次可靠方向确认，但此前只用于决定窗口退化容忍度，没有参与输出门控。

因此static的短时补偿残差可以在“初步方向锁定”到“可靠运动正式建立”之间输出1~3帧提前。继续调整趋势滤波、几何门控或积分无法从根上消除这个时间窗口。

## r19设计

1. 方向初步锁定后继续维护速度和方向，但在 `predictionEstablished_` 成立前预测偏移保持为零。
2. 连续三次可靠同向方向确认后才正式释放提前量；约95 FPS DML下比r18增加约30 ms预测启用确认。
3. 确认期间基础控制器仍瞄准当前滤波位置，不会停止追踪或导致目标逃出检测区域。
4. 持续运动豁免的五帧计数从提前量正式释放后开始，不能使用尚未输出的初步锁定帧提前获得static豁免。
5. 停止、反向、自运动门控、趋势滤波和积分保持的既有逻辑不变。

控制器修订号升级为19。单元测试延长匀速观测序列，验证可靠方向建立后仍能按观测年龄输出预测，并保持延迟补偿、单轴吸附和持续运动豁免行为。

## 下一轮测试

- 先复测static九段，要求X/Y预测生效率不高于0.5%，不得出现1~3帧短时提前段。
- static通过后复测left/right各4轮，允许每轮开始约30 ms只有基础跟随；全轮预测生效率至少94%，X轴P95不明显回退于8.29/8.88 px。
- 持续预测阶段 `FilterTrendActive` 应接近预测生效率，自运动抑制低于5%。
- 两项通过后进入jump、reverse和 `Target_static_Role_reverse` 各20秒，以及三类目标丢失各10次。

## 验证要求

执行预测、滤波和控制器专项单元测试、分析脚本回归、DML/CUDA Release构建和两套完整CTest；中文提交并推送后按干净提交重新配置双后端，核对UI、CSV和可执行文件为同一提交且 `ControllerRevision=19`。
