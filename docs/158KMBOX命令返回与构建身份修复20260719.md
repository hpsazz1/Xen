# KMBOX 命令返回与构建身份修复

## 背景

孤立物理响应采集器的 CSV 曾将 KMBOX_NET 的 `CommandSucceeded` 记为成功，但该值只代表连接对象已打开，底层 `kmNet_mouse_move` 的返回码被丢弃。另有旧 CMake 构建目录未重新配置时，产物会继续写入旧的 `BuildRevision`。

## 修复

`KmboxNetConnection::move` 现在返回 `kmNet_mouse_move` 是否返回 0，`KmboxNetMouseInput` 将该结果传递到采集器。采集器的无效响应仍立即停止并保留已采集帧，避免在设备未响应时继续发送脉冲。

发布构建必须先运行正式构建脚本或重新执行 CMake 配置，再执行 Release 构建；仅对旧构建目录执行 `cmake --build` 不会刷新 Git 构建身份。

## 验证

本次修复需完成 DML/CUDA Release clean-first 构建、两端 CTest、CSV 身份核对，并确认 `CommandSucceeded=0` 时不会继续发送下一轮命令。
