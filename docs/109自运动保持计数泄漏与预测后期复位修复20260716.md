# 自运动保持计数泄漏与预测后期复位修复

## 背景与现象

r47使用DML、NDI、正式补偿12/0 ms和预测50/50/1.0完成left/right各三轮实机采集。控制周期修复有效：left/right最终输出同方向步进P95均由r46的7.33/8.00 counts降至3.00 counts，脉冲频率由35.4/34.0 Hz降至2.70/2.09 Hz，证明PI已不再把NDI成批观测间隔误作控制执行周期。

但实机仍观察到预测后期突然归零。正式分析器统计left/right分别有2/4次预测中断；排除轮次切换和采集结束后，内部中断均由`PredictionSelfMotionSuppressed=1`触发，随后进入`PredictionMotionEvidenceSuppressed=1`重新建向。典型样本是left第三轮Frame 2224：此前预测已连续有效10帧、约108 ms，X提前量从-13.2 px直接归零。

## 排查过程与根因

CSV显示故障前`PredictionDirectionLocked=1`、预测方向没有翻转、停止/高速/往返门控均未命中，控制周期也保持正常，因此排除方向锁、NDI周期混用和停止门控。

`TargetPredictor::applySelfMotionSuppression()`原有顺序先判断`artifactDetected && sustainedMotionConfirmed_`并直接返回，再处理`selfMotionSuppressionFramesRemaining_`。早期伪迹启动四帧保持后，预测器内部回归仍会继续累计连续预测帧；一旦内部持续运动锁存成立，后续伪迹帧会提前返回，使已有保持中途穿透，同时保留未消费的保持计数。伪迹暂时消失时，遗留计数又恢复执行，最终在成熟预测阶段突然撤销提前量，并在计数归零时丢弃运动学证据。

这解释了实测中“多帧自运动抑制—恢复预测—数十帧后单帧再次抑制—重新建向”的完整链路。r46加入的持续运动锁存只阻止了新伪迹门控，却没有约束已经启动的保持生命周期。

## r48设计与实现

状态机采用两个明确不变量：

1. 持续运动豁免只在没有活动保持时阻止新保持启动。
2. 一旦保持已经启动，伪迹持续期和其后四个响应尾迹帧必须连续抑制，直到计数归零；不得被内部持续运动锁存穿透，也不得把计数遗留到成熟预测阶段。

实现仅调整`applySelfMotionSuppression()`的判断顺序，不修改伪迹几何判定、四帧尾迹、方向建锁、预测参数、PI参数或自运动补偿模型。控制器修订号升级为r48，便于UI、CSV和构建产物区分。

单元测试新增早期保持与内部持续运动锁存交叠场景：在首次有效预测后启动保持，继续提供真实同向运动并持续命中伪迹，要求已有保持全程无穿透；伪迹结束后要求连续消费四个尾迹帧，下一帧不得再出现陈旧自运动抑制。
