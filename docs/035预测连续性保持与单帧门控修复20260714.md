# 预测连续性保持与单帧门控修复

> 状态：已被 `docs/036透视坐标修正与jump视野逃逸优化20260714.md` 取代。r9解决了持续预测中断，但暴露出线性FOV换算和2000 counts/s上限造成的静止误补偿与高速目标逃逸。

## 背景与版本

2026-07-14使用 `BuildRevision=720e661b9ed3`、`ControllerRevision=8`、DML+NDI和固定参数50 ms前瞻、50 ms速度窗口、1.0强度、2000 counts/s完成第二轮实测。六份文件版本和参数身份一致。除 `Target_static_Role_reverse` 为目标静止、角色移动外，其余场景角色自身静止。

## r8收益

r8的稳健回归方向有效。相对r7，持续left/right的X轴P95误差从约60 px降到37.98/38.27 px，reverse从28.24降到17.25 px，`Target_static_Role_reverse` 从58.96降到26.15 px。static的X轴预测生效率从26.7%降到2.8%，X轴提前P95从4.99降到1.10 px；持续左右的Y轴提前基本归零。

jump X轴P95从122.78回退到137.32 px，仍需后续单独处理高速响应；本轮先解决所有运动场景共同存在的预测连续性缺陷，避免同时改变速度预算或提前强度。

## 连续性根因

r8虽然使用50 ms窗口回归速度，但方向状态机仍保留了“相邻两帧坐标差分反向就立即撤销预测”的旧规则。该单帧差分会受到检测框量化、闭环移动残差和观测间隔波动影响，绕过稳健窗口直接把提前量清零。

正式分析新增预测中断和连续运行长度后，旧数据结果如下：left/right分别发生215/222次有效到无效中断，中位连续运行仅5/4.6帧；jump中断189次、中位4帧；reverse中断129次、中位7帧；`Target_static_Role_reverse` 中断155次、中位9帧。持续左右方向锁定率实际约98%，但约12%的帧在方向仍锁定、回归速度仍充足时被单帧门控清零，形成肉眼可见的开关式晃动。

## r9设计

1. 删除单帧坐标差分直接判定反向的控制路径。停止和反向均只由稳健回归窗口与连续状态确认决定。
2. 不可靠窗口只更新加速度噪声诊断，不覆盖最后一次可靠控制速度；已锁定方向最多跨过两帧短暂窗口退化，第三帧才撤销、第五帧才释放方向锁定。
3. 单帧低速不再切断已建立提前；连续两帧低速才撤销，连续三帧才释放方向。按当前约93~106 FPS，相对旧首帧规则只增加约10 ms确认时间。
4. 回归方向首次确认反向时撤销旧侧提前，连续两次可靠反向后切换新侧。预测不会在未经窗口确认时左右翻转。
5. 目标丢失、目标ID切换、瞄准状态变化和异常时间戳仍立即清空，不使用保持期，避免把旧目标速度带入新目标。

控制器修订号升级为9。单元测试覆盖单帧反向异常不得打断、稳健反向必须先撤销旧侧再建立新侧、单帧低速保持、连续两帧停止撤销、静止噪声不得启用和水平运动不得生成Y提前。

## 新分析指标

`tools/analyze_moving_target.ps1` 新增：

- `PredictionInterruptionCount`：一次有效预测后转为无提前的次数，不把初次预热计为中断。
- `P50PredictionActiveRunFrames`：每段连续有效预测的帧数中位数；场景汇总输出 `MeanP50PredictionActiveRunFrames`。

生效率只能说明有效帧占比，无法识别“一帧开、一帧关”；后续必须同时核对中断次数和连续运行长度。

## 下一轮测试

保持参数 `50/50/1.0/2000` 不变，只更换r9程序：

- left/right各5轮，目标连续移动，角色静止；单轮 `PredictionInterruptionCount<=10`，场景 `PredictionActivePct>=95%`，`MeanP50PredictionActiveRunFrames>=30`，预测侧翻转为0。
- reverse和jump各20秒；中断次数相对r8至少下降70%，预测侧翻转不得多于真实反转次数。
- `Target_static_Role_reverse` 20秒；中断次数相对155至少下降70%。
- static九段；X/Y预测生效率不高于5%，X/Y提前P95不高于2 px，不允许以连续性保持重新引入静止漂移。
- 目标飞出、目标死亡、角色死亡各10次；丢失后下一帧不得继续输出预测。

连续性安全项通过后，再单独处理jump高速误差；本轮不调整预测强度、前瞻时间或设备速度，避免无法判断收益来源。
