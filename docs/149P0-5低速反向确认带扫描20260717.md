# P0-5低速反向确认带扫描

## 背景与证据

r63 已完成 static、jump 和用户当前最快 reverse 的 DML/NDI 实机影子门禁，但门控 CA 跨域回放仍为 `721/810`，active 必须继续强制降级为 shadow。为避免继续扩大 CA 或修改真实运动释放，本轮只分析代表性 static 翻转失败域 `domain_fov120_2560x1440_60fps_40ms_0.75x`。

该域逐帧 trace 共 2564 帧，候选 settled 2494 帧，发生 9 次 settle release、12 帧低速反向抑制、50 个非零发送和 9 次输出翻转。9 次释放的相对 LOS 速率为 0.375~1.115 度/秒，全部低于 1.2 度/秒；由 `FeedbackX/Y` 反算的控制器输入误差为 0.1206~0.1389 度，全部刚超过固定的 `0.08 × 1.5 = 0.12` 度退出边界且不超过 0.14 度。

`CandidateError` 是控制后的真值误差，不是控制器使用的估计误差，不能直接用于 settle release 归因。证据表明问题不是 CA 驻留或速率尖峰，而是 settled 在估计误差超过 0.12 度时正确释放后，低速反向确认也使用同一个 1.5 倍误差上限，导致 0.12~0.14 度的反向脉冲绕过确认。

本轮使用用户新上传的 `build/dml/Release/models/yolo12n_cs2.onnx`，文件大小 10504355 字节，SHA-256 为 `525D3CF4F7317BF6A725877443A2E49EF3DD4A84CBC34F977842C5B057C8FDB3`。模型位于被忽略的构建运行目录，不纳入 Git 提交；正式扫描仍需在 decision 中核对模型、程序和参数身份。

## 最小设计

`LosAimController::Settings` 新增 `reverseConfirmationErrorMultiplier`，默认 1.5，允许范围 1.5~2.0。静止锁存退出仍固定为 1.5 倍，真实 LOS 速率达到 1.2 度/秒时仍立即旁路，只有已经退出 settled、误差仍处于窄带且方向反转的低速请求需要持续 80 ms 才放行。

离线入口为 `--reverse-confirm-error-multiplier`。该参数只由跨域视频回放 CLI 传入，不加入 `config.ini`、UI 或正式 active 配置；summary 和 decision 显式记录实际倍率。默认 1.5 必须精确复现历史 `721/810`，之后只扫描 1.75 和 2.0，不扩大范围。

单元测试分别锁定以下契约：默认 1.5 倍时区外反向请求立即放行；1.75 倍时 0.13 度低速反向请求进入确认；即使配置 1.75 倍，settle 锁存仍在超过 0.12 度的同一帧释放。

## 晋级门槛

三组倍率使用相同模型、视频、检测队列和冻结的 `maneuver_gated_ca + 8000/12/120` 参数运行完整 810 项矩阵。候选只有同时满足以下条件才可进入正式 DML shadow 讨论：

1. 1.5 倍精确复现 `721/810`，证明新增入口没有改变默认行为。
2. 1.75 或 2.0 对原 721 个通过项零损失，并减少 static 输出翻转失败。
3. jump、reverse、left 和 right 不出现任何新增失败，static P95 也不回退。
4. 结果不满足零回退时立即否决，保持默认 1.5 和 `HOLD_SHADOW`，不继续放宽 settle 或低速确认带。
