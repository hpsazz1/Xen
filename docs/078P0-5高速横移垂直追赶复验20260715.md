# P0-5高速横移垂直追赶复验

## 背景与排查

低速反向确认后矩阵为732/845，jump仍有40组因垂直P95未比旧链路改善10%而失败，且失败集中在1.5倍速域和`role_opposite`。基准逐帧数据中，候选垂直误差最差5%的59帧有66.1%受到二维速度限幅：Y轴平均反馈需求9.33 counts，限幅后只保留6.05 counts，而X轴同期平均占用12.17 counts。根因是高速横移占用大部分二维预算时，角度控制器缺少旧r30已有的大误差轴向追赶，垂直修正长期被同比缩小。

## 设计

在`LosAimController`中加入可关闭的角度域垂直追赶，但严格限制生效条件：

- 垂直角误差至少0.8°；0关闭功能。
- 水平相对LOS速率至少5°/s，只处理真实高速横移。
- 垂直误差至少达到水平误差的15%，防止微小Y残差从单向水平跟踪中抢占预算。
- 生效时沿用r30连续曲线：有效响应时间从阈值处的基础响应0.5倍，平滑缩短到两倍阈值处的0.375倍；二维总速度预算、Kalman、前馈、积分和轨迹执行层不变。

正式配置只暴露`aim_shadow_vertical_catch_up_deg`，默认0.8°、范围0~20°；5°/s和15%是防误触安全边界，不继续增加UI参数。流水线CSV记录生效标记与Y轴有效响应时间，跨域汇总记录生效占比。

## 参数反证

全局0.8°垂直追赶会把矩阵降到626/845，static产生明显振荡，证明不能只按误差放大增益。加入5°/s水平速率门控后可达751/845，但0.8°、10%误差比例仍使两组left轻微回退；把比例提高到20%虽消除回退，却只剩739/845。最终15%比例在不回退四类既有场景的前提下取得最大净收益。

最终命令在上一阶段参数后增加：

```powershell
--vertical-catch-up-deg 0.8
```
