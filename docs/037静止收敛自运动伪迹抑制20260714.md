# 静止收敛自运动伪迹抑制

## 背景与版本

2026-07-14使用 `BuildRevision=6ae7b53b6d86`、`ControllerRevision=10`、DML+NDI、3200 counts/s和预测50/50/1.0完成第四轮实测，全程未使用手动鼠标辅助。六份CSV版本和参数身份一致。

## r10收益

透视坐标与速度范围修正收益明确：horizontal_jump首次形成单个21.65秒完整试次，X轴P95从r9碎片试次均值123.09 px降到52.68 px；left从34.29降到22.26 px，right场景均值19.97 px，`Target_static_Role_reverse` 从30.60降到17.53 px。目标不再贴边丢失，说明3200 counts/s与透视控制换算解决了320裁剪视野逃逸。

r10尚未通过静止安全项。static X/Y预测生效率为15.3%/19.9%，提前P95为4.76/6.48 px；部分静止试次仍出现明显上下或左右提前。right第三轮预测几乎未生效但X P95仍为14.56 px，属于单轮运动证据不足，不影响本轮静止根因判断，下一轮继续保留该场景审计。

## 剩余根因

static不是“目标从一开始就在准星中心”，而是静止目标出现在不同位置后，程序主动旋转视角把它拉回中心。即使透视比例正确，检测时间戳、设备实际响应、像素量化和模型框中心变化仍会产生少量运动补偿残差。r9为保证连续预测又允许短暂使用最后可靠速度，因此该残差一旦通过运动确认，就可能持续数帧。

数据存在可验证的联合特征：static有效预测中约66%的预测速度与当前误差同向，即预测声称目标继续向准星外移动；与此同时目标实际屏幕位置正在向中心收敛，且程序自身视角移动与误差同向。持续left/right和jump出现同类特征的比例仅约12%~20%。因此不能只看预测速度或误差方向，必须联合判断屏幕收敛、自身视角移动和预测方向。

## r11设计

1. MouseThread保存最近三帧“观测时刻目标屏幕坐标”和“程序累计视角位移”，使用约30 ms窗口而非单帧差分判断收敛。
2. 仅在以下条件同时成立时撤销本帧提前：目标距中心至少4 px；程序视角朝当前误差方向移动；目标屏幕位置明确向中心收敛并与程序视角反向；预测速度却与当前误差高度同向；预测速度至少64 px/s。
3. 门控只把本帧预测偏移归零，不重置稳健速度和方向状态。真实移动若短暂满足条件，下一帧可立即恢复，不重新引入r8的长时间预测中断。
4. 新增CSV字段 `PredictionSelfMotionSuppressed`，分析器输出 `PredictionSelfMotionSuppressedPct`。下一轮可直接验证门控是否主要命中static，而不是通过误差变化间接猜测。
5. 目标丢失、ID切换和瞄准状态变化仍立即清空预测与三帧上下文；手动鼠标输入仍禁止进入标准测试。

控制器修订号升级为11。单元测试覆盖自运动拉回中心时撤销、真实目标向外运动时保留、CSV字段、旧CSV兼容和UI版本身份。

## 下一轮测试

参数保持 `move_max_speed_cps=3200`、预测 `50/50/1.0`，不手动移动鼠标：

- 首先只采集static九段。要求X/Y `PredictionActivePct<=5%`、提前P95不超过2 px；`PredictionSelfMotionSuppressedPct` 应明显大于0，证明门控实际工作。
- static通过后再采集left/right各4轮。要求X误差不高于r10的22.26/19.97 px，预测生效率至少95%，自运动抑制比例建议低于5%。
- jump连续20秒，要求仍为单个至少15秒有效试次、X P95不高于60 px、自运动抑制比例低于10%。
- reverse和 `Target_static_Role_reverse` 各20秒；重点核对抑制只发生在回中心阶段，不增加预测侧翻或目标丢失。
- 目标飞出、目标死亡、角色死亡各10次，丢失后无输出。

如果static仍不达标，下一步根据新字段与三帧上下文增加正式诊断，不再继续修改速度、FOV换算或jump参数。
