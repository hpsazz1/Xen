# 高频反转框内保持与jump双轴追赶

## 背景

r24复测中，用户仍观察到reverse左右晃动且准星不在检测框内，jump仍跟不上。数据来自构建`088056a1125b`、控制器r24。

## r24结果

### reverse

- X/Y P95为12.37/3.16 px。
- 高频往返抑制94.6%，预测生效率2.4%，说明预测换边已经不是主要问题。
- 水平准星99.5%的帧位于检测框内，但控制器仍持续追踪框内pivot，因此肉眼仍看到左右运动。
- 完整二维框内率只有76%；主要是准星位于框顶上方1~4 px，Y轴框内率76.5%。
- 原始pivot 100%位于检测框内，滤波及预测点95.7%位于框内，检测框数据本身有效。

结论：用户要求的是“高频慢速目标保持框内”，而不是继续精确追逐框内pivot。即使误差数值不大，追pivot仍会制造无必要的左右视觉晃动。

### horizontal_jump

- 预测点X/Y P95为75.15/16.88 px，目标框中心P95为72.60/15.98 px，预测已不是主要误差来源。
- 高速瞬态抑制19.3%，往返抑制0%，速度分流符合设计。
- Y轴追赶命中6.2%，Y误差已明显改善。
- 总速度限速5.7%，X轴仍保留80 ms基础响应，成为当前主要跟随瓶颈。

## 设计

### 高频反转框内保持

- 仅在`PredictionOscillationSuppressed`生效时启用，不影响普通static、left/right和jump。
- 把观测时刻检测框补偿到控制时刻，与预测点使用同一视角运动坐标系。
- 准星已经位于框内时，控制目标直接设为当前准星中心，停止追逐pivot。
- 准星位于框外时，只追到最近边缘内缩2 px的位置，不追目标中心。
- X/Y同时应用，解决框顶轻微越界和水平框内摆动。

### jump双轴追赶

- 保留16 px阈值和40 ms快速比例响应。
- X/Y分别判断，只有超阈值的轴缩短响应；另一轴仍保持80 ms。
- 积分、稳定半径、预测门控和3200二维总速度不变。
- 新增`EffectiveResponseSecondsX`、`HorizontalCatchUp`与分析汇总`HorizontalCatchUpPct`。

## 验证方案

- 单元测试覆盖框内返回准星中心、左右越界返回最近内边缘，以及X/Y追赶独立触发。
- DML/CUDA Release及完整CTest通过，UI、CSV、EXE统一显示r25与同一干净提交。
- reverse复测要求高频抑制继续命中；进入框内后`PredictedX/Y`应等于160或最近内边缘，二维框内率显著高于76%，肉眼不再追着pivot左右摆动。
- jump复测要求`HorizontalCatchUpPct`和`VerticalCatchUpPct`均大于0，X/Y P95低于75.15/16.88 px，目标不丢失，限速不得持续饱和。
- 专项通过后回归static九段与left/right各4轮，确认框内保持没有泄漏到普通预测模式。
