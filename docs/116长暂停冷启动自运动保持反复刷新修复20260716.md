# 长暂停冷启动自运动保持反复刷新修复

## 背景

最新复测数据位于 `C:\Users\16143\Desktop\Xen\DML\ndi`，包含右移 1192 行、左移 1221 行，构建身份为 DML、`bcc1b645700d`、`20260716T113553Z`、控制器 r53。配置仍为 120 ms 基础响应、3200 counts/s 上限、50 ms 预测提前和 50 ms 速度窗口。

本轮首先验证 r53 的短暂停追赶尾迹保护，再排查仍然存在的预测启动延迟。

## 数据分析

共识别 18 次 350 ms 内短暂停恢复和 4 次长暂停恢复：

- 18 次短暂停恢复后前 150 ms 的 `PredictionOffsetX` 归零、换向、`PredictionHighSpeedSuppressed`、`PredictionMotionEvidenceSuppressed` 和 `PredictionSelfMotionSuppressed` 均为 0；
- 150～400 ms 内也没有延迟复位，说明 r53 已完整修复追赶补偿尾迹误门控；
- 中小积压仍能在 60.7～116.4 ms 进入 16 px，大积压受 3200 counts/s 上限和持续目标速度影响，不应继续全局提高控制增益；
- 4 次长暂停中，3 次在 83.1～91.6 ms 建立首次提前量；右移 Frame 392 在 1156.5 ms 暂停后却延迟到 459.4 ms；
- 异常段在约 50 ms 已得到正确正向预测速度，随后 `PredictionSelfMotionSuppressed` 连续出现 26 帧，具体形成 Frame 397～412 的 16 帧保持和 Frame 420～429 的 10 帧保持；
- 该段正式分析器的观测稳态 P95 误差为 22 px、框内比例 81.6%，其余右移轮次观测 P95 为 5～13.5 px且框内比例 100%。

根因是自运动保持启动后，每个连续伪迹都会把四帧尾迹计数重新写回 4。内部稳健回归虽已连续确认真实单向运动，但由于保持仍在活动，既不能使用持续运动豁免，也不能消费尾迹，最终反复丢弃运动证据并重新建向。

## 设计与实现

控制器修订号升级为 r54，自运动保持调整为：

1. 早期伪迹仍启动四个后续观测的安全保持，保持尾帧结束前始终撤销提前量；
2. 保持期间若内部预测连续三帧成熟，并且原始伪迹条件仍持续存在，则把它标记为真实闭环跟随被误判；
3. 标记后不再用新伪迹刷新计数，连续消费剩余尾迹；
4. 尾迹结束时保留成熟方向与可信速度，下一帧恢复提前量；
5. 若伪迹已经消失才在保持中形成运动，仍按原逻辑丢弃证据并要求超过 8 px 的新净位移重新建向，避免静态目标回弹泄漏预测；
6. 已成熟运动在没有活动保持时仍直接豁免新自运动门控，停止、目标丢失和显式重置仍清空全部状态。

## 单元测试

- 早期保持遇到持续单向运动和持续伪迹时，至少完整抑制四帧尾迹，随后在八个样本内有界恢复并保留正向提前量；
- 普通四帧尾迹仍完整执行且无计数残留；
- 伪迹结束后的不足 8 px 回弹仍不能重新建立预测；
- 持续运动豁免、停止后重新开放门控、短暂停追赶保护、真实反向、高速和振荡测试保持通过；
- UI 与流水线 CSV 控制器身份断言升级为 r54。

## 复测建议

- 使用 r54 重复长于 350 ms 的松开/重按，持续同方向移动时首次非零 `PredictionOffsetX` 应恢复到约 80～120 ms；
- `PredictionSelfMotionSuppressed` 可以短暂出现，但单次应有界，不得连续形成 10～26 帧保持或重复丢弃同方向运动证据；
- 同时保留短暂停复测，前 150 ms 不得再次出现提前量归零或任何预测门控；
- 增加静止目标和停止操作，确认静态控制回弹仍被抑制且不会泄漏旧方向提前量。
