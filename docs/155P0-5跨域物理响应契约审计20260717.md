# P0-5跨域物理响应契约审计

## 背景

r63实机影子观测使用命令响应中心20 ms、完整响应宽度20 ms，而跨域闭环基准一直使用命令发送后60 ms整步生效的物理相机。两者都以成功发送的counts描述自运动，但用途不同：实机模型用于解释检测帧中的已发生视角运动，跨域相机则参与生成后续检测、裁剪区可见性和legacy/candidate闭环轨迹。因此不能仅凭参数同名就假定两套模型等价。

本轮给跨域回放增加默认关闭的离线入口：

- `--physical-command-center-ms`：响应中心，范围0~250 ms，默认60 ms；
- `--physical-command-response-ms`：完整响应宽度，范围0~100 ms，默认0 ms；
- 宽度为0时保持原有60 ms整步语义；宽度非零时，legacy与candidate共用同一线性有限响应物理模型；
- `cross_domain_summary.csv`逐行记录`CommandCenterMs`与`CommandResponseMs`，`cross_domain_decision.txt`记录相同的归一化实际值。

这些参数只属于离线契约审计，不接入`config.ini`、UI、正式shadow或active。控制器修订保持r63。

## 实现语义核对

有限响应相机复用`AppliedViewMotionModel`。每条成功提交的命令从“中心减半宽”时刻开始，在完整宽度内线性生效；模型的`at()`按查询时刻累计历史响应，因此同一循环先查询观察时刻、再推进到控制时刻不会丢失历史状态。

目标丢失时，已提交给物理相机的命令不能撤销。有限响应模式下`clearPending()`不删除这些命令；legacy与candidate各自拥有独立状态，但配置完全相同。默认宽度为0时仍走原有pending队列，保留既有基准结果和历史重置语义。

## 完整矩阵结果

冻结r63算法和其余物理域参数，使用相同五场景、每场景162域运行三组810项矩阵：

| 场景 | 60/0基线 | 20/20 | 20/20且static固定真值 |
| --- | ---: | ---: | ---: |
| jump | 156/162 | 162/162 | 162/162 |
| left | 145/162 | 6/162 | 6/162 |
| reverse | 155/162 | 160/162 | 160/162 |
| right | 147/162 | 4/162 | 4/162 |
| static | 118/162 | 130/162 | 128/162 |
| 合计 | 721/810 | 462/810 | 460/810 |

20/20下left有156项、right有158项因“候选P95相对legacy回退”失败。候选的平均P95实际上由left 7.6870度降至6.6566度、right 7.6151度降至6.5561度，但legacy下降更多，分别达到6.3428度和6.2291度。说明失败来自比较基准改变后的相对优势消失，不是候选绝对误差单调恶化。

jump、left、right分别有144、149、160个域的有效检测样本数变化，合计样本变化为+20592、+539、+913；reverse和static样本队列不变。物理响应进入裁剪区闭环后会改变检测队列，因此20/20结果不能作为只替换实机自运动参数的固定样本反事实。

static固定真值只把static通过数由130降至128，动态四场景和样本数完全不变。这排除了逐帧检测中心噪声是20/20整体回退主因的解释。

## 验证与构建注意事项

提交前使用仓库正式脚本对DML和CUDA执行Release全量清洁构建，两端均成功；随后两端CTest均为9/9通过，新增单元测试确认有限物理响应会同时改变legacy和candidate闭环，而不是只移动候选比较器。

直接从未初始化VS开发环境的PowerShell执行`cmake --build ... --clean-first`会导致MSVC找不到标准库与Windows SDK，典型错误为`vector`、`windows.h`不存在。这不是源码失败。规范入口是`tools/build_dml.ps1`和`tools/build_cuda.ps1`，两者会先导入`VsDevCmd.bat`环境，并按项目规则清洁重建应用目标。

## 结论与下一步

实机shadow的20/20自运动模型参数不能直接等价为跨域闭环物理对象。跨域正式基准继续冻结60/0，新增入口保留为默认关闭的离线契约反证工具；不得用20/20矩阵替换721/810基线，也不得据此调整门禁、扫描horizon或覆盖局部损失。项目继续`HOLD_SHADOW`。

下一步不再修改settle局部状态机。应先用可独立测量的输入到画面响应数据建立物理对象：固定相机与目标，发送孤立counts脉冲并以高帧率记录画面角位移，分别拟合响应起点、中心、宽度和尾部；随后验证该模型能同时解释实机影子帧与跨域闭环检测队列，再决定是否更新60/0基准。
