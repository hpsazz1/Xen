# P0-5 剩余失败聚类与参数冻结

## 1. 背景

`docs/084P0-5Kalman时间轴与自适应过程噪声修复20260715.md` 选定自适应过程噪声 `90→360°/s²`、速度阈值 `8°/s`，统一关闭速度前馈和经验提前参考，跨域矩阵达到 `688/810`，仍为 `HOLD_SHADOW`。本轮只分析剩余 122 项失败并做有界反证，不修改门槛，不以继续扫参代替根因判断。

实战范围仍以“目标运动/角色静止”和“目标静止/角色静止”为主；角色运动组合暂不作为阻断条件。

## 2. 聚类结果

### 2.1 按场景

| 场景 | 通过 | 失败 |
|---|---:|---:|
| jump | 136/162 | 26 |
| left | 145/162 | 17 |
| reverse | 142/162 | 20 |
| right | 147/162 | 15 |
| static | 118/162 | 44 |

### 2.2 按物理域

| 维度 | 失败集中情况 |
|---|---|
| 速度倍率 | 1.5× 失败 59 项；jump 的 26 项和 reverse 的 20 项全部为 1.5× |
| 观测延迟 | 40 ms 失败 63 项；单向 P95 失败 32 项中有 30 项位于 40 ms |
| 帧率 | 60 FPS 失败 55 项；17 项 static P95 失败全部位于 60 FPS |
| FOV | 120° 失败 54 项，90° 失败 43 项，106° 失败 25 项 |
| 分辨率 | 1920×1080 失败 65 项，2560×1440 失败 57 项，没有数量级差异 |

主边界因此不是分辨率，而是高速机动、40 ms 延迟和低帧率静止收敛。

## 3. 门槛实际差距

| 失败原因 | 数量 | 实际差距与伴随收益 |
|---|---:|---|
| 单向 P95 回退 | 32 | P95 平均仅回退 `0.084°`、平均比值 `1.011`，最坏比值 `1.044`；输出方向翻转平均减少 `11.09` 次 |
| jump 垂直 P95 未改善 10% | 26 | 候选 P95 平均仍改善 `0.191°`，平均比值 `0.981`，即平均改善约 1.9%；输出翻转平均减少 `4.50` 次 |
| reverse 未满足 P95 10% 或框内 +10 点 | 20 | P95 平均改善 `0.138°`，平均比值 `0.932`，即平均改善约 6.8%；输出翻转平均减少 `19.40` 次 |
| static P95 回退超过 5% | 17 | 绝对 P95 平均只增加 `0.010°`，但旧基线极小使相对门槛敏感 |
| static 输出翻转增加 | 27 | P95 平均比值 `0.711`，即精度平均改善约 28.9%；方向翻转平均增加 `2.11` 次，仍按安全门槛判失败 |

这些失败不构成跨域数量级退化，但正式晋级门槛是硬约束，不能因差距较小而改写为通过。

## 4. 有界反证

### 4.1 静止端 Q=45

保持运动端 Q=360、阈值 8°/s，仅把静止端 Q 从 90 降到 45：

| 候选 | 总通过 | jump | left | reverse | right | static |
|---|---:|---:|---:|---:|---:|---:|
| Q=90→360 | 688 | 136 | 145 | 142 | 147 | 118 |
| Q=45→360 | 682 | 130 | 144 | 137 | 147 | 124 |

static 增加 6 项，但 jump/reverse 分别减少 6/5，总体回退，否决。

### 4.2 静止误差阈值 0.10°

保持 Q=90→360，只把静止锁存误差从 `0.08°` 放宽到 `0.10°`：总通过降到 `669/810`，其中 static 从 118 降到 99。原因是更早锁存保留了较大残差，而不是更稳定地归零，否决。

## 5. 冻结结论

1. 冻结 Q=`90→360°/s²`、阈值 `8°/s`、静止误差 `0.08°`、静止速率 `1.2°/s`。
2. 保持速度前馈、经验提前和轨迹整形关闭；旧 `742/810` 不再使用。
3. 不继续扫描 Q、静止阈值或放宽门槛；剩余失败按高速机动、40 ms 延迟和低帧率静止三类保留。
4. P0-5 仍为 `HOLD_SHADOW`，禁止正式 active；允许进入 P0-6 第一步，只采集 DML/NDI shadow CSV，验证实机时间分解、NIS、协方差和回放方向一致。
5. 若实机 shadow 证明 40 ms 观测延迟是主导瓶颈，下一控制变量应是 P 反馈响应时间与实际发送能力；不能重新用经验提前量掩盖延迟。
