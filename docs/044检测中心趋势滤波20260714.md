# 检测中心趋势滤波

## 背景与版本

2026-07-14使用 `BuildRevision=15f46ce1f0a5`、`ControllerRevision=16`、DML+NDI、3200 counts/s和预测50/50/1.0完成left/right复测。NDI接收平均约238 FPS，DML推理约94 FPS，用户反馈明显震荡已经减小，仅剩轻微晃动。

## r16收益

r16预测期间积分保持收益显著。left/right X轴平均P95误差从r15的22.61/21.68 px降至13.31/9.25 px，预测生效率保持97.7%/98.6%，自运动抑制仅0.6%/0.3%。最终设备输出换向在多数轮次降为0次，用户主观观察与数据一致。

## 框宽假设验证

检测框宽度在不同姿态和距离下约31~64 px变化，但宽度帧差与原始枢轴X帧差的相关系数仅为-0.025/-0.027，几乎没有线性关系。宽度变化与“枢轴相对身体框中心”的变化相关也只有-0.233/-0.164，不能解释剩余水平晃动。

当前基础锁定器在头身合并后优先使用头部检测支点，因此 `RawPivotX` 通常不等于身体框中心。枢轴相对身体框中心的帧间变化P95约1.5 px，原始枢轴帧差P95约2.5~3.0 px。剩余轻微晃动来自检测中心和头部支点的小幅跳变，不应使用身体框宽度反推或修正支点，否则遮挡、只露头部和姿态变化会重新引入错误。

## 滤波根因

`BasicTargetFilter` 原先直接使用相邻两帧的瞬时速度控制滤波时间常数：速度越高，时间常数从30 ms降至6 ms。单帧检测中心跳变同样会产生数百px/s瞬时速度，滤波器因此在最需要抑制噪声时反而快速放行。

## r17设计

1. 保留CSV中的原始 `ObservedVelocityX/Y` 和 `ObservedSpeed`，继续用于诊断真实单帧观测。
2. 新增约30 ms有符号速度趋势，对X/Y速度分别低通后再计算模长；只有持续同向运动才能形成较高趋势速度。
3. 自适应滤波时间常数改由趋势速度控制。交替方向的框中心跳点会互相抵消并保持较强平滑，持续横移仍逐步打开到原有高速响应。
4. 新增CSV字段 `FilterTrendSpeed`，下轮可直接比较瞬时速度与滤波实际采用的运动趋势，避免再次依靠主观判断。
5. 不使用目标框宽高估算支点，不修改预测、积分、3200速度、透视换算和丢失策略。

控制器修订号升级为17。单元测试覆盖交替3 px高速度跳点保持窄滤波范围、趋势速度低于120 px/s，以及持续300 px/s横移的滤波滞后小于8 px、趋势速度高于250 px/s。

## 下一轮测试

- 保持参数不变，复测left/right各4轮并记录肉眼轻微晃动是否继续下降。
- left/right X轴P95不得回退于r16的13.31/9.25 px；预测生效率至少95%，自运动抑制低于5%。
- 对比CSV中的 `ObservedSpeed` 与 `FilterTrendSpeed`：短时检测跳点应表现为瞬时速度峰值高、趋势速度明显较低；持续横移时趋势速度应稳定。
- 补static九段，要求X/Y预测生效率不高于0.5%，且基础误差不因趋势滤波新增明显滞后。
- 通过后进入jump、reverse和 `Target_static_Role_reverse` 各20秒，以及三类目标丢失各10次。
