# 生产链越心积分解卷绕

## 数据证据

最新 r32 数据使用 `move_integral_time_ms=500`。分析显示 left/right 的 `FinalMx` 方向翻转约 3.5~3.9 Hz，`IntegralCountsX` 翻转约 2.9~3.5 Hz；两者同步。jump 预测激活仅 8.3%，P95 误差回升到约 44 px，继续拉长积分窗口会牺牲跟随而不能消除晃动。

## 原因

生产链在预测偏移有效时设置 `preserveMovingIntegral=true`，允许积分跨越准星中心继续保留。检测框轻微越心或预测撤销时，旧侧积分会被带入新侧，形成周期性反向输出。独立控制器仍保留该选项，由专项测试覆盖；生产链不应默认启用。

## 实现

生产 legacy 链固定 `preserveMovingIntegral=false`，越过稳定半径后立即解卷绕，由当前误差和预测重新建立积分。`move_integral_time_ms=500`、`3200 cps`、80 ms 响应和 50/50/1.0 预测保持不变。控制器修订号升至 r33。

## 验证

下一轮重点比较 `FinalMx` 与 `IntegralCountsX` 的翻转频率、left/right 稳态误差、jump 预测激活率和 reverse 低速输出。若翻转仍同步，下一步再检查检测框中心滤波，不继续盲调积分窗口。
