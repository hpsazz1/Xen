# 趋势滤波移动验收通过

## 背景与版本

2026-07-14使用 `BuildRevision=a76c1e4ba53d`、`ControllerRevision=17`、DML+NDI、3200 counts/s和预测50/50/1.0完成left/right复测。CSV包含新增 `FilterTrendSpeed` 字段，NDI接收平均约238 FPS，DML推理约93~94 FPS，全程未使用手动鼠标辅助。

## 移动验收结果

left/right X轴平均P95误差为8.29/8.88 px，相比r16的13.31/9.25 px继续改善37.7%/4.0%，明显优于原验收线22.26/19.97 px。预测生效率为99.0%/98.8%，自运动抑制为0%/0.1%，趋势滤波没有造成预测中断。

设备最终输出换向每轮仅0~1次，说明r16积分保持后的稳定输出仍然成立。用户未报告新的明显震荡，本轮数据也未出现跟随速度不足或周期性反向控制。

## 趋势诊断

持续横移的瞬时 `ObservedSpeed` 中位约249~267 px/s，但P95达到737~1338 px/s，证明检测中心仍有明显单帧跳点。`FilterTrendSpeed` 中位约241~254 px/s、P95稳定在302~319 px/s，能够保留真实持续速度并拒绝异常尖峰。

按“瞬时速度大于200 px/s且超过趋势速度1.75倍”统计，约11.2%~14.4%的观测属于被趋势滤波降权的短时尖峰。该比例与模型检测中心跳动相符；趋势速度没有跟随这些峰值放大，因此r17设计目标已实现。

`FilterResidual` P95约7.4~7.6 px，其中包含持续移动下低通位置与当前观测之间的正常差值，不能直接解释为7 px静态抖动。闭环X轴P95反而继续下降，说明该残差没有形成额外跟随滞后。

## 决策

left/right移动阶段正式通过并冻结r17参数。检测框宽度与水平枢轴跳动缺少相关性，不增加框宽补偿；趋势时间常数、预测50/50/1.0、PI积分保持和3200 counts/s均不再继续调整。

本轮只更新分析文档和待办，不修改代码、配置、脚本或运行资源。按项目第16条执行纯非代码精简闭环：内容检查、`git diff --check`和中文本地提交；无需重新构建或推送，当前 `a76c1e4ba53d/r17` 测试版本继续有效。
