# 积分A/B与分轴稳定修复（2026-07-16）

## 背景

用户现场观察到持续震荡，而此前主要分析X轴，错误地把低幅输出换向视为可接受修正。补查同一批r40数据后发现，水平left/right场景的Y轴真实稳态P95只有约2 px，但`FinalMy`以约4.3 Hz持续`+1/-1 count`换向，能够直接解释视觉上的上下震荡。

为验证是否由500 ms积分造成，用户在`C:\Users\16143\Desktop\Xen\DML\ndi\500`和`...\0`分别采集相同left/right三轮数据。四个CSV均为`DML / c8e8a9266d11 / r40`，并由CSV自身的`IntegralTimeSeconds`确认实际运行0.500/0.000秒积分。

## 配置核对

两组核心控制配置仅`move_integral_time_ms`为500和0；响应120 ms、速度上限3200 counts/s、预测50/50/1.0一致。完整配置另有`ml_termination_frames`、`ml_coast_frames`和overlay位置差异，说明本轮并非字节级单变量配置，但所有试次均持续检测、队列最大1且Y震荡结论在左右两侧一致，因此这些差异不能解释关闭积分后Y换向仍不下降。

关闭积分没有降低Y震荡，反而使X稳态跟踪明显回退。500 ms组Y换向帧的积分分量平均仅0.027/0.037 counts；0 ms组积分严格为0，但换向仍存在。换向时连续控制请求平均只有约0.26~0.42 counts，最终由整数余量累计成`±1 count`。因此积分不是震荡主因，生产配置继续保留500 ms。

## 根因

r40的`BasicAimController`使用单一二维`settled_`和二维`errorDistance`。水平移动时X误差长期位于稳定区外，导致整个控制器不能稳定；即使Y没有可靠运动且误差只有约1~2 px，Y比例项仍逐帧追逐检测噪声。全局`preserveMovingIntegral`和`allowMovingInsideSettle`又把X轴可靠运动扩散到Y轴，进一步阻止Y独立清理状态。

整数余量不是原始噪声源，但会守恒累计这些亚count比例请求，周期性发送`±1 count`，使数据中的小误差变成可见震荡。

## r41设计与实现

控制器修订由r40升级为r41，采用按轴状态而不是继续增加中心反向确认：

1. `preserveMovingIntegralX/Y`分别控制越心积分保持；
2. `allowMovingInsideSettleX/Y`分别控制回差内运动释放；
3. `settledX_/settledY_`分别使用原分量稳定和释放半径，当前轴锁存不再受另一轴误差阻止；
4. 只有对应轴预测提前至少达到稳定半径10%（默认0.6 px），或可靠速度分量至少占总速度35%，才保持该轴运动状态；水平500 ms数据的Y提前P95仅0.061 px且Y速度P95为0，不再由数值残差打开Y轴；
5. 单轴锁存时清除该轴积分、历史输出、反向候选和整数余量，另一轴继续正常控制；
6. 大误差、可靠垂直和斜向运动按对应轴立即释放，不增加用户配置或场景名分支。

CSV新增`ErrorMotionX/Y`、`MovingInsideSettleX/Y`和`SettledX/Y`。移动分析器新增`AxisMovingInsideSettlePct`和`AxisSettledPct`，旧CSV缺少分轴字段时回退到原二维诊断。

## 单元测试

核心测试建立持续12 px水平误差并交替输入`±2 px`垂直噪声，要求X持续输出、Y全程锁存且控制量为0；随后提供可靠8 px垂直运动，要求只释放Y锁存并立即输出。现有积分越心、静止回差、高频释放、速度限幅、CSV列数和版本身份测试同步更新到r41。
