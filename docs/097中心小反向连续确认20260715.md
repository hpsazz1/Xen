# 中心小反向连续确认（2026-07-15）

最新同版本 NDI 数据显示，left 在中心附近的换向多为 `+1/-1` 至 `+3/-2 counts`，而 reverse 和 jump 的真实反向幅度明显更大。此前单帧幅度滞回只能过滤最多 2 counts，无法阻止检测帧间隔抖动造成的短脉冲。

本轮将中心小反向改为两帧连续确认，并把内部过滤上限提高到 4 counts；只对释放半径内的小幅反向生效，大幅 reverse/jump 仍立即放行。上一帧方向只在实际发送非零输出时更新，避免被抑制的零帧打断确认状态。

该逻辑没有新增用户参数，不改变速度上限、响应时间或预测前瞻。DML/CUDA Release 和两套 CTest 完成后，下一轮重点比较 left/right 的 `FinalMx` 换向频率及 reverse 低速误差，确认是否以最多一帧确认延迟换取更少的中心脉冲。
