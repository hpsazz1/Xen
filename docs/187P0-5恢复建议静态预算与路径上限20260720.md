# P0-5恢复建议静态预算与路径上限

## 背景

186号使用权威`horizontal_left1.csv`和`horizontal_right.csv`完成双方向恢复建议覆盖：left有1个建议启用后自然退出的完整代表事件，right有4个，安全违规为0。用户确认同方向重复测试条件相同，因此停止机械追加等价样本。本阶段继续执行186号下一步，判断同帧静态预算能否解释正式1440真实退出，并评估冻结实测路径上的1800建议首次不再裁剪序列。

## 方法与边界

分析器在每个建议启用且自然退出的reset代次中记录：首个启用序列、首帧1440/1800静态预算、正式启用帧数、正式退出序列，以及`RecoverySpeedAdviceLimited`首次变为0的序列。

“静态预算”是`ceil(同帧无限制请求模长/同帧限额)`，不包含后续误差、估计器和相机响应。“建议路径退出”只在已发生的1440实测观测路径上寻找1800首次不再裁剪帧；候选命令一旦与正式命令不同，后续真实画面本应改变，因此该序列不是1800闭环预测、实机测量或因果效果。

冻结实测路径上，5/5事件都观察到1800建议先于正式退出不再裁剪；提前帧overall P50为24、P95为37.4、最大39，left为2，right P50为27.5、P95为37.8。该结果证明1800在既有路径上具有更宽的请求包络，但路径在第一条不同命令后已经失去反事实有效性，不能把“提前39帧”解释为真实设备收益。

## 结论与下一步

被动1440 CSV的证据上限已经到达：它能证明建议旁路安全、方向覆盖和冻结路径包络，但不能识别1800命令改变画面后的闭环退出、过冲或设备能力。继续采集同类1440 left/right文件不会解除该因果缺口。

P0-5继续`HOLD_SHADOW`，正式1440、配置、UI、active和设备输出均不修改。下一步只有在用户明确授权后，才能单独设计1800设备能力与过冲安全协议；协议必须先限定设备、场景、最大位移、紧急停止、1440对照和零目标风险环境，再决定是否允许任何高于1440的物理命令。在获得该授权前停止推进，不得用冻结路径结果替代实机能力证据。
