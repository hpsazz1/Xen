# 预测持续期间积分保持

## 背景与版本

2026-07-14使用 `BuildRevision=6af7aa2f3fb9`、`ControllerRevision=15`、DML+NDI、3200 counts/s和预测50/50/1.0完成left/right各4轮复测。NDI接收平均约237~238 FPS，DML推理约94 FPS，全程未使用手动鼠标辅助。

## r15收益

r15持续运动豁免已解决门控反复误杀。left/right预测生效率由r14的63.9%/83.8%提升至98.0%/98.6%，自运动抑制由23.6%/10.6%降至0.2%/0.1%。说明真实持续横移不会再被static门控周期性打断。

X轴平均P95误差改善到22.61/21.68 px，但right仍高于19.97 px基线，用户继续观察到来回震荡。预测侧翻和自运动门控已经接近零，因此剩余问题不再属于预测生命周期或方向识别。

## 剩余根因

持续横移时预测偏移中位约14.8~15.1 px且方向稳定。控制器的PI积分项平均占请求输出绝对值的55%~73%，用于维持目标匀速；与此同时预测误差每轮在5 px阈值两侧穿越约7~13次，最终鼠标输出约每秒换向2次。

现有反向解卷绕规则只看“误差与旧积分反向且达到5 px”。当预测点因检测噪声或闭环修正在中心两侧摆动时，这个条件会清空维持目标速度的积分。相机随后减速，目标重新拉开误差，积分再次建立，形成周期性的速度下降、追赶和越心。预测提前和PI积分并非简单重复：预测定义希望准星位于目标前方的位置，积分负责维持目标本身的持续速度；问题在于移动状态下错误地清空积分。

## r16设计

1. 仅当预测偏移非零时标记为持续移动积分保持；预测点的小幅越心不再清空该轴旧积分。
2. 停止、窗口退化、门控抑制或真实反向确认时，预测偏移会先归零；控制器同一帧恢复原有5 px反向解卷绕，不会拖拽旧方向。
3. 静止目标预测偏移为零，继续使用原有积分清零、稳定半径和锁存逻辑，static安全路径不变。
4. 积分最大状态、设备速度上限、饱和时禁止提交候选积分等既有抗饱和规则全部保留。
5. 不修改3200 counts/s、50/50/1.0预测参数、稳健速度、透视换算或目标丢失策略。

控制器修订号升级为16。单元测试覆盖预测有效时跨越-6 px仍保留正向移动积分，以及预测撤销后同样误差立即恢复积分解卷绕。

## 下一轮测试

- 保持参数不变，复测left/right各4轮，每轮4~5秒、轮间至少300 ms。
- 对比r15的X轴P95 22.61/21.68 px、预测误差穿越和最终输出换向次数；除指标外必须记录肉眼震荡是否明显下降。
- left/right预测生效率至少95%、自运动抑制低于5%；X轴P95分别不高于22.26/19.97 px。
- 移动通过后补static九段，要求X/Y预测生效率仍不高于0.5%，且无持续过冲。
- 然后测试jump、reverse和 `Target_static_Role_reverse` 各20秒，以及三类目标丢失各10次。
