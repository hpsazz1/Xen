# r40单向三轮复测与稳态分析（2026-07-16）

## 背景

上一轮r40数据只有left/right各两轮，最终设备输出换向分别为16次和6次，方向差异较大，不能据此继续增加中心门控。本轮按计划使用同一程序独立复测left/right各三轮，数据位于`C:\Users\16143\Desktop\Xen\DML\ndi`。

两个CSV身份一致：`BuildBackend=DML`、`BuildRevision=c8e8a9266d11`、`ControllerRevision=40`、`BuildTimestampUtc=20260715T162405Z`。配置保持120 ms响应、3200 counts/s、500 ms积分和预测50/50/1.0。

## 数据链路

left/right分别为1220/1286行，分析器识别出各3个有效试次。DML推理平均86.2/92.1 FPS，NDI接收平均153.6/160.8 FPS，观测年龄P95为20/19 ms，队列最大1，速度限速均为0%。接收FPS低于上一轮约235~239 FPS，但观测年龄、队列和控制输出没有同步异常，因此不能把接收计数变化直接归因控制器。

## 排查过程

全段真实P95受起点影响明显：left三轮起始控制误差为23、42.5、29.5 px，right为3、9、21 px。left平均全段真实P95为17.67 px，高于right的14.83 px，但这与left更远起点同时出现，不能直接判断方向性增益不一致。

为排除起始捕获，按每轮最后1000 ms重新计算原始支点相对准星的真实稳态误差；同时逐次提取`FinalMx`非零方向变化，并记录换向后首个整数命令的幅度。该诊断具有重复使用价值，因此正式加入`tools/analyze_moving_target.ps1`，不保留一次性脚本。

## 优化实现

本轮不修改生产控制器、不增加用户参数，控制器修订保持r40。正式分析器新增：

1. `ObservedSteadyMeanAbsAxisErrorPx`和`ObservedSteadyP95AbsAxisErrorPx`，场景汇总新增对应`MeanObservedSteady*`字段；
2. `OutputSideFlipMeanAbsCounts`和`OutputSideFlipMaxAbsCounts`，场景平均按实际换向次数加权；
3. 默认控制台摘要同步显示真实稳态P95和换向幅度；
4. 旧CSV缺少原始支点时继续以`ObservedTrackingAvailable=0`安全降级。

单元测试覆盖试次稳态误差、场景稳态汇总、换向平均幅度、最大幅度和旧CSV兼容。

## 结论

r40单向回归通过并冻结。left/right稳态P95完全一致，预测侧没有翻向，设备输出反向只是低幅整数修正且跨轮不一致。继续增加中心反向确认、扩大抑制幅度或改变积分会以可观测延迟换取未证明的收益，不符合当前证据。
