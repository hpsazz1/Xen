# 反转抑制窗口低速跟随（2026-07-15）

## 数据结论

干净 r38 数据显示 left/right 的中心小反向过滤有效：换向分别降到 4 次和 8 次；reverse 的 P95 误差约 16.4 px，末段误差约 -15 px。reverse 中约 92% 帧处于 `PredictionOscillationSuppressed=1`，预测前瞻被正确撤销，但仍使用 120 ms 基础响应，造成低速反转期间持续滞后。

## 修复

仅在“反转抑制模式 + 方向锁定速度至少 80 px/s + 未命中自运动抑制”同时成立时，将基础反馈响应限制为最多 80 ms。该分支不改变预测距离、最大设备速度或用户配置；left/right、static 和 jump 不满足该条件，保持原响应。

下一轮重点验证 reverse 末段稳态误差、恢复时间和换向次数，确保加快反馈没有重新引入中心脉冲。
