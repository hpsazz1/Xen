# 跨游戏预测与控制架构重评

## 背景

当前r30已经在CS、106°×74° FOV、2560×1440完整视场、中心320×320 ROI、约100 FPS DML检测和指定鼠标参数下，针对static、left、right、reverse与jump完成了多轮实机优化。现有方案包含时间戳、观测年龄补偿、稳健短窗速度回归、自身视角运动补偿、透视像素与视角换算、预测方向确认、高频反转抑制、检测框内保持、逐轴追赶、PI积分和设备总速度限制，并非简单的“单帧速度乘固定时间”。

但r7到r30持续出现同一模式：修正static后影响持续移动，修正reverse后误伤jump，修正jump后又需要为left/right补充速度和方向门槛。这说明主要矛盾已经不是某个阈值不够精确，而是观测、状态估计、机动识别、延迟补偿和控制保护仍耦合在同一个手工状态机中。大量阈值虽然按检测跨度归一，但其数值仍来自当前游戏的运动速度、FOV、帧率、灵敏度和设备响应分布，直接换游戏存在失配风险。

本轮目标不是立即替换算法，而是重新确认各类成熟方法分别解决哪一层问题，并给出可离线验证、可逐步迁移的小型架构。

## 排查范围与现状

### 当前预测器

`TargetPredictor`使用约50 ms稳健线性回归估算二维像素速度，将观测年龄与额外50 ms前瞻相加并限制到120 ms，再按常速度模型外推。方向建立、静止、短时回摆、高频反转和高速瞬态由多组按检测跨度缩放的门槛管理，最大提前距离为检测跨度的7.5%且至少12 px。

该实现对已覆盖的CS场景有效，但存在四个结构性限制：

1. 单一常速度模型无法同时描述静止、匀速、突然停止、跳跃、加速和高频反转。
2. “速度是多少”和“这次速度是否可信”没有用统一不确定度表达，而由多个门槛分别判断。
3. 预测偏移和控制反馈共同改变后续观测，闭环回摆容易被误判为目标换向。
4. 固定额外前瞻与速度门槛隐含当前游戏、后端和设备的延迟与运动分布。

### 当前控制器

`BasicAimController`本质是带死区、限速和可选积分的一阶比例响应。大误差时逐轴缩短响应时间，积分用于消除持续运动的稳态误差。它不是完整PID，未使用D项；这一点反而符合检测噪声较大的现实，因为直接对误差求导会放大检测框抖动。

当前系统已经接近“位置反馈 + 目标速度提前”的组合，但两部分尚未形成清晰接口，预测器输出的是一个提前位置，控制器不知道该提前量的置信度、状态协方差或模型来源。

## 三维视角到二维控制的关键约束

### 屏幕速度不是世界速度

游戏目标在三维世界中运动，检测器只给出二维屏幕投影。针孔模型中，目标相机坐标为 `(X,Y,Z)` 时：

```text
u = fx * X / Z + cx
v = fy * Y / Z + cy
```

因此水平投影速度为：

```text
du/dt = fx * (Vx * Z - X * Vz) / Z² + 相机运动项
```

同样的世界横移速度在不同深度会产生不同屏幕速度；目标向相机靠近或远离时，`Vz`也会改变屏幕速度。角色自身平移、相机yaw/pitch旋转和目标三维运动最终都叠加在二维观测上。只根据二维框中心，无法唯一分离这些来源。

### 单目二维观测无法可靠恢复距离

一个屏幕点只确定从相机出发的一条视线，沿该射线的多个三维位置会投影到同一个像素。检测框高度又会受遮挡、姿态、只露头部、动画和检测类别影响，不能作为稳定测距依据。因此：

- 当前输入条件下不能宣称得到目标真实距离、世界速度或严格弹道交点。
- 不应再次用框高、固定身高或单一游戏人物尺寸制造伪距离。
- 真正的三维弹道预测必须额外获得可靠深度或游戏遥测，以及武器弹速、重力、阻力等参数；这应是可选扩展，不应阻塞基础追踪。

### 当前可做的是视线角预测

鼠标最终控制的是相机yaw/pitch两个旋转自由度，所以基础模块最合适的状态不是原始像素，也不是不可观测的世界坐标，而是目标相对准星的视线角：

```text
thetaX = atan((u - cx) / fx)
thetaY = atan((v - cy) / fy)
```

工程现有`AimCoordinateSpace`已经使用完整源尺寸和FOV完成`pixel -> atan -> angle`、`angle -> tan -> pixel`透视换算，这是正确基础。下一代估计器应直接在角度空间维护`thetaX/thetaY`及角速度，而不是继续在320 ROI像素速度上分类。这样可以消除分辨率和FOV变化造成的主要尺度差异，并让程序发送的鼠标counts自然转换为已知相机角运动进行补偿。

由于中心ROI只覆盖完整画面中央小区域，yaw/pitch近似解耦已经足够；不需要为当前最小闭环引入完整三维位置、四元数或场景重建。若未来支持大范围非中心ROI、滚转或自由相机，再升级为单位视线向量和三维旋转模型。

## 成熟算法对比

| 方法 | 所属层 | 主要收益 | 主要限制 | 跨游戏能力 | 结论 |
|---|---|---|---|---|---|
| 固定EMA | 观测平滑 | 实现简单、成本低 | 抖动与滞后不可同时优化；固定系数依赖FPS | 中 | 不作为主方案 |
| One Euro Filter | 观测平滑 | 低速强滤波抑制抖动，高速自动提高截止频率降低滞后 | 不估计目标身份、运动状态或未来位置 | 高 | 推荐用于检测pivot预处理或自适应测量噪声参考 |
| α-β滤波 | 状态估计 | 小型、时间戳友好，可统一位置与速度 | 固定增益对噪声和机动变化适应有限 | 较高 | 推荐作为首个离线基线 |
| Kalman滤波 | 状态估计 | 显式表达过程噪声、测量噪声、协方差、创新和短时缺测 | 运动模型和噪声设错仍会失效 | 高 | 推荐作为主候选 |
| 轻量IMM/多模型 | 机动估计 | 并行组合静止、常速度和高机动模型，用概率替代手工速度分类 | 比单Kalman复杂，需要离线标定与状态切换测试 | 很高 | 第二步推荐，先做轻量双/三模型 |
| SOT/光流 | 帧间观测 | 可在DML检测帧之间提供更高频位置更新 | 遮挡、形变和快速运动会漂移，必须由检测器重置 | 较高 | 后续按收益评估，不是当前第一步 |
| SORT | 多目标关联 | Kalman加Hungarian保持多目标ID | 不负责提前量和准星控制，依赖检测质量 | 高 | 当前单目标锁定链路不优先 |
| 纯延迟补偿+直追 | 降级控制 | 稳定、可解释，低置信度时不盲目外推 | 持续运动存在固定滞后 | 高 | 保留为安全降级模式 |
| Smith Predictor | 延迟控制 | 固定纯延迟且过程模型准确时可预测无延迟响应 | 对延迟和模型失配敏感；NDI、推理和调度延迟会波动 | 中 | 只借鉴时间戳外推，不照搬完整结构 |
| P/PI反馈 | 控制 | 对当前屏幕误差闭环纠偏，PI可消除匀速稳态误差 | 不能独立预测未来目标位置 | 高 | 保留并明确职责 |
| PID/PIDF | 控制 | 前馈与反馈分离后可同时跟随运动并纠偏 | 检测噪声会使D项放大抖动，积分需防饱和 | 高 | 采用P或PI + Feedforward，不优先加D |
| 贝塞尔轨迹 | 输出整形 | 可限制鼠标路径外观与曲率 | 增加路径和延迟，不改善观测与状态估计 | 低 | 不用于实时跟踪主链路 |

“目标追踪”不是单一算法名。检测器负责发现目标，SOT负责短时位置延续，SORT负责多目标身份关联，Kalman/IMM负责状态估计，PIDF负责执行控制；把其中任意一个单独替换进当前链路都不能同时解决提前量、震荡和目标丢失。

## 推荐的下一代最小架构

### 1. 观测层

- 输入检测框、pivot、置信度、目标ID、观测时间戳和完整FOV几何。
- 将完整源坐标转换为yaw/pitch视线角。
- 使用One Euro或基于检测残差的自适应测量噪声抑制pivot抖动。
- 检测框尺寸只用于框内安全区、可见性和测量可信度，不用于推断距离。

### 2. 状态估计层

- 首版维护`thetaX, thetaY, omegaX, omegaY`，使用真实`dt`的α-β或Kalman滤波。
- 使用创新残差和协方差判断观测是否符合当前模型，不再以“640 px/s”等游戏速度阈值作为核心模式开关。
- 第二版增加静止/常速度/高机动轻量多模型；模型概率负责缩短或撤销提前量。
- 目标ID切换、死亡、角色死亡或可靠观测丢失仍立即停止输出；只允许经过上限约束的短时coast。

### 3. 延迟与提前层

- 用观测时间戳将估计状态外推到实际控制时刻，补偿捕获、NDI、推理和调度的真实观测年龄。
- 基础延迟补偿来自测得时间戳，不再为所有后端固定增加同一延迟。
- 用户可控“额外提前”保留为角时间或强度参数，表达玩法偏好，而不伪装成弹道学距离预测。
- 没有深度和武器参数时，产品描述应为“视线角运动提前”，不能承诺真实三维弹道提前。

### 4. 控制层

使用明确的前馈与反馈组合：

```text
控制输出 = 当前角误差的P/PI反馈 + 估计目标角速度的前馈
```

- P/PI负责把准星拉回目标和修正模型误差。
- 角速度前馈负责维持匀速目标，不再通过不断把预测点推到框外间接形成速度输出。
- 保留设备总速度限制、死区、抗积分饱和、反转解卷绕和目标丢失停止。
- 暂不加入D项；只有在角状态已充分滤波且离线数据证明有净收益时再评估。

### 5. 置信度降级

| 状态 | 行为 |
|---|---|
| 高置信度、模型稳定 | 时间戳外推 + 角速度前馈 + 位置反馈 |
| 创新突然增大或机动模型占优 | 自动降低前瞻和前馈，只保留快速位置反馈 |
| 高频反转 | 由模型概率和创新自然降低常速度可信度，保持框内走廊 |
| 短时漏检 | 在严格时间和协方差上限内coast |
| 目标飞出、死亡、角色死亡或身份改变 | 立即清空状态并停止控制 |

## 为什么比继续调r30更通用

1. 用角度状态替代像素速度，消除分辨率、ROI尺寸和FOV的主要比例差异。
2. 用时间戳和协方差替代固定FPS假设，DML、CUDA、NDI和UDP可共享同一逻辑。
3. 用创新和模型概率表达“预测是否可信”，减少按某个游戏速度分布手工分类。
4. 用速度前馈和位置反馈分离职责，避免预测点越心后控制器反复追逐框外位置。
5. 明确二维单目能力边界，不以不可靠框高或武器猜测制造三维精度假象。

这仍不能让同一组参数在所有游戏中完全免配置。不同游戏的鼠标yaw/pitch、灵敏度曲线、FOV定义、输入加速度、目标动画和检测质量仍需配置或标定；但这些差异会被限制在坐标与设备适配层，不再渗透为预测器内部的速度门槛。

## 迁移与验证方案

### 阶段A：离线影子评估，不改运行链路

1. 冻结r30作为现有基线，不继续追加场景阈值。
2. 在同一CSV/视频回放中并行计算r30、角度α-β、角度Kalman三套输出，不发送新控制指令。
3. 对static、left、right、reverse、jump和角色自运动分别比较角误差P50/P95/P99、提前连续性、创新异常、方向翻转、目标框内比例和丢失前边缘距离。
4. 使用至少两档FOV、两档回放帧率和不同速度缩放做合成变换，验证候选参数是否保持稳定，而不是只在CS原速度通过。

### 阶段B：拆分控制

1. 保持状态估计输出不变，离线比较“预测点直追”和“位置P/PI + 角速度前馈”。
2. 单独搜索前馈增益与低置信度衰减，不再同时调整滤波、状态机和控制响应。
3. 候选必须同时通过静止无假前馈、持续移动低稳态误差、reverse框内保持和jump不逃逸。

### 阶段C：轻量多模型

当单Kalman仍在停止、反转和jump之间明显冲突时，再加入静止/常速度/高机动模型概率。只有离线数据证明相对单模型有稳定净收益，才进入运行代码。

### 阶段D：评估SOT

DML检测约92~106 FPS，而NDI可达约238 FPS。只有状态估计与控制拆分后仍能证明检测帧间空档是主要瓶颈，才评估光流或轻量SOT。SOT输出必须携带独立置信度，由新检测周期性重置，并禁止在目标真实丢失后长期滑行。

## 本轮结论

当前r30对已测CS场景有效，但继续依靠速度、加速度、方向驻留和反转频率阈值扩展到其他游戏，边际收益低且回归风险高。推荐停止局部门槛补丁，下一阶段先建立角度空间的时间戳α-β/Kalman离线影子评估，再拆分为P/PI位置反馈和角速度前馈；单模型不足时加入轻量多模型，最后才评估SOT。
