# r60机动影子首轮实机阻断

## 背景与测试口径

r60已经把离线冻结的`maneuver_gated_ca + 8000°/s³ + 12°/s + 120 ms`接入DML shadow。本轮使用`C:\Users\16143\Desktop\Xen\DML\ndi`中的最新配置和五场景CSV，检查真实NDI时间线上的安全契约、模型选择和static误驻留。程序在检测到目标后即连续记录流水线，而不是只在按住追踪键时记录；因此`AimPipelineOutputPaused=1`是有意保留的松开观测，不能当作无效帧删除。

`static.csv`采用九点测试法：目标保持静止，依次从左侧上中下、中间上中下、右侧上中下附近开始回中。各场景的多轮数据合并在单个CSV内，轮次应按暂停恢复边界和场景操作解释，不能用CSV文件数代替。

## 配置与身份

- 构建身份统一为`DML/abeef0f423a9/r60`，无`-dirty`；
- 请求和生效模式均为`shadow`，估计器为`maneuver_gated_ca`；
- jerk、速率阈值和保持时间分别为`8000°/s³`、`12°/s`和`120 ms`；
- `aim_shadow_command_to_frame_delay_ms=60`，被动Profile标定关闭；
- 五个文件合计8248帧，全部具有有效估计，其中暂停观测3548帧。

## static失败归因

static的40次切换均满足当前实现的速率门控和计数连续性，不是CSV损坏或选择计数错误。20次启用中，11次发生在松开追踪后的人工九点换位阶段；单目标角度无法区分手动移动视角和目标运动，因此这部分不应与运行态设备风险混为一谈。

更关键的是，按下追踪后的9轮全部发生一次运行态误启用。每轮恢复首帧的基线LOS速率仅为0~0.84°/s、NIS接近0，说明估计器跨松开连续性有效，旧速度没有直接泄漏；但恢复后第3~5个观测、约33.7~57.4 ms时，画面中的静止目标已经开始回中，而配置为60 ms的`AppliedViewMotionModel`尚未应用对应相机变化。稳定LOS因此产生2.08~4.36°瞬态漂移并越过12°/s门槛。

9段运行态误启用各持续约217.6~256.9 ms，共211帧；其中151帧已经处于legacy的`Settled=1`。模型角度差最大1.535°，模型速率差最大78.82°/s，不能视为无害诊断噪声，也不能通过忽略热身或删除static硬门槛放行。

根因是本机DML/NDI/设备链路的真实命令到画面响应早于固定60 ms模型，造成自运动补偿相位滞后。九点人工换位说明验收还必须分开统计暂停态、恢复边界和运行态；但即使排除暂停态，当前运行态仍明确失败。

## 结论与下一步

1. r60安全契约通过，但机动候选保持`HOLD_SHADOW`；不得开放active或CUDA shadow。
2. 不调整8000/12/120参数，不通过放宽static门槛掩盖自运动补偿失配。
3. 下一轮先启用已有`profile_calibration_enabled`，在目标、角色和配置不变且单轮锁定期间不手动移动鼠标的条件下，完成至少3轮静止目标被动标定，记录X/Y延迟、比例、相关性、RMSE、有效样本和可信度。
4. 依据重复标定结果设计有界的command-to-frame延迟A/B；每个候选必须同时复跑九点static和jump/reverse，检查运行态static零误启用、恢复首帧有效以及机动场景仍能启用。
5. 后续分析脚本应分别输出暂停态、恢复边界和运行态机动样本。只有暂停态人工换位可以单独解释；恢复后和运行态误启用仍是硬失败。该脚本调整应与延迟证据一起实现，不能先修改验收口径制造通过结果。
