# 稳健常速度预测与实机振荡排查

> 状态：已被 `docs/035预测连续性保持与单帧门控修复20260714.md` 取代。r8解决了静止假预测和非主轴抖动，但单帧反向门控仍会频繁切断提前量。

## 背景

2026-07-14使用 `BuildRevision=7ee1fe18d2ab`、`ControllerRevision=7`、DML+NDI、50 ms额外前瞻、15 ms速度平滑、1.0预测强度和2000 counts/s完成首轮实机复测。除 `Target_static_Role_reverse` 外，角色自身均静止：`horizontal_left/right/jump/reverse` 只有目标运动，`static` 中目标和角色均静止；`Target_static_Role_reverse` 是目标静止、角色左右移动。该边界意味着普通场景中的预测振荡不能归因于角色移动，角色移动场景中的相对屏幕运动则是真实控制需求，不能简单从速度中全部扣除。

## 数据与现象

X轴场景汇总如下：jump P95误差122.78 px、预测提前P95 106.17 px；left/right P95误差分别60.49/61.30 px、提前P95约56 px；reverse P95误差28.24 px；九次static平均P95误差9.04 px，但预测仍有26.7%时间生效、提前P95 4.99 px；`Target_static_Role_reverse` P95误差58.96 px、预测生效34.9%、提前P95 55.22 px。

Y轴本不应跟随水平动作，jump、reverse和static的P95误差却分别达到37.53、25.89和15.24 px，提前P95分别为32.18、26.92和12.31 px。直接CSV统计显示static仍被估计出约381/313 px/s的X/Y速度和约5700/5500 px/s²的X/Y加速度。自身视角补偿在static中与原始屏幕速度反向比例约99.5%，方向没有接反，但补偿残差与检测框噪声被后续预测继续放大。

## 根因

r7使用相邻两帧差分、15 ms快速低通和 `速度×时间+二分之一加速度×时间平方`。DML约106 FPS时15 ms只覆盖约两帧，检测框中心、闭环控制和时间间隔的微小波动会被差分放大为数百px/s，再被二阶差分放大为数千px/s²。50 ms前瞻继续把这些假速度和假加速度转成可见准星位移。固定视角视频轨迹比实机检测结果干净，因此先前视频模拟明显高估了显式加速度模型的收益。

## r8设计

预测控制改为稳健常速度模型：

1. 使用40~120 ms短窗内全部真实时间戳和补偿后坐标做最小二乘直线回归，默认窗口50 ms；旧配置键 `prediction_velocity_tau_ms` 为兼容保留，但UI名称改为“速度窗口”。
2. 只有窗口至少四帧、持续至少25 ms，并同时满足净位移、速度和轨迹直线性门槛时，才把运动视为可靠；回归窗口成立后还需连续确认，合计至少五帧运动证据。
3. 显式加速度不再参与预测位移；`PredictionAccelerationX/Y` 只保留为CSV噪声诊断。提前量只计算 `稳健速度×前瞻时间×预测强度`。
4. 横向速度明显占优时将小Y速度吸附为0，纵向同理，避免水平目标移动产生垂直准星摆动。
5. 停止或单帧明确反向立即撤销预测；反向连续三次后切换方向。目标丢失、ID切换和瞄准状态变化仍立即清空全部历史。

该模型不估算目标真实距离、身体高度或武器弹速。`Target_static_Role_reverse` 中角色平移造成的目标相对屏幕运动仍应被追踪；程序只补偿自身已经发送的鼠标视角移动，不假设能够读取或扣除键盘位移。

## 测试与验收

单元测试新增检测框交替抖动不得锁定方向、持续水平移动才能启用预测、水平运动的Y速度和Y提前必须为0、停止首帧收回、反向不得保留旧侧提前，以及加速度诊断不得改变控制位移。控制器修订号升级为8。

下一轮固定 `prediction_lead_ms=50`、`prediction_velocity_tau_ms=50`、`prediction_strength=1.0`、`move_max_speed_cps=2000`，不再人工扫描初始参数：

- `static`：目标和角色均静止，连续采集9段，每段至少3秒；要求 `PredictionActivePct<=5%`，X/Y预测提前P95均不超过2 px。
- `horizontal_left/right`：角色静止，各4次连续单向移动；要求Y预测提前P95不超过1 px，单向预测侧翻转不超过1次。
- `horizontal_reverse/jump`：角色静止，各连续20秒；核对预测在真实反向首帧收回，不得出现比实际反转更多的预测侧翻转。
- `Target_static_Role_reverse`：目标静止、角色持续左右移动20秒；该文件单独评估相对屏幕运动，不与“目标和角色均静止”的static合并。
- 目标飞出、目标死亡、角色死亡各10次；目标丢失后不得新增预测或控制输出。

若static假预测和水平场景Y提前达到验收线，再比较r8与r7的X轴P95误差决定是否继续调整强度；若安全项仍失败，优先关闭预测而不是继续增加局部补丁。
