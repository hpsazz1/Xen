# r60 DML机动估计影子接入

## 背景与目标

离线跨域回放已经冻结`maneuver_gated_ca + 8000°/s³ + 12°/s + 120 ms`：总通过由688提高到721/810，原688项零损失，救回31/38个估计受限项，static机动模型驻留为0。本阶段把完全相同的估计器接入DML shadow，用真实NDI时间线检查模型选择、松开连续性和静止误驻留；不修改legacy控制器，不接管设备队列，不开放active，也不把CUDA纳入候选实测。

## 顶层设计

新增`ManeuverLosEstimator`作为离线回放与实机shadow共用实现，内部同帧更新两条状态链：

1. 原自适应Q常速度Kalman作为冻结比较器；
2. 三状态`[角度, 角速度, 角加速度]`Kalman，过程噪声为离散白jerk的`GqG'`。

模型选择只读取常速度比较器的二维LOS速率。达到12°/s后选择常加速度，低于阈值后最多保持120 ms。两条后验都停留在观测时刻，控制时刻只投影副本。目标切换、目标丢失和配置重载同时重置两条模型与选择计数；短暂松开不是目标丢失，继续消费观测并保持同一代次。

默认`aim_shadow_estimator_mode=kalman`保证旧配置与旧shadow行为不变。只有DML构建显式读取`maneuver_gated_ca`；CUDA构建即使遇到该值也强制保持常速度Kalman。无论选择哪个模型，`activeAvailable=0`、候选控制与轨迹命令仍双重抑制，正式设备输出继续由legacy r60链路产生。

## 配置与诊断

新增配置：

```ini
aim_shadow_estimator_mode = maneuver_gated_ca
aim_shadow_ca_jerk_std_dps3 = 8000.00
aim_shadow_maneuver_rate_threshold_dps = 12.00
aim_shadow_maneuver_hold_ms = 120.00
```

流水线CSV保留原`AimPipelineAngle/Rate/Covariance/Nis`作为当前选中状态，并新增：

- `AimPipelineBaseline*`：原常速度比较器；
- `AimPipelineCa*`：常加速度模型；
- `AimPipelineEstimatorMode`、`AimPipelineManeuverModelActive`；
- `AimPipelineEstimatorSelectionChanged/Count`；
- 冻结参数、剩余保持时间、两模型角度/速率差。

`tools/analyze_shadow_pipeline.ps1 -RequireManeuverCandidate`新增硬检查：配置必须精确为8000/12/120，比较器和候选协方差必须有限且为正，同一目标代次的选择计数必须连续，static启用样本必须为0，jump和reverse必须各至少启用一次。原shadow命令抑制、暂停零设备输出、时间顺序和身份门槛继续同时生效。

## 实机门槛与测试顺序

使用DML、NDI、320检测分辨率和shadow模式，static、left、right、reverse、jump各采3轮。每个文件建议保留至少6000帧，并在导出后立即按场景命名。验收顺序：

1. 身份必须是本阶段最终DML提交、控制器r60、无`-dirty`；
2. 全部观测保持`shadow/shadow`、`activeAvailable=0`，暂停和正常帧均无候选设备命令；
3. static机动模型启用为0；jump和reverse必须出现合理启用；
4. 模型选择计数连续，切换时模型角度差、选中角度和请求counts无非有限值或异常尖峰；
5. 松开与重新按住时观测序列连续，恢复首帧估计有效，不能因保持窗口产生旧方向尾迹；
6. left/right不能出现相对r59的新回退。

任一门槛失败均保持`HOLD_SHADOW`并回到CSV定位；不得降低12°/s阈值、扩大120 ms保持或让常加速度常驻static来追求局部跟随。
