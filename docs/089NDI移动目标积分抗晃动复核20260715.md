# NDI移动目标积分抗晃动复核

## 现象

最新五类数据使用提交 `6c9887aa6b7d`、DML、控制器 r31 和 `3200/320` 配置。速度限制造成的 jump 丢失已明显改善：jump 水平 P95 约 32 px、限速比例约 1.3%。但输出方向翻转频率仍为 jump 3.5 Hz、left 3.5 Hz、right 2.2 Hz、reverse 2.9 Hz；reverse 的预测振荡抑制比例为 76.7%。

## 判断

本轮主要矛盾已从设备速度不足转为积分累积斜率过快。320 ms 积分在检测噪声、目标框轻微越心和预测撤销交替出现时，会反复积累并在越心后解卷绕，表现为持续左右晃动。不能同时改动响应时间、预测窗口和速度上限，否则无法区分因果。

## 优化

将移动目标默认积分窗口从 320 ms 调整为 500 ms，降低积分累积斜率；保留 `move_response_ms=80`、`move_max_speed_cps=3200`、预测 50/50/1.0 和现有反向清除规则。控制器修订号升至 r32，现场 `ndi/config.ini`、代码默认值、文档和测试同步更新。

## 验证

下一轮必须继续使用五类标准场景，重点比较输出方向翻转 Hz、MovingInsideSettle、reverse 振荡抑制比例和 left/right 稳态误差。若翻转频率下降但稳态偏差回升，再单独评估积分输出上限；不得在同一轮重新提高 P 增益或预测前瞻。
