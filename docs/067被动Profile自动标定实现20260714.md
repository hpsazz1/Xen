# 被动Profile自动标定实现

## 背景

现有`config.ini [Games]`保存游戏灵敏度、yaw、pitch和FOV缩放，只能推导理论`degrees/count`。真实设备、OBS/NDI链路、源画面尺寸和输入后端还会共同决定“发送多少counts后，画面何时移动多少像素”。此前CSV中的`FinalMx/FinalMy`只是控制请求入队值，不是设备实际发送时刻，不能直接用于可靠系统辨识。

本轮实现首版被动Profile标定。它不主动发送测试动作，不改变r30控制器，不覆盖用户Profile，也不保存机器缓存；仅在用户显式开启后收集影子数据，在UI和流水线CSV展示估算结果。

## 调用链与时间基准

标定命令只在`IMouseInput::move()`返回成功后记录，并与运动补偿共用同一个`steady_clock`发送时间。这样排除了移动队列等待、被丢弃请求和设备发送失败。

观测在`MouseThread::moveMousePivot()`入口使用检测器发布的观测时间和未经滤波、预测或运动补偿的raw pivot。完整源宽高优先采用捕获后端发布的`screenWidth/screenHeight`，无效时才回退检测裁剪尺寸；FOV继续使用运行配置的HFOV/VFOV。

## 拟合模型

每个轴独立拟合：

```text
相邻raw pivot位移
= counts响应系数 × 延迟对齐后的区间counts
+ 恒定目标漂移速度 × 相邻观测时间
```

相机朝正方向旋转时，静止目标在画面中朝反方向移动，因此可信的counts响应系数必须为负，最终展示其绝对值`pixelsPerCount`。模块在0~250 ms内以2 ms步长搜索command-to-frame延迟，并用三轮Huber稳健加权降低检测框量化和少量异常帧影响。

漂移项只能容忍短时近似匀速的小幅目标漂移，不代表模块能区分角色平移、目标运动、手动鼠标和相机旋转。正式标定仍要求目标与角色静止，且标定期间不要手动移动鼠标。

## 有效性与可信度

每轴至少需要24个相邻观测和8个含非零设备命令的有效窗口。可信度联合使用：

- counts与去漂移画面位移的反向相关性；
- 拟合RMSE相对于有效信号幅度的比例；
- 含命令样本数量。

只有响应范围、相关性和可信度同时满足要求时，该轴才标记为有效。X与Y独立有效，垂直控制激励不足时允许只得到X结果，UI不会把无效Y轴冒充为已标定。

## 输出内容

UI“瞄准 → 游戏配置”新增“被动Profile标定”区域，显示：

- 开关、观测数和实际成功命令数；
- X/Y `px/count`与`degrees/count`；
- X/Y command-to-frame延迟；
- 恒定漂移、RMSE、相关性、有效样本和可信度；
- 综合可信度和手动重置按钮。

流水线CSV新增同名`ProfileCalibration*`字段，覆盖开关、轴有效性、比例、角度、延迟、漂移、拟合质量、样本数和综合可信度。CSV结果属于当前运行链路，不应直接复制为其他游戏或其他机器的Profile。

## 推荐实机测试

1. 使用当前已验收的静止场景，保持目标、角色、FOV、灵敏度、OBS裁剪、NDI源和输入设备不变。
2. 让目标初始位置离准星中心约检测裁剪宽度的四分之一，再开启被动标定并正常锁定；标定期间不要手动移动鼠标。
3. 保持5~10秒，观察X轴有效样本、相关性和可信度；需要Y轴时使用明显垂直偏差的静止目标单独复测。
4. 点击重置后重复3轮。同一链路的`px/count`应接近，延迟允许随OBS/NDI排队产生一定波动。
5. 导出CSV，分别统计三轮X/Y比例和延迟。若比例稳定但延迟波动，下一阶段应保存P50/P95而不是单值；若比例也明显变化，先排查源尺寸、FOV、输入设备或手动移动污染。

## 验证结果

合成测试使用4 ms观测、已知60 ms延迟、X/Y响应0.52/0.51 px/count、恒定漂移和确定性小噪声。测试验证双轴有效、比例误差不超过0.03 px/count、延迟误差不超过6 ms、反向相关性小于-0.8，并验证禁用和重置行为。

配置默认关闭且可保存；UI可见性和CSV表头均有自动测试。完整DML/CUDA构建与干净版本身份核对记录在本次提交验证结果中。

## 后续阶段

首版不自动应用结果，避免低质量标定改变控制行为。下一阶段按优先级推进：

1. 用三轮static实机CSV验证不同设备和NDI链路的重复性。
2. 将单次延迟扩展为滚动P50/P95，并补齐帧接收、推理发布、控制和入队时间轴。
3. 设计不提交仓库的机器标定缓存及严格失效键。
4. 只有缓存重复性和失配检测通过后，才允许高可信结果进入角度状态估计；低可信时继续使用现有用户Profile和r30控制。
