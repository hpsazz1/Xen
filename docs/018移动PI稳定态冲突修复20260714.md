# 移动PI稳定态冲突修复

## 背景

2026-07-14 收到 `C:\Users\16143\Desktop\XenMovingPI\CUDA\ndi` 与 `udp` 下六份 PI 复测 CSV。两条链路均使用 320×320 检测区域、2560×1440 源画面、80 ms 比例响应、320 ms 积分时间和 1440 counts/s 上限。本轮按用户约束先比较 NDI/UDP 是否存在采集专属异常；若不存在，则后续实机验收以推理更慢、更容易暴露控制间断的 DML+NDI 为准。

## 排查过程与根因

对六份文件逐帧统计 `Settled`、`IntegralCountsX` 和中心误差后发现：

| 文件 | 稳定停发帧 | 近中心停发帧 | 最大单帧积分counts |
|---|---:|---:|---:|
| NDI 向右 | 23 | 16 | 14.95 |
| NDI 反转 | 125 | 107 | 11.80 |
| UDP 向左 | 15 | 10 | 20.63 |
| UDP 向右 | 15 | 12 | 19.64 |
| UDP 反转 | 88 | 70 | 14.88 |

旧控制顺序在误差进入 5 px 稳定半径时无条件执行三件事：标记 `Settled`、清空积分、停止发送。该规则适合纯比例静止目标，却与 PI 跟踪匀速目标的物理需求冲突：移动目标到达中心时，积分项正代表维持目标速度所需的持续 counts/s；将其清零会使准星停住，误差重新扩大后再从零积累，形成“追上—停发—落后—重建”的周期。

DML 推理较慢时相邻控制帧间隔更长，单次停发造成的目标位移更大，因此以 DML+NDI 复测能更严格地验证该问题。CUDA 数据已足以确认根因位于共享控制器，不需要先修改 NDI 或 UDP 采集实现。

## 设计与实现

保持原有 PI、限速、抗饱和和配置值不变，只调整共享控制器的状态顺序：

1. 首版启用积分时按单轴误差符号换向清除旧积分；DML复测发现亚像素越心也会误触发，现已改为反向误差达到稳定半径才解卷绕，详见 `docs/019DML移动复测与积分解卷绕20260714.md`。
2. 将保留积分换算为当前帧 counts。绝对合量达到 0.5 count 时，说明经过整数发送后仍有可执行的持续运动需求。
3. 有可执行积分时，即使误差进入 5 px 中心区也继续控制，不进入静止停发状态。
4. 无可执行积分，或误差换向清除了旧积分后，继续沿用原 5 px 进入、8 px 释放的静止回差。
5. `move_integral_time_ms=0` 时路径与原纯比例控制完全一致。

0.5 count 不是新的移动死区，而是现有 `round()` 整数设备接口的真实可执行边界。该判定随实际观测 `dt` 换算，因此 DML、CUDA 和不同捕获帧率使用同一物理口径。

## 分析器修正

复测中的 `horizontal_right.csv` 被旧分析器误报一次反转。原因是单向场景结束、短暂过冲或目标切换也可能让误差持续进入另一侧。现在只有文件名明确包含 `reverse` 或 `reversal` 的往返场景才计算反转与恢复；`horizontal_left/right` 固定输出零反转，避免把测试脚本语义与闭环瞬态混淆。

## 单元测试

- 60 FPS 模拟 DML 推理节奏，800 counts/s 匀速目标运行 6 秒；预热后不得进入移动稳定停发，最终误差必须小于 5 px。
- 积分仍有至少 0.5 count 输出时，同侧进入 5 px 中心区必须保留积分。
- 误差真正越过中心时必须先清积分，并允许静止稳定逻辑接管。
- 原 120 FPS 匀速、静止 140 px 阶跃、限速抗饱和和纯比例基线继续保留。
- 分析器同时覆盖 `horizontal_reverse` 正常统计和 `horizontal_right` 不误报反转。

## 验证与后续复测

代码级验证必须执行基础算法、配置、移动分析脚本测试以及 DML/CUDA Release 构建。实机验证按以下最小闭环执行：

1. 使用 DML Release 与 NDI，保持 `move_response_ms=80`、`move_integral_time_ms=320`、`move_max_speed_cps=1440`。
2. 先复测水平向左、向右、反转三份 CSV；单向中心附近不应再连续出现 `Settled=1` 且积分归零的周期。
3. 再执行上中、左下、右下三个静止点冒烟；稳定态退出必须为 0，误差不得越过 8 px 释放半径。
4. 若 DML+NDI 同时通过移动与静止验收，则共享控制逻辑可视为覆盖 CUDA；仍不自动把源码积分默认值从 0 改为 320 ms，待现场验收后再决定推广。

## 后续建议

本轮不修改滤波时间常数、不提高 1440 counts/s 上限、不接入速度预测，也不为 NDI/UDP设置不同参数。若修复后的 DML+NDI 单向稳态仍有持续同号误差，再基于新的 `IntegralCountsX/Y` 与 `Settled` 数据单独评估积分时间；禁止同时调整响应、积分和速度上限。
