# 高频反转自适应关闭与jump提前量收紧

## 背景

r21通过七帧渐进释放和64 px上限改善了jump，但用户实测仍认为jump预测过多；reverse依然无法稳定锁在目标框内。最新数据来自构建`1a21ea629761`、控制器r21，预测参数`50/50/1.0`、最大设备速度3200。

## r21复测结果

### reverse

- X轴P95为17.90 px，几乎没有改善r20的17.81 px。
- 去除预测偏移后的目标框中心X轴P95仅10.96 px，明显优于预测点。
- 30次反转全部恢复，但恢复P95从66 ms升至111 ms。
- 预测生效率26.9%，方向换边45次、中断82次，中位连续预测仅3帧。
- 只有14.5%的预测生效帧使X误差小于不预测时。

结论：这类目标移动慢、往返频繁，提前量收益低于换边扰动。保持准星跟随检测框中心更合理。

### horizontal_jump

- X轴P95从r20的118.61 px降至73.47 px，限速从10.9%降至4.9%，r21方向正确。
- 目标框中心X轴P95为54.52 px，当前预测点仍放大到73.47 px。
- 64 px预测上限显著大于目标框宽度中位数42 px。
- 只有36.1%的预测生效帧改善X误差，说明仍存在过度提前。

对现有CSV替换预测上限的反事实结果显示：24 px时jump X轴P95约49.52 px，优于32 px的53.22 px、完全关闭预测的54.52 px和当前64 px的73.02 px。因此jump不应完全关闭预测，但应把异常高速提前控制在约一个目标框半宽附近。

## 设计

### 高频往返识别

- 每次稳健回归确认方向翻转时记录观测时间。
- 最近1.5秒内出现三次可靠翻转，进入高频往返抑制状态。
- 抑制状态只把预测偏移归零；目标检测、基础滤波、PI控制和目标丢失规则继续工作。
- 高频反转持续时，每次新翻转都会延长1.5秒保持；停止往返1.5秒后自动恢复普通预测。
- r21数据中reverse有38组三次换向不超过1.5秒，jump最短三次换向跨度为1.847秒，当前阈值可区分两类场景。

### jump提前量

- 保留r21七帧渐进释放。
- 最大预测距离从检测跨度20%收紧为7.5%，320检测区域对应24 px。
- 普通left/right预测提前P95约20 px，基本不触发24 px上限；高速jump不再把准星推到64 px远处。

### 可观测性

新增`PredictionOscillationSuppressed`流水线字段和分析汇总比例，现场数据可以区分高频往返门控、自运动门控和普通停止/反向撤销。

## 验证方案

- 单元测试覆盖1.5秒内三次可靠换向后偏移归零、高速成熟预测封顶24 px、七帧渐进释放继续有效。
- DML/CUDA Release构建及完整CTest必须通过，UI、CSV、EXE统一显示r22与同一干净提交。
- reverse复测要求`PredictionOscillationSuppressed`持续命中，X轴P95向无预测基线10.96 px靠拢，恢复P95低于111 ms，肉眼不再在目标两侧摆动。
- jump复测要求预测提前P95不高于24 px，X轴P95低于73.47 px，限速比例不高于4.9%，不得新增目标丢失。
- 专项通过后回归static与left/right，确认static预测生效率不高于0.5%，left/right预测生效率不低于94%，X轴P95不明显回退于7.62/8.49 px。
