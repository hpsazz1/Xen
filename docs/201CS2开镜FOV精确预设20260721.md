# CS2开镜FOV精确预设

## 背景

原开镜FOV缩放已经进入运行控制链，但全局`fovX/fovY`仍使用整数，默认`106/74`无法精确保留CS2在16:9下的实测投影参数。Profile也只有自定义数值，用户需要自行抄写单倍镜FOV，容易把垂直FOV误填到水平FOV字段。

本次依据`cs2_fov_reference.md`中的`fov_cs_debug 90`参考表优化：腰射HFOV/VFOV为`106.2602/73.7398`，Zoomed 1为`51.7740/30.5369`。按当前需求，Zoomed 2暂不实现。

## 设计与实现

全局水平、垂直FOV从整数改为双精度浮点，配置加载使用浮点解析，保存固定保留四位小数。鼠标线程构造和热更新接口同步使用双精度，避免配置进入运行链前被截断。

Profile页面新增`自定义`和`CS2 右键单倍镜`预设。选择CS2预设会一次性写入：

- 全局腰射HFOV `106.2602`和VFOV `73.7398`；
- Profile腰射基准HFOV `106.2602`；
- 单倍镜HFOV `51.7740`；
- 开镜FOV缩放启用状态。

运行时继续使用针孔投影比例`tan(currentHFOV/2) / tan(baseHFOV/2)`，垂直FOV由腰射水平/垂直投影比例推导。单元测试用参考表的VFOV验证单倍预设，允许四位小数测量值的末位舍入误差。

## 右键单倍切换

CS Profile启用FOV缩放后，`button_zoom`默认绑定的右键改为上升沿锁存：第一次点击进入单倍镜，松开右键仍保持单倍FOV；第二次点击退出并恢复腰射FOV。按住右键只产生一次切换，防止监听线程重复翻转。其他Profile仍保持按住即开镜的原语义，Profile切换时强制清除锁存状态。

旧三字段CS Profile自动迁移为单倍预设；只有旧默认整数`106/74`会同步迁移为精确值，显式配置的自定义FOV不覆盖。该实现按输入事件推断游戏切镜状态，切枪、死亡或游戏拒绝切镜可能造成状态不同步；双倍镜和游戏状态自动校准留待后续可靠遥测方案，本次不处理。

## 验证要求

配置测试覆盖四位小数默认值、读写往返和旧CS迁移；算法测试覆盖单倍参考VFOV、投影灵敏度比例、右键边沿锁存及非CS按住语义；UI测试覆盖预设入口。正式交付按项目铁律完成DML/CUDA Release clean-first构建、相关CTest、UI测试、版本身份核对、中文提交和远端推送。控制器修订号升级为r68。
