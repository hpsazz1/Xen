# P0-1完整时间轴贯通

## 背景与目标

P0-0 已建立并行影子数据结构，但捕获时间仍只作为一个旧的 `time_point` 传入检测器，NDI/UDP 队列中的实际到达时刻没有随图像移动，控制线程也只能看到帧时间而看不到推理发布和设备发送年龄。本阶段贯通同一 `steady_clock` 域中的捕获、推理、控制和命令生命周期，保证 CSV 可以复算每段延迟。

## 排查结果

- 捕获主循环在 `GetNextFrameCpu()` 返回后重新写 `now()`，网络队列等待被丢失。
- `DetectionBuffer::swapLocked()` 只返回 `frameTimestamp`，没有把发布时间和源元数据带到鼠标线程。
- DML/TRT 的异步提交只保存单独时间点，推理线程没有统一时间契约。
- 鼠标移动队列只有 `dx/dy`，没有序号、入队时间、设备尝试结果和成功发送时间；队列淘汰无法在诊断中区分。

## 设计与实现

### 捕获契约

`CapturedFrame` 将图像和 `FrameTiming` 绑定。`FrameTiming` 现在包含：

- `backendReceiveTime`、`frameSequence`、源宽高、ROI；
- `captureSubmitTime`、`inferenceStartTime`、`inferencePublishTime`、`controlTime`；
- NDI/DDA/WinRT 可提供的原始 `sourceTimestamp/sourceTimecode` 及可用标志；
- `sourceTimestampMapped` 明确保持 `false`，不把协议时钟伪装成本机 `steady_clock`。

NDI 和 UDP 的 `NetworkFrame` 将时间与图像一起入队，最新帧替换时两者同步移动。DDA、WinRT、虚拟摄像头通过后端成功取得帧时调用 `RecordSourceFrame()`，GPU DDA 路径在捕获成功后立即读取该快照。DDA `LastPresentTime` 和 WinRT `SystemRelativeTime` 只保存原始值。

### 推理与检测缓冲

DML/TRT CPU/GPU 提交接口统一接收 `FrameTiming`。推理线程取出帧时填写 `inferenceStartTime`，`DetectionBuffer::set()` 在发布检测结果时填写 `inferencePublishTime`。`swapLocked()` 现在返回整个时间结构，鼠标线程和游戏覆盖层都使用其中的 `backendReceiveTime` 作为观测基准。

### 命令生命周期

移动队列项扩展为 `ViewCommandSample`：

```text
sequence → enqueueTime → sendAttempted → deviceSendTime/appliedCounts
                             ├─ sendSucceeded
                             └─ droppedBeforeSend
```

队列满时淘汰最旧项，目标丢失或配置重置清空队列时，都标记 `droppedBeforeSend`。设备线程只有在底层 `move()` 成功后才填写 `deviceSendTime` 和实际 counts；失败尝试不会伪造成功发送时间。`PipelineTracer` 按命令序号异步回填同一帧，处理“设备发送快于帧提交”的竞态。

## 时间有效性规则

CSV 同时写入每个时间点的 steady-clock 纳秒值和分段年龄：

- `BackendQueueMs`：后端取得到捕获提交；
- `SubmitToInferenceMs`：捕获提交到推理开始；
- `InferenceToPublishMs`：推理开始到检测发布；
- `PublishToControlMs`：检测发布到控制消费；
- `LocalObservationAgeMs`：后端取得到控制消费。

时间未知时字段为空；时间完整但顺序反常时保留有符号负值，并设置 `TimingAnomaly=1`，不把异常裁剪成零。正常完整帧要求：

```text
backendReceive ≤ captureSubmit ≤ inferenceStart ≤ inferencePublish ≤ controlTime
```

## 验证

- 新增后端默认时间契约、最新帧替换同步时间、检测缓冲完整往返、完整但乱序时间标记和负时长保留测试。
- 新增命令成功确认、失败尝试、队列淘汰和追踪器异步回填测试。
- DML Release 正式重配置构建通过，CTest `8/8` 通过。
- CUDA Release 正式重配置构建通过，CTest `8/8` 通过。
- 算法测试目标复制当前后端 OpenCV DLL，避免测试目录运行时依赖缺失。

## 边界与后续

本阶段没有把协议时间映射到本机时间，也没有修改旧控制器、预测器或队列策略。下一阶段 P0-2 才在这条可审计时间轴上实现角度坐标和固定延迟相机响应模型。
