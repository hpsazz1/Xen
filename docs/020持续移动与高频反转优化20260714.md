# 持续移动与高频反转优化

## 背景

2026-07-14 更新 `C:\Users\16143\Desktop\XenMovingPI\DML\ndi` 三份测试数据：`horizontal_left/right` 改为长时间持续移动，每个文件包含三段约 3～4 秒试次；`horizontal_reverse` 改为约 8.9 秒的高频左右晃动。配置继续保持 320×320 检测、2560×1440源画面、80 ms比例响应、320 ms积分时间和1440 counts/s上限。

本轮不再使用“低频往返至少5次”的旧场景假设。单向文件用于验证长期稳态误差，高频文件用于验证中心回差、积分解卷绕和快速方向变化。

## 采集链路结论

- DML检测发布约86～93 FPS。
- NDI实际到达约220～237 FPS。
- 观测年龄平均约14 ms，P95稳定为20 ms。
- 全部试次最大移动队列深度为1。
- 分析器有效试次中 `SpeedLimitedPct` 全部为0。

因此当前表现不受 NDI 网络、解码、DML观测积压、发送队列或1440 counts/s上限限制。优化范围继续限定在共享控制器和分析口径。

| 持续向右 | 3 | 23.27 px | 23.90 px | 20.6 ms | 443.2 counts/s | 0% |

持续向左第一段在扣除预热后的351个样本中，平均绝对误差1.03 px、P95为2.38 px、闭环滞后3.1 ms，证明PI能够在DML低推理帧率下长期维持目标速度，不再出现旧版约80 ms的结构性稳态误差。

其余试次的P50多为1～9 px，P95约21～24 px。高分位主要来自每段开始、目标运动变化和偶发较大检测残差，不是持续限速或队列积压。当前没有数据依据同时修改响应、积分或最大速度。

## 剩余根因

上一轮已经避免亚像素越心直接清积分，但当积分输出自然衰减到0.5 count以下时，控制器仍会进入静止锁存。锁存后只按8 px释放半径恢复，不检查误差是否正在快速变化。

高频数据中出现7次锁存起点、累计35个 `Settled=1` 帧。典型序列为：

```text
ErrorX: +1.775 -> -3.310 -> -5.704 -> -7.929 px
Settled: 1      -> 1      -> 1      -> 1
FinalMx: 0      -> 0      -> 0      -> 0
```

目标已经在回差内部快速反向移动，但控制器连续停发，直到误差越过8 px。该行为不会造成积分拖拽，却人为增加约3～7个DML控制帧的高频反转滞后。

## 设计与实现

在PI稳定态增加帧间误差运动释放，不增加配置项：

1. 保存上一有效控制帧的二维误差。
2. PI启用时计算相邻误差变化量。
3. 变化量达到 `max(1 px, 稳定半径×0.25)` 时判定目标仍在移动；当前320检测默认稳定半径5 px，因此阈值为1.25 px/有效观测。
4. 即使误差仍在8 px释放半径内，也立即解除锁存并执行本帧控制。
5. 同一帧禁止因仍位于5 px稳定半径而重新锁存。
6. 误差变化低于阈值时继续保持静止，保留检测量化抑制。
7. `move_integral_time_ms=0` 的纯比例路径不启用快速释放，保持既有静止基线不变。

阈值使用“每个有效观测的像素变化”而不是速度估计，避免不稳定时间戳放大；同时只在PI模式生效，因为持续运动能力来自积分状态。

## 单元测试

- PI中心目标首次进入稳定态。
- 稳定态内误差由1 px快速变化到3 px时必须当帧恢复输出。
- 1 px变化到1.5 px的亚像素检测抖动必须继续保持稳定。
- 静止过冲从-6 px快速回到-4 px时先保持控制；下一帧误差不再变化后必须稳定。
- 原DML 60 FPS匀速、轻微越心、5 px积分解卷绕、静止阶跃和限速抗饱和测试继续通过。
- 移动分析器默认12 px阈值必须识别连续中等摆幅反转。

## 验证与后续复测

代码验证执行相关单元测试、DML/CUDA Release构建以及两后端完整CTest。下一轮仍使用相同DML+NDI三文件场景和参数：

1. 持续左右移动各三段，确认P95不回退且 `SpeedLimitedPct=0`。
2. 高频晃动保持约9秒，重点检查 `Settled=1` 是否只剩静止间隙，不再连续跨越快速误差变化；后续复测发现程序身份无法从旧CSV验证，详见 `docs/021复测版本身份与陈旧程序防护20260714.md`。
3. 反转次数应继续约25次且全部恢复；恢复均值目标低于本轮96 ms。
4. 高频通过后执行上中、左下、右下静止冒烟，确认快速释放阈值不会被正常检测噪声触发。
5. 移动与静止均通过后，再决定是否将320 ms积分从现场候选推广为源码默认。

## 后续建议

本轮不引入预测、不修改滤波器、不提高最大速度，也不改变80/320参数。持续单向已经证明PI闭环有效；后续只根据高频复测判断快速释放是否充分，避免重新堆叠速度前馈或多层平滑。
