# r28 jump 单轮复测与冻结

## 背景

本轮只上传了`horizontal_jump.csv`，数据来自DML构建`1788db06c01d`、控制器r28。文件包含1651行、约17.7秒连续数据，分析器只识别出1个有效试次，没有reverse复测文件。

## 指标对比

相对r27单轮数据：

- X轴P95由53.62降至37.54 px，改善30.0%。
- X轴P99由109.87降至51.30 px，改善53.3%。
- Y轴P95由18.34降至15.75 px，改善14.1%。
- 总速度限速率由7.5%降至3.7%，水平闭环滞后由31.0降至27.9 ms。
- 预测提前P95由24.00降至19.83 px，高速瞬态抑制20.6%，没有误入高频往返抑制。

r28的误差、尾部和限速指标全部达到上一轮设定目标，说明可信同向加速即时撤销至少没有造成回退。

## 不能继续调参的原因

- r27与r28都只有1个连续试次，不满足原定4轮复测要求。
- r28目标X速度P95为1908.5 px/s，低于r27的2500 px/s；检测残差P95为23.52 px，也低于r27的27.88 px。
- 两轮起始X误差分别为79和54.5 px，动作起点和强度并不一致。
- r28预测中断为54次、连续预测中位2帧，虽然高于r27的46次和3帧，但整体误差反而明显下降，不能仅凭中断次数认定预测仍有问题。

因此本轮冻结`1788db06c01d / r28`，不修改预测门控、追赶曲线、速度上限或reverse走廊。继续基于单轮数据调整会把测试动作差异拟合进控制逻辑。

## 下一轮标准

- 继续使用当前r28 DML程序，不重新构建。
- 在同一个`horizontal_jump.csv`中完成4轮，每轮持续至少5秒。
- 每轮开始前先把目标移动到准星附近，再按下瞄准键，减少起始误差差异。
- 每轮结束后松开瞄准键至少300 ms；分析器按150 ms空档拆分试次，300 ms可留出稳定余量。
- 4轮均要求X/Y P95不高于53.62/18.34 px、X P99不高于109.87 px、限速不高于7.5%。
- 通过后回归left/right各2轮和reverse 1轮，再决定是否结束预测模块优化。
