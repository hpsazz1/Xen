# P0-5 实机重捕获实际退出帧重评

## 背景

182号复核把每个reset代次截断为固定20帧，并用“第20帧前是否退出限速”判定恢复包络。用户补传`horizontal_left4.csv`和`horizontal_right4.csv`，要求以实际可观测长度重新统计退出帧。复查发现原六个文件的合格代次本身保留了131～496帧，固定截断遗漏了第21～45帧的真实退出，因此此前57.14%/60.00%的退出率是假阴性，不应继续作为门禁结论。

## 身份与批次边界

上传目录包含两个干净DML/r64身份：

- 原批次left/right各3个文件：`DML / 010aed0538a5 / 20260720T004343Z / r64`；
- 补传`horizontal_left4/right4.csv`：`DML / 8e6e3a87e6cd / 20260720T030226Z / r64`，SHA-256分别为`E605DFC8F4C2A1D3FFC59AD4F71A0C1982937965285E1296A73BC35B09A4CC08`和`12C986837D0043B4DCCAD3055E4176F5531EBA07FB8F20852139D118ADFF378D`。

分析器默认继续拒绝混合身份；新增`-ExpectedBuildRevision`只允许显式选择一个完整12位提交分别复核，不把两个批次聚合成同一批数据。两个身份之间没有控制器或运行参数变更，但审计输出仍保留批次边界。

## 口径修正

`RecoveryFrames=20`现表示最低可观测长度，而不是强制截止帧：

1. reset代次不足20帧即再次丢失或文件结束时，记为右删失候选，不计入事件和覆盖；
2. 满足最低长度后继续扫描同一reset代次，直到首次`AimPipelineControlSpeedLimited=0`，输出真实`SaturationExitFrame`；
3. 若整个合格代次结束仍未退出，才记为未退出并阻断总体门禁；
4. 汇总新增退出帧P50、P95和最大值，总体仍要求两个方向退出观测率均为100%。

回归测试覆盖“退出发生在最低窗口之后”、同代次始终未退出、短代次排除、混合身份默认拒绝和显式身份筛选。

## 补传批次作用

补传批次纳入left 6、right 5个长代次，每段保留524～564帧，证明现场能够提供远超过45帧的观测长度。其首帧残差P50仅为66.06/59.39 counts，退出帧P50均为1，最大为4/1；它们不是约300 counts恢复包络的同类事件，且right不足6次，因此不用于替代原批次代表性统计。补传批次的events和summary SHA-256分别为`50626DF7829531E9E1B376E08E415EEFC774F9FC65B5BA796540CFD68BB4BB28`和`7BE77B7DCF636FE62E8E350868B942F7A77A2EA21D232A2729FC559886A892D8`。

补传文件中另有若干约281～285 counts候选只持续1～2帧便再次丢失，属于右删失，不能虚构退出帧，也不进入覆盖分母。

## 结论与下一步

实机重捕获包络门禁按实际退出帧重评后通过。约300 counts首帧残差、100%首帧限速和最长45帧后全部退出均有实机证据。该结论只解除“实机包络不代表回放”的证据阻断，不授权直接提高1440上限、修改运行时控制器或进入Active；P0-5仍为`HOLD_SHADOW`。

下一步按181号协议只允许离线评估“恢复代次内、实际退出前”的限域速度包络反事实，并重新验证179号四端点共同域485项零损失。任何候选都必须保持非恢复帧、static、jump和reverse行为不变，在共同队列门禁通过前不得进入配置、UI、Shadow或Active。
