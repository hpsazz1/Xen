## 计时口径修正

旧 `DmlTotalMs` 没有覆盖输入形状和 `Ort::Value` 包装阶段，因此理论流水线约104 FPS而实际发布约94 FPS。新版本增加：

- `DmlTensorSetupMs`
- `DmlModel`

并将 `DmlTotalMs` 改为从预处理开始到后处理结束的完整墙钟耗时。下一轮不得把旧总耗时和新总耗时直接混合；旧数据只用于确认推理占绝对主导。

## 优化选择

当前发布目录同时具备 `yolo12n_cs2.onnx` 与 `yolo12n_cs2_fp16.onnx`。FP16是DirectML成熟支持的模型精度路径，能够直接减少算术和带宽压力，且无需改变检测分辨率、采集、控制器或后处理契约。因此下一轮优先做同架构FP32/FP16单变量A/B，不改ORT线程、批大小或控制参数。

不采用批处理：实时瞄准目标是降低单帧延迟，批量推理会等待多帧并增加观测年龄。暂不切换DirectML适配器选择API：多GPU设备可能改变实际执行设备，必须在模型精度A/B无收益后单独评审。

## 下一轮测试方案

1. 使用新DML构建，保持320×320、2560×1440、NDI、80/320/1440、置信度、NMS和最大检测数不变。
2. 基线模型为 `yolo12n_cs2.onnx`，候选为 `yolo12n_cs2_fp16.onnx`；CSV中的 `DmlModel` 必须与目录一致。
3. 按 `FP32→FP16→FP16→FP32` 执行ABBA。每次切换模型后先预热30秒，不记录预热数据。
4. 每轮分别采集静止、持续右移、高频反转各2000行，目录为 `XenDmlPerformanceFP32` 与 `XenDmlPerformanceFP16`。
5. 使用 `tools/analyze_dml_performance.ps1` 比较 `Mean/P95InferenceMs`、`Mean/P95TotalMs`、`PublishedInferenceFps`、`ObservationAgeP95Ms` 和 `DmlTensorSetupMs`。
6. FP16通过门槛：推理均值或P95至少改善10%，发布FPS至少提高8%，总耗时P95至少改善8%；观测年龄P95、持续移动稳态P95、高频误差与恢复P95均不得回退超过10%。
7. 若FP16性能通过，再补上中、左下、右下静止精度冒烟；检测类别、框位置或置信度出现明显差异则回退FP32。
8. 若FP16未达到门槛，保持FP32并单独评审DirectML高性能适配器选择与ORT格式模型，不继续优化0.17 ms预处理路径。
