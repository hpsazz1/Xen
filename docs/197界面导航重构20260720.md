# 界面导航重构

## 背景

现有 Dear ImGui 配置面板延续了上游项目的英文信息架构，中文名称多为逐字翻译；同时侧边栏混入外观、游戏绘制和性能诊断入口，核心调参路径较长。部分控件有悬浮说明，但侧边栏说明字段没有真正显示，用户只能逐页进入确认用途。

## 原因与排查

界面入口统一定义在 `Xen/overlay/overlay.cpp` 的 `kOverlayTabs`。每项已经包含 `description`，但 `DrawSidebarTabButton` 只绘制图标和标题，没有使用该字段。面板外观、游戏叠加、绘制样式、目标标记和性能统计均为独立绘制函数，因此可以仅移除导航注册而保留配置读取与运行兼容，不会导致旧配置失效。

当前应用是 Windows 原生叠加层，渲染、配置保存和运行线程共享进程内状态。直接改为局域网页面需要额外设计鉴权、监听地址、跨线程配置快照、保存事务和网络暴露边界；本次没有在缺少安全契约的情况下加入 HTTP 服务。

## 设计

1. 导航按真实工作流归并为“感知、瞄准、控制、系统”四组。
2. 菜单使用操作目的命名，例如“画面采集、目标检测、移动响应、目标预测、输入设备、快捷键”，避免逐字翻译。
3. 所有侧边栏项统一从 `description` 显示悬浮说明。
4. 从导航移除面板外观、游戏叠加、绘制样式、目标标记和性能统计；保留底层函数与配置兼容，控制本次改动范围。
5. 主题改为低饱和深灰和青绿色强调色，缩小圆角与间距，降低装饰性视觉元素对连续调参的干扰。

## 实现

- 重写 `kOverlayTabs` 的中文分组、菜单名和说明。
- 在 `DrawSidebarTabButton` 中对所有带说明的菜单统一绘制 Tooltip。
- 更新图标分组颜色匹配逻辑，使其直接使用中文分组。
- 扩充 `tests/ui_visibility_tests.ps1`，同时约束核心入口存在、精简入口缺席和 Tooltip 接线完整。

## 后续建议

若继续实现局域网调参，建议先限定为显式启用、仅监听私有网卡、随机访问令牌、配置白名单和单写者事务模型，并将 Web 服务放在独立适配层，不让网络线程直接持有运行配置引用。该工作应作为独立功能开发，配套威胁模型、协议测试和并发测试。

## Codex 客户端配色调整

用户提供了完整的 `codex-theme-v1` 定义，因此主题不再采用近似深色方案，而是精确切换为 `light` 变体：`surface #ffffff`、`ink #1a1c1f`、`accent #339cff`、`diffAdded #00a240`、`diffRemoved #ba2623`、`skill #924ff7`，窗口保持透明合成。主面板、共享设置区、弹出层、表格和控件使用白色与连续浅灰表面，标题、导航选中态、滑块、复选框和焦点状态统一使用蓝色 accent。

旧的霓虹背景渐变、外圈光晕和 HUD 角标已经移除，避免仅换色后仍保留与 Codex 客户端不一致的视觉语言。本次只修改显示颜色和装饰绘制，不改变布局、配置字段、线程模型或运行行为。UI 静态测试新增精确 accent 与旧颜色缺席约束。
