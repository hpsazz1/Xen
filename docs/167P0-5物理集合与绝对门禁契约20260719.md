# P0-5 物理集合与绝对门禁契约

## 背景

`docs/166P0-5实测物理响应与门禁敏感性20260719.md`证明，原逐物理对象、逐域严格击败legacy的门禁会被小于240 Hz单帧周期的响应宽度显著扰动。孤立KMBOX脉冲只支持`t50≈14 ms、t90≈15 ms`和亚帧宽度无法继续辨识，不能把单个15/0模型伪装成精确物理真值，也不能通过任意增加5%或8%非劣界恢复通过数。

本轮冻结r64程序、五段视频、162域矩阵、门控CA `8000/12/120`、候选响应80 ms和1440 counts/s，只补齐物理中心下界并定义可执行的两层晋级契约。目标是明确下一轮算法改进必须满足什么，不在候选仍明确失败时提前改写运行代码。

## 实测物理不确定集合

物理对象采用以下四个端点，不在端点之间扫参：

| 中心/宽度 | 依据 |
|---|---|
| 14/0 ms | 孤立脉冲`t50`下界与整步响应 |
| 14/2 ms | `t50`下界与不可分辨亚帧宽度上界 |
| 15/0 ms | 孤立脉冲`t90`集中位置与整步响应 |
| 15/2 ms | `t90`集中位置与不可分辨亚帧宽度上界 |

2 ms小于240 Hz单帧4.17 ms，不解释为已精确测得的线性核；它只表示当前采样分辨率不能排除的有限响应端点。60/0保留为历史假设回归基准，不再属于实测物理集合；DML shadow的20/20和35 ms不确定度尾部用于解释实机观测，不替代闭环物理对象。

## 两层晋级契约

### 第一层：逐域绝对安全包络

每个物理端点和每个离散域必须独立满足，任何一项失败均保持shadow：

1. 比较样本非空、全部核心指标有限，legacy与候选使用相同视频、检测时轴、重置边界和构建身份。
2. 候选二维误差P95不得越过该FOV下320×320中心检测裁剪区的内接角半径，即水平半角与垂直半角的较小值。两轴半角直接由现有透视投影、`160/2560`水平归一化半径和`160/1440`垂直归一化半径计算，不新增经验百分比。该保守边界只证明二维P95仍处于裁剪区内接圆，不能替代命中精度门槛。
3. 目标丢失后非零输出不超过一个控制周期；不得出现非有限输出、样本队列变化或相对legacy数量级退化。
4. static候选输出方向翻转不得高于已验证legacy；该项是设备静止安全不变量，不能被更小平均误差抵消。

四端点候选P95与检测裁剪内接角半径的最大比值依次为102.00%、96.03%、102.00%、97.20%。14/0和15/0均由`jump/domain_fov120_2560x1440_144fps_40ms_1.5x`越界约2%；14/2和15/2虽然未越界，最差也只剩3.97%和2.80%余量。由于物理集合要求四端点全部安全，候选已在第一层失败，不能把有限宽度端点的较好相位用于覆盖整步端点。static每组另有14域因候选翻转增加失败。

### 第二层：物理集合级场景质量

通过第一层后，再按每个物理端点的完整162域场景集合评价质量，避免单一离散域的相位微扰改变整个晋级结论：

- left/right：候选场景平均P95不得高于legacy场景平均P95，且任何域控制中断率不得增加超过5个百分点；不引入5%或8%误差非劣界。
- reverse：保留逐域“P95至少改善10%或框内率提高10个百分点”，并保留后半段不得变慢；高速反转不能由场景均值掩盖局部漏跟。
- jump：保留逐域Y轴P95至少改善10%和丢失后一周期归零；垂直安全收益不能被其他轴平均。
- static：除第一层翻转不变量外，每个物理端点的场景平均P95不得比legacy回退超过5%；不得用均值覆盖任一翻转失败域。
- 四个物理端点必须全部通过；不得选择最有利端点，也不得把四组样本混合后只算一个总均值。

这套契约不是把失败改写为通过：它用绝对裁剪边界阻止集合均值掩盖局部失控，同时用冻结的场景级零容差比较消除亚帧宽度对单个域排序的偶然翻转。

把每个域的legacy P95改成四端点最坏legacy包络仍只有498/810，证明“集合内挑一个较宽松legacy上限”也不能形成有效契约，正式否决该替代方案。
