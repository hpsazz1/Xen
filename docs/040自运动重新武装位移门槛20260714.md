# 自运动重新武装位移门槛

## 背景与版本

2026-07-14使用 `BuildRevision=183f5145189b`、`ControllerRevision=13`、DML+NDI、3200 counts/s和预测50/50/1.0完成static九段复测。CSV身份与正式DML构建一致，全程未使用手动鼠标辅助。

## r13收益

r13收益明确：static X/Y预测生效率从r12的2.5%/2.6%降至0.4%/1.1%，平均P95误差从3.31/4.47 px降至2.26/3.57 px。r12 trial 9在保持期间重学反向回弹并连续输出9帧的问题完全消失，说明保持尾端丢弃受污染运动证据是正确方向。

## 剩余根因

残余主要集中在trial 4。保持尾帧约311 ms时基础误差为 `(0.5,-0.6) px`，约360 ms重新锁定时为 `(1.3,4.5) px`，净位移约5.2 px。该变化是目标越过中心后的短距离回弹，却刚好超过普通预测约4 px的净位移门槛，因此重新形成向上的稳定回归，随后连续6帧输出约5.9~7.8 px垂直提前。

继续延长固定保持时间不是合适方案：不同DML帧率下固定帧数对应时间不同，而且真实移动在保持结束后应尽快恢复。应直接提高重新武装所需的空间证据，并保持普通首次预测行为不变。

本轮NDI声明240 FPS，但CSV实际接收平均约158.5 FPS、P95约164 FPS，推理平均约88 FPS，明显低于r12接近238 FPS的接收状态。static结论仍有九段一致性支撑，但进入left/right和jump验收前必须先排除OBS、网络或接收端负载导致的采集降速。

## r14设计

1. 普通首次预测仍使用 `max(3 px, detectionSpan×1.25%)`，320检测跨度下为4 px，不影响持续移动首次响应。
2. 只有命中过自运动门控并完成保持后，重新武装阶段改用 `max(普通门槛, detectionSpan×2.5%)`，320下为8 px。
3. 小于8 px的越心回弹不能重新锁定方向；真实目标继续移动超过8 px后，仍使用原有稳健回归和两次方向确认恢复预测。
4. 一旦重新锁定真实方向，立即退出加强门槛，后续连续预测、停止和反向逻辑均保持现状。
5. 完整跟踪重置清除重新武装状态，禁止跨目标继承。

控制器修订号升级为14。单元测试覆盖保持后5.5 px反向回弹不得锁定，以及继续移动超过8 px后能够恢复反向预测。

## 下一轮测试

- 先排查OBS NDI输出、网络和接收端负载，使 `SourceReceiveFPS` 恢复接近声明240 FPS；至少应稳定高于220 FPS再做移动场景横向比较。
- 保持3200与预测 `50/50/1.0`，复测static九段，要求X/Y预测生效率均不高于0.5%，不得出现保持结束后的连续预测段。
- static通过后测试left/right各4轮，X轴P95不高于22.26/19.97 px，预测生效率至少95%，自运动抑制低于5%。
- jump连续20秒，单试次至少15秒且X轴P95不高于60 px；随后测试reverse和 `Target_static_Role_reverse` 各20秒。
- 目标飞出、目标死亡、角色死亡各10次，确认丢失后无输出。
