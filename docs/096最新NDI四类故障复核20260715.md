# 最新 NDI 四类故障复核（2026-07-15）

## 数据结论

本轮数据身份为 `BuildRevision=ecb5ec243260`、`ControllerRevision=37`，配置仍为 `120 ms / 3200 counts/s / 500 ms`。队列最大年龄约 0.9 ms，丢弃命令为 0，因此不是设备队列阻塞。

- `jump` 误差 P95 约 115 px，预测多次被自运动伪迹门控撤销，基础反馈追不上高速目标；
- `reverse` 末段仍有约 7~9 px 单侧滞后，预测偏移暂时归零时移动积分被清除，表现为漂移；
- `left/right` 仍有 18 次左右输出换向，说明闭环在观测抖动和控制延迟下反复修正；
- `static` 多个试次在到达中心前出现反向误差，近中心误差变化被当成持续移动，延迟了静态锁存。

## 修复设计

1. 自运动伪迹改为连续两帧确认后才撤销预测，避免单帧误判切断 jump 的前瞻；
2. 移动积分保持条件从“当前有预测偏移”扩展为“方向已锁定且速度可靠”，覆盖 reverse 反向确认期间的低速跟随；
3. 基础控制器新增内部 `allowMovingInsideSettle` 状态，生产链路只有确认移动目标时才阻止静态锁存；静态稳定半径由 5 px 提高到 6 px，减少终点延迟停发。该状态不是用户配置项。

以上改动不提高全局速度、不增加用户参数，仍保留旧配置兼容性。必须用下一轮相同五类数据验证 jump P95、reverse 稳态误差、left/right 换向频率和 static 首次反向过冲。
