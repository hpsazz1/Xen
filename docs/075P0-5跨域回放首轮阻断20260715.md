# P0-5跨域回放首轮阻断

## 背景与目标

P0-4A与P0-4B已经形成角度估计、P控制和确定性轨迹影子链，但原`--video-replay`只搜索r30预测参数，不能证明新链在同一观测和跨域变体下优于r30。本轮新增独立P0-5回放核心，目标不是强行产出可晋级参数，而是把同帧对照、变体矩阵、诊断和硬门槛固化为可重复执行的程序。任一安全门槛失败时必须输出`HOLD_SHADOW`，不得修改active能力。

## 数据与调用链

使用`C:\Users\16143\Desktop\Xen\Video`中的`static`、`horizontal_left`、`horizontal_right`、`horizontal_reverse`和`horizontal_jump`五段2560×1440、240.004 FPS视频，DML以94 FPS重新提取支点。提取结果新增真实检测框宽高和置信度，分别用于框内比例、边缘余量和Kalman测量噪声；重捕获结果也保留其对应元数据。

`--cross-domain-replay`复用既有DML检测与目标选择，只把参数搜索替换为以下同帧闭环：

1. r30侧使用`BasicTargetFilter + TargetPredictor + BasicAimController`，固定当前50 ms提前、50 ms速度窗口、1.0强度、80 ms响应和1440 counts/s；
2. 候选侧使用`RelativeLosKalman + LosAimController(P-only) + CommandTrajectoryShaper(off)`，前馈、积分和经验提前均保持关闭；
3. 两侧共享同一个目标角轨迹、观测时刻、观测延迟与60 ms命令到画面延迟，但维护各自相机状态；
4. 目标离开320×320有效检测窗时两侧都停止本段同帧评价，候选紧急重置并清除尚未生效的旧命令，禁止离屏后继续使用真值；
5. 逐帧明细只保存106°、2560×1440、94 FPS、20 ms、1x基准域，全矩阵保存汇总，避免机械复制数GB相同诊断。

## 跨域矩阵

物理域使用3档HFOV（90/106/120°）、2档分辨率（1920×1080/2560×1440）、3档回放FPS（60/94/144）、3档观测延迟（10/20/40 ms）和3档速度缩放（0.75/1/1.5x），共162组。另加7组相对运动变体：同向目标更快、近似等速、角色更快、反向，以及角色前移时目标位于左/中/右，共169组/场景、845组比较。

角色变体是对真实检测角轨迹做确定性相对视线缩放，用来验证“相对速度接近0”和“角色更快时自然反号”的控制语义，不伪装成含真实世界深度的实机角色移动数据。P0-6前仍需补齐真实角色同/反向与前后运动采集。

## 指标与门槛

汇总CSV记录角误差P50/P95/P99、Y轴P95、框内比例、边缘余量P05、控制中断、二维输出反转、估计方向错误与速率翻转、NIS、协方差、前馈可信度以及requested/shaped/sent counts。逐帧CSV保留创新、分量输出与三条counts时间线。

二维输出反转只在相邻非零向量点积为负时计数。首轮曾把“主轴从X切到Y”误算为反转，修正后重新跑完整矩阵；目标离开检测窗后仍使用真值的首版模型也已废弃。两项修正都只改变回放统计与可见性，没有调整候选参数。

门槛直接实现068文档要求：static P95回退不超过5%且反转不增加；left/right P95不回退且中断不增加5个百分点；reverse P95改善10%或框内率提高10个百分点并且后半段不变慢；jump Y P95改善10%且丢失后归零；跨域不得出现数量级退化。

## 首轮结果

最终矩阵为613/845通过、232失败，决策为`HOLD_SHADOW`：

- left/right：338/338通过；基准域P95分别由8.264/8.208°降至7.687/7.657°；
- static：9/169通过，160组因输出反转增加失败；基准域P95虽由0.136°降至0.059°、框内率由99.00%升至99.60%，但二维反转由4次增至170次，说明P-only候选缺少静止回差，不能用更小误差掩盖高频抖动；
- reverse：137/169通过，32组未达到P95降低10%或框内率提高10个百分点；基准域P95由1.517°降至1.209°，但框内率26.19%略低于r30的27.14%；
- jump：129/169通过，40组Y轴P95未改善10%；基准域Y P95由2.101°降至1.653°，但不是所有FPS/FOV/延迟组合都保持该收益；
- 所有同向、反向和角色更快的变体都按相对LOS自然缩放；当前前馈增益为0，因此本轮只验证P反馈安全基线，不宣称速度前馈已通过。

## 结论与后续

本轮完成了P0-5可重复回放基础设施和首轮晋级判定，但候选未通过全部门槛，`AimPipelineRuntime`继续强制active降级为shadow，控制器修订保持r30，P0-6不得开始。

下一次只允许修改`LosAimController`的静止状态机：采用角误差与相对LOS速率共同驱动的角度回差，先消除static反转，同时要求left/right不回退。通过后用完全相同845组矩阵复验；static通过之前不得同时调整Kalman Q/R、P响应、前馈增益或经验提前。轨迹`trapezoid`的固定240 Hz回放仍需在P0-5后续加入，当前结果只覆盖P0-4B的`off`等价基线，不能据此允许整形器active。
