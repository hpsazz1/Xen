# 独立机器Profile缓存与四级降级

## 背景

分轴主动协议已用`DML|13f67257f5c2|r64`完成3轮360脉冲，得到X/Y为0.515625/0.500000 px/count、典型`t90`约14.5 ms的稳定候选。该结果绑定当前设备、NDI链路、分辨率、裁剪和控制器，不能写入代表游戏坐标语义的`[Games]`，也不能因一次协议通过就自动启用active。

本轮目标是建立独立、可删除、默认关闭的机器缓存格式和运行决策层。它只允许读取人工审核产物，不在运行时创建或覆盖文件；所有已知身份精确匹配后才允许进入完整标定角度等级，否则明确降级并在UI和CSV留下原因。

## 数据流与边界

运行时在配置刷新时构造`MachineProfileKey`，读取显式绝对路径缓存，再生成不可变的`MachineProfileDecision`：

```text
[Games]用户坐标 + 捕获/设备/后端/控制器上下文
                         |
                         v
              严格字段级缓存匹配
                         |
                         v
               Level 0/1/2/3 决策
                         |
                         +--> UI与流水线CSV
                         +--> shadow角度比例与前馈可信度上限
```

legacy移动、`[Games]`保存、设备队列和active命令抑制均未改变。Level 0/1没有可信角度比例时，shadow停止角度Kalman和角度控制器，禁止把归一化/安全状态伪装成角度估计；Level 2/3只影响本来就不发送设备命令的shadow。因正式active语义未变化，控制器修订继续为r64。

## 缓存格式与写入安全

`MachineProfileRecord` schema v1分为严格键和证据两部分。键包含：

- 游戏Profile与瞄准模式；
- 捕获方式、捕获源（NDI源名等）、完整源宽高；
- 中心ROI的X/Y/宽/高；
- 推理后端、输入方式与设备身份；
- 灵敏度、yaw、pitch、HFOV/VFOV、FOV缩放及基准FOV；
- 控制器修订号。

证据包含X/Y的px/count与deg/count、t50/t90、可信度、trial数、协议名、采集构建身份和证据摘要。加载时要求schema为1、所有必需字段存在且类型有限、比例和时序为正、`t50≤t90≤100 ms`、可信度至少0.90、trial至少120，并拒绝空协议/身份/摘要。相同字段重复、格式错误、相对路径或不完整文件均加载失败。

`saveNew`只供后续人工审核工具调用，要求绝对路径且拒绝覆盖已有文件；MouseThread运行时从不调用写入API。新配置项默认值为：

```ini
machine_profile_cache_enabled = false
machine_profile_cache_path =
machine_profile_aim_mode = hipfire
```

因此升级后没有隐式缓存路径，也不会因同目录恰好存在文件而自动命中。当前用户配置未修改，主动三轮结果仍只作为人工候选。

## 四级策略

- Level 3 `calibrated_angle`：用户Profile有效，缓存显式启用、完整加载且所有键精确匹配；shadow使用缓存deg/count、完整前馈可信度倍率1.0，并标记标定响应可用。当前应用响应模型以X/Y实测`t50`均值作为命令响应中心，响应宽度固定为0；协议未直接测得的宽度不做推断。
- Level 2 `conservative_angle`：用户Profile有效但缓存关闭、缺失、损坏、证据不足或任一键失配；shadow继续使用用户角度，前馈可信度最多乘0.25，不采用缓存响应。
- Level 1 `normalized_image`：用户Profile无效但源/ROI几何有效；决策声明归一化图像空间，角度前馈和积分关闭。当前active仍冻结，因此只输出降级诊断，不生成设备命令。
- Level 0 `safety_direct_pursuit`：用户Profile与几何均无效，或未来运行失配监测给出严重失配；预测、前馈和积分均关闭。当前只作为安全决策和审计状态，不改动已冻结legacy链。

严重运行失配参数具有最高优先级，直接进入Level 0。键失配原因精确到首个字段，例如`cache_key_mismatch:captureSource`或`cache_key_mismatch:controllerRevision`，不会静默回退到名称相近的游戏缓存。

## UI、CSV与测试

Profile页面新增“机器标定缓存”只读状态区，显示路径、瞄准模式、Level、请求/加载/匹配、有效deg/count、前馈倍率和原因。流水线CSV新增`MachineProfileLevel`、`MachineProfileCacheRequested/Loaded/Matched`、倍率、双轴deg/count及原因列；构建后端、Git身份和控制器修订仍来自原统一构建身份。

新增独立C++测试覆盖：显式新建与拒绝覆盖、保存后加载、精确匹配进入Level 3、NDI源和控制器修订变化降到Level 2、缓存关闭、无用户Profile的Level 1、无几何及严重失配的Level 0。配置测试覆盖默认关闭、无隐式路径和字段往返；算法测试覆盖CSV列与shadow前馈倍率；UI测试覆盖入口可见。DML CTest由11项增加为12项，增量验证12/12通过。

提交前使用正式脚本对DML与CUDA Release执行clean-first全量构建，两个后端均成功生成规范产物；随后两个构建目录的CTest均为12/12通过。Level 2运行时测试还验证了0.25前馈可信度上限以及全局积分已配置时仍强制关闭积分。构建日志中的OpenCV、CUDA头文件及既有设备代码警告不属于本次改动，未出现新增编译错误。

## 下一步

下一步实现“人工缓存候选生成器”：只接受`ProtocolPassed=1`、`Recommendation=MANUAL_REVIEW_ONLY`、`ProfileAutoWrite=0`的主动协议结果，结合当前配置和明确ROI生成一个新的、不可覆盖的schema v1候选文件，并由同一C++加载器反向校验。生成候选仍不修改config或启用缓存；随后用shadow CSV验证精确命中与逐字段失效，再决定是否允许人工设置`machine_profile_cache_enabled=true`。active与legacy保持冻结。
