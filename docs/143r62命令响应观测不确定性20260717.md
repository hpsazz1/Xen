# r62命令响应观测不确定性

## 背景

r61 的中心 20 ms、宽度 20 ms 线性相机响应在三份九点 static 中仍有 97 个运行态机动启用帧、50 个收敛态侵入帧和 7 个启用段；共享、分轴及局部响应核搜索均无法得到零误启用候选。P0-6F 因此冻结响应核和 `maneuver_gated_ca + 8000/12/120`，改为评估“成功设备命令能解释多少观测速率”，并要求同时通过 static 负样本、jump 正样本和用户可达到最快速度的 reverse 正样本。

## 数据与正样本边界

数据位于 `C:\Users\16143\Desktop\Xen\DML\ndi`。三份 static 合计 27 个运行段；`jump\jump_r61_positive.csv` 有 5 个运行段；`reverse\reverse_r61_positive_fast.csv` 是用户当前能稳定采集到的最快 reverse，共 4 个运行段。慢速 `reverse_r61_positive.csv` 的峰值仅 9.29~10.12 度/秒，低于冻结的 12 度/秒物理门槛，只作为低速反证，不要求激活，也不再要求用户继续提速。

最快 reverse 的 4 段分别有 491/472/589/632 个基线机动启用帧，合计 2184；收敛态启用 721 帧；X 速率峰值 38.01/36.38/34.29/34.13 度/秒，四段均有明确换向。响应复放逐帧复现基线选择，差异为 0。

## 排查过程

离线复放首先按观测时刻计算仍处于 20 ms 线性响应区间的成功命令净角速度，逐轴不确定度为其绝对值乘以候选增益。门控证据改为：

```text
evidenceX = max(0, abs(kalmanRateX) - uncertaintyX)
evidenceY = max(0, abs(kalmanRateY) - uncertaintyY)
evidence = hypot(evidenceX, evidenceY)
```

该结构不修改稳定 LOS 测量、Kalman 状态、常加速度状态、12 度/秒门槛或 120 ms 保持。增益 0 完全保持历史行为。

增益 1.0 将 static 降至 11 帧、7 个收敛态帧和 1 段，残留全部位于 `static_r61_repeat2.csv` 的第 2 轮。逐帧诊断显示首次运行态触发时相机响应率为 3.08 度/秒，门控速度估计仍保留前序响应，扣除后证据为 12.34 度/秒，刚越过门槛；两条重叠命令同向，因此不是命令符号抵消，也不是仅由 120 ms 保持尾部造成。继续进行有界幅度搜索后，1.25 已消除残留，无需增加新的时间尾部。

## 离线结果

固定 20/20 响应核和精确 Profile 角度换算后：

| 增益 | static运行启用帧/收敛帧/段 | jump运行启用帧/覆盖轮次 | fast reverse运行启用帧/收敛帧/覆盖轮次 |
|---:|---:|---:|---:|
| 0 | 97 / 50 / 7 | 2122 / 5/5 | 2184 / 721 / 4/4 |
| 1.0 | 11 / 7 / 1 | 1150 / 5/5 | 1599 / 500 / 4/4 |
| 1.25 | 0 / 0 / 0 | 977 / 5/5 | 1440 / 463 / 4/4 |

1.25 是本轮首个同时满足全部硬门槛的候选。更高增益虽仍保留部分正样本，但没有必要扩大抑制范围，因此不采用。

## r62实现

`AppliedViewMotionModel` 新增观测时刻净相机角速度查询；DML shadow 以固定增益 1.25 形成逐轴不确定度并传入正式 `ManeuverLosEstimator`。CSV 新增固定增益、相机角速度、逐轴不确定度和扣除后证据，分析器校验有限值、1.25 身份、角速度乘积关系以及运行链传递一致性。离线工具增加 `--uncertainty-gains` 搜索、逐帧 trace 输出和汇总峰值，便于复核触发原因。

CUDA 继续强制 0 ms 阶跃响应和常速度 Kalman；新增不确定度不会改变 CUDA 选择。r62 仍为 `HOLD_SHADOW`，`active` 请求继续降级为 shadow，设备正式输出仍由 legacy 链路产生。
