# P0-5在途命令静止进入保护

## 背景

`docs/151P0-5静止真值与在途命令时轴20260717.md`证明，完整60 ms Smith投影能降低static误差，却会破坏left/right并增加static翻转。根因不是所有控制参考都需要未来投影，而是控制器进入settled时没有检查已经发送、尚未完全反映到画面的命令终点。本轮冻结r63门控CA、当前P误差、释放逻辑和全部门禁，只在settle进入判定增加独立离线输入。

## 设计与实现

`LosAimController::Input`新增可选`settleEntryError`。该误差只替换未锁存状态的连续安静样本判定，不参与P/I/前馈、低速反向确认或settled释放。未提供或数值非法时精确回退当前误差，正式运行路径因此保持原语义。

跨域回放新增默认关闭的`--candidate-settle-entry-command-guard 1`。它要求同时设置非零`--candidate-command-commit-horizon-ms`，使用已发送命令模型查询承诺终点。逐帧CSV记录`SettleEntryErrorX/Y`，summary与decision记录实际开关身份。

另增加默认关闭的`--candidate-settle-entry-command-hold 1`反事实：当当前误差和速率已经安静、承诺终点仍越界时，不累计quiet sample且暂时不发送新命令；任一当前量离开静止带便恢复原控制链。CSV用`SettleEntryCommandHeld`审计该过渡态。

## 有界反事实

本轮不扫描horizon，只使用既有物理命令响应延迟60 ms：

| 候选 | 总通过 | jump/left/reverse/right/static | 相对721基线 |
|---|---:|---:|---:|
| guard继承正式视角时轴 | 497/810 | 非隔离 | 作废，混入已否决的view-motion候选 |
| guard并外推阈值内目标速率 | 752/810 | 156/145/155/147/149 | 救回36、损失5 |
| guard + 过渡hold | 748/810 | 156/145/155/147/145 | 救回36、损失9，否决hold |
| guard冻结当前目标角 | 753/810 | 156/145/155/147/150 | 救回37、损失5，当前最佳 |

阈值内速率最大仍可达到1.2°/s，外推60 ms会人为加入最多0.072°，接近完整0.08°静止带。冻结当前目标角、只投影已承诺相机运动更符合“静止进入”假设，也比速率外推多通过1项。

## 最佳候选审计

冻结目标guard使static候选P95均值从0.08159°降到0.07005°，static总输出翻转从344降到218；dynamic四场景通过数与基线完全一致。代表明细域中guard确实阻止了当前误差已安静、承诺终点仍越界的过早进入。

但它仍损失5个原通过域，全部为`FOV120 + 40 ms`的static翻转失败：

- `1920x1080 / 144 FPS / 1.5x`；
- `2560x1440 / 94 FPS / 0.75x`与`1.5x`；
- `2560x1440 / 144 FPS / 0.75x`与`1.5x`。

这些域legacy翻转为2~3次，guard候选增到5~6次。虽然总量和绝大多数域改善，P0-5门禁要求原721项零损失，不能用净收益覆盖局部安全回退。

## 结论与下一步

冻结目标guard保留为当前最佳离线候选，默认关闭；过渡hold正式否决。两者均不接入`config.ini`、UI、正式shadow或active，控制器修订保持r63，`HOLD_SHADOW`不变。

下一步只允许对上述5个损失域输出逐帧明细，比较guard前后首次新增翻转的当前误差、承诺终点、quiet sample、settle进入与迟到命令时序。目标是判断损失来自guard持续过长、承诺终点跨零，还是整数counts方向历史；不得扫描horizon或放宽翻转门禁。
