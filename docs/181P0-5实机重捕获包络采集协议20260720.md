# P0-5 实机重捕获包络采集协议

## 背景

180号诊断在四端点跨域回放中确认：left/right检测恢复后的首帧残差约303～305 counts，主导相位是1440 counts/s二维上限下约106 ms的连续限速；估计器冷启动仅持续2～3帧。下一步必须验证该残差是否真实出现在设备重捕获，而不是继续修改控制器或端点守卫。

当前工作区和既有 Desktop 归档均未发现符合本协议的最新 left/right 实机重捕获 CSV，因此本文件只定义可复现采集和判定口径，不宣称已有实机结论。

## 采集约束

1. 使用当前 DML Release 程序和当前已验收配置；保持`move_max_speed_cps=1440`、响应、FOV、检测分辨率、Profile和Shadow设置不变。不要在本轮打开active、轨迹整形或端点守卫。
2. 开启流水线追踪，`pipeline_tracer_max_frames`设为至少2000。每个方向单独导出CSV，文件名必须包含`left`或`right`，例如`horizontal_left_reacquisition_01.csv`。
3. 每个方向至少完成6次独立重捕获：保持目标水平移动，故意让目标离开检测区直到发生完整目标reset，再让目标重新进入检测区；每次恢复后至少保留20个检测观测帧。轮次之间不要重启程序或改变配置。
4. 目标重新进入前的检测空洞必须达到当前过期门槛；不要用仅松开瞄准的短暂停顿替代目标丢失。短暂停不会触发本协议要测量的完整reset。
5. 导出前确认文件没有被其他场景混入；将同一批次所有文件放在同一个目录，例如`C:\Users\16143\Desktop\XenReacquisition\DML\ndi`。

每条记录必须来自同一个构建身份，并在CSV中保留`BuildBackend`、`BuildRevision`、`BuildTimestampUtc`、`ControllerRevision`。`AimPipelineObservationSequence=1`且`AimPipelineResetGeneration`递增的首个有效观测是恢复首帧；`ControlTimeNs`用于计算空洞和控制周期。

## 分析器与输出

采集后运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools/analyze_real_reacquisition_envelope.ps1 `
  -DataRoot C:\Users\16143\Desktop\XenReacquisition\DML\ndi `
  -OutputEventsCsv C:\Users\16143\Desktop\XenReacquisition\recovery_events.csv `
  -OutputSummaryCsv C:\Users\16143\Desktop\XenReacquisition\recovery_summary.csv
```

`tools/analyze_real_reacquisition_envelope.ps1`只接受left/right文件，并强制检查：

- 所有文件身份完全一致、无`-dirty`、控制器修订为r64；
- 有效Shadow观测和`ViewMotionShadowValid=1`；
- 每行仍使用1440 counts/s上限；
- reset代次递增、观测序列从1开始，且检测空洞不短于`max(100 ms, command delay + response/2)`；
- 恢复首帧水平残差用`RelativeErrorYawDegrees / DegreesPerCountX`换算为counts；
- 连续限速时长使用逐帧`ControlTimeNs`的实际中位间隔，不把命令响应宽度误当控制周期。

输出事件文件保留恢复窗口逐帧残差、估计速率、无限请求、单帧上限、限速、方向请求和命令发送结果。汇总文件按方向给出事件数、空洞、残差P50/P95、跨域参考偏差、限速帧数/时长和命令失败事件。

## 判定门禁

只有同时满足以下条件，才能把实机结果用于下一轮离线候选评估：

1. left和right各至少6次完整事件；
2. 恢复首帧请求方向正确率为100%，窗口内无命令发送失败；
3. 两方向首帧残差P50相对跨域参考（left 303.442641、right 304.589065 counts）偏差均不超过20%；
4. 恢复窗口的连续限速均能观察到退出，且构建身份、Shadow状态和1440上限无违例。

若门禁通过，下一步只评估“恢复窗口限定”的离线速度包络反事实，并重新验证485个四端点共同域零损失；若门禁不通过或实机残差明显低于参考，则不提高设备上限，把跨域动态失败归因改回检测覆盖/回放包络不一致，不修改运行时控制器。
