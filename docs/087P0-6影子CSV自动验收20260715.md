# P0-6 影子 CSV 自动验收

## 1. 背景

P0-6A 已提供 UI 灰度入口，但现场 CSV 若继续人工逐列检查，容易遗漏模式降级、命令抑制、构建身份或后续阶段组件误开启。仓库现有 DML/NDI 三份 CSV 均来自旧提交 `90e3aeccb441`，没有 `AimPipeline*` 影子字段，不能作为 P0-6 数据。

本轮新增正式、可复用的 `tools/analyze_shadow_pipeline.ps1`，只负责 P0-6 第一步的安全契约和数据完整性验收，不评判 active 收益，也不修改任何运行参数。

## 2. 验收内容

每个 CSV 自动检查：

1. `BuildBackend=DML`、控制器 r30、单一构建提交、无 `dirty`；可通过参数指定预期提交。
2. 请求模式和有效模式均为 `shadow`。
3. `ActiveAvailable=0`、影子处理有效、控制与轨迹命令均保持抑制。
4. 完整时间戳样本必须满足时间顺序。
5. 有效估计样本的协方差、innovation variance 必须为正有限值，NIS 必须为非负有限值。
6. P0-6 第一步必须保持轨迹模式 `off`，速度前馈、经验提前和积分输出均为 0。
7. 每文件有效估计样本达到下限。
8. 使用 `-RequireStandardScenarios` 时，必须同时包含 static、left、right、reverse、jump 五类文件。
9. 多个文件不得混用构建身份。

任一条件失败时，脚本仍可先写出逐文件汇总，随后以错误退出，禁止把部分成功误报为整轮通过。

## 3. 正式命令

完成五类采集后执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools/analyze_shadow_pipeline.ps1 `
  -DataRoot "C:\Users\16143\Desktop\Xen\DML\ndi" `
  -ExpectedBuildRevision <本轮12位程序提交> `
  -MinEstimateSamples 30 `
  -RequireStandardScenarios `
  -OutputCsv "C:\Users\16143\Desktop\Xen\DML\ndi\shadow_pipeline_summary.csv"
```

输出包含每文件及 Overall 的样本数、构建身份、模式违例、时间违例、Kalman 诊断违例、阶段越界违例、状态和原因。

## 4. 测试设计

新增 `tests/shadow_pipeline_analysis_tests.ps1`：

- 构造五场景、同一 DML 提交、shadow 契约完整的数据，要求通过并正确汇总 10 个估计样本。
- 构造请求 active 且未抑制命令的数据，要求脚本失败。
- CMake/CTest 注册为 `xen.analysis.shadow_pipeline`，纳入双后端正式测试。
