# 五类NDI移动目标标准优化

## 背景

2026-07-15 对 `C:\Users\16143\Desktop\Xen\DML\ndi` 的 `static`、`horizontal_left`、`horizontal_right`、`horizontal_reverse` 和 `horizontal_jump` 进行复核。CSV 身份统一为 DML、提交 `f9ffe6a501c5`、控制器 r30，源画面 2560x1440，检测 320，推理约 92~98 FPS，观测年龄 P95 约 19~20 ms。

## 现象与原因

- `horizontal_jump` 的 P95 水平误差为 108.38 px，36.2% 样本触发 1440 counts/s 限速，说明现场配置仍使用旧速度上限，无法覆盖高速横跳。
- `horizontal_left/right` 稳态绝对误差约 12~13 px，且左右分别存在相反的固定偏差；比例控制在无积分时无法消除匀速目标的稳态误差。
- `horizontal_reverse` 的闭环滞后约 91.8 ms，平均输出仅 127 counts/s。低速反向确认会抑制短脉冲，但没有积分保持，目标在中心附近持续漂移。

## 设计与实现

将五类 NDI 移动目标统一采用已验证的现场标准：

1. `move_max_speed_cps = 3200`，按实际观测间隔换算每帧预算，解除 jump 的 1440 cps 硬瓶颈。
2. `move_integral_time_ms = 320`，保留现有积分限幅、越心解卷绕和稳定半径清除规则，消除持续移动偏差并维持 reverse 低速贴目标。
3. `move_response_ms = 80`、预测 50/50/1.0 和 legacy 输出模式保持不变，避免通过激进 P 增益放大换向抖动。

控制器修订号由 r30 升至 r31。配置默认值、缺省加载值、文档和现场 `ndi/config.ini` 同步更新，配置测试同步校验新标准。

## 验证要求

提交前执行 `git diff --check`、配置和控制器单元测试、DML Release 与 CUDA Release 构建及完整 CTest。重新采集五类 CSV 时必须核对 UI 标题、CSV 和可执行文件均为同一干净提交、DML 后端和 r31；重点观察 jump 限速比例、left/right 稳态偏差、reverse 低速输出和恢复时间。

## 后续建议

若 r31 复测仍出现 reverse 中央走廊抖动，下一轮只调整积分释放半径或积分输出比例，不同时改变响应时间、预测窗口和速度上限。
