# 标准视频闭环模拟与预测候选

## 背景

人工重复右移、左移和高频反转容易产生速度、起点和持续时间差异，不适合作为预测参数的初始大范围搜索。2026-07-14录制四段固定角色视角视频，完整画面为2560×1440，OBS容器实际帧率均为240.004 FPS，实机NDI仍使用固定中心裁剪 `x=1120,y=560,width=320,height=320`。录像期间镜头不跟随目标，因此同一目标轨迹可以供所有参数候选共享。

## 设计与实现

新增DML命令行回放模式。工具使用与实机相同的 `yolo12n_cs2.onnx` 和目标支点选择逻辑，在跟随目标的320×320观察窗口内提取全屏目标坐标。主窗口丢失时每250 ms执行一次带25%重叠的全宽重捕获；正常连续检测不增加额外推理。

闭环模拟以提取后的全屏轨迹为外部输入，按游戏FOV、灵敏度和yaw/pitch把 `BasicAimController` 输出counts换算为虚拟镜头位移，再重新计算目标相对固定中心裁剪的位置。每个候选复用真实 `BasicTargetFilter`、`TargetPredictor`、`BasicAimController`，保持80 ms响应、1440 counts/s和320 ms积分。观测年龄同时覆盖10、15、20 ms，使用三档中的最差指标排名。

左右视频包含约6次独立横移，目标在每段结束后瞬移回起点。工具沿用项目移动分析契约：相邻检测间隔超过100 ms或全屏位置跳变超过64 px时切分试次并重新居中；左右只统计每段末尾1秒稳态。反转保留连续换向序列，静止排除首秒收敛。候选必须使左右稳态P95至少改善10%，且静止双轴回退不超过0.5 px、反转P95回退不超过10%、裁剪外丢失不增加，否则输出关闭预测。

正式运行方式：

```powershell
build\dml\Release\Xen.exe --video-replay C:\Users\16143\Desktop\Xen\Video --video-model E:\Dev\Xen\x64\DML\models\yolo12n_cs2.onnx --video-output C:\Users\16143\Desktop\Xen\Video\analysis --crop-x 1120 --crop-y 560 --crop-width 320 --crop-height 320 --inference-fps 94 --observation-ages-ms 10,15,20 --fov-x 106 --fov-y 74 --sensitivity 1.4 --yaw 0.022 --pitch 0.022 --response-ms 80 --max-cps 1440 --integral-ms 320
```

输出包括逐采样轨迹 `video_observations.csv`、完整参数网格 `prediction_grid.csv` 和最终建议 `prediction_recommendation.txt`。

## 参数结果

| 指标 | 关闭预测 | 20 ms前瞻 / 75 ms平滑 | 变化 |
|---|---:|---:|---:|
| 左移稳态P95 | 22.212 px | 16.642 px | 改善25.1% |
| 右移稳态P95 | 8.480 px | 8.399 px | 改善1.0% |
| 左右平均稳态P95 | 15.346 px | 12.520 px | 改善18.4% |
| 高频反转P95 | 15.681 px | 14.409 px | 改善8.1% |
| 静止水平P95 | 0.512 px | 0.512 px | 不变 |
| 静止垂直P95 | 4.381 px | 3.782 px | 改善13.7% |

`20/75`是唯一综合得分最优且满足全部安全门槛的候选。10/75的持续收益略低；40 ms及以上前瞻开始明显放大右移误差，因此不进入实机测试。视频模拟用于统一筛选，不包含真实NDI到达抖动、输入设备队列和游戏镜头的全部非线性，不能替代最终实机验收。
