# 框外身位预测与防翻向设计

> 状态：已被 `docs/033运动学自适应提前重构20260714.md` 取代。目标框可能只包含局部身体，固定框外身位不能作为可靠提前依据；本文件仅保留阶段性排查记录。

## 背景

2026-07-14使用 `BuildRevision=2d2663a1b17f`、`ControllerRevision=5`、DML+NDI、10 ms额外前瞻、75 ms速度平滑和2000 counts/s完成复测。jump P95由上一轮77.04 px降至66.80 px，限速比例由10.1%降至5.9%；持续左右P95分别为7.27 px和3.19 px，反转19.24 px，静止5.94 px。基础控制速度收益明确，但jump预测偏移P95仅6.12 px，普通左右仅1~3 px，绝大多数预测点仍落在目标框内部，不符合“准星主动领先目标”的使用目标。

## 原因

旧预测只计算“目标速度×观测年龄与固定前瞻时间”，并把最大距离限制为检测宽度的10%。10 ms额外前瞻与75 ms速度平滑组合会把实际提前压到数像素。更重要的是，检测坐标中的速度同时包含目标自身移动和程序移动准星造成的反向屏幕位移；若直接强制把预测点推到框外，准星越过目标后可能把自身视角运动误判为目标反向，产生左右翻向。

## 设计

预测器先把程序实际发送的鼠标counts换算为完整源画面的像素视角位移，并按检测观测时间戳恢复目标自身运动坐标。预测方向只由该补偿后速度决定，不使用“目标当前位于准星左侧还是右侧”作为翻向条件。

新增 `prediction_outside_box_scale`：沿确认方向从pivot走到目标框边缘，再继续前置指定数量的目标方向投影宽度。0表示只保留时间前瞻，0.50表示框外半个身位，1.00表示框外一个身位。水平移动使用目标框宽度，垂直或斜向移动使用目标框在运动方向上的投影尺寸，因此不同距离和目标大小下含义一致。

方向状态采用固定保护规则，避免继续增加人工参数：初始移动需要连续三次同向真实观测才启用框外提前；第一帧停止、低速或疑似反向立即撤销框外提前；连续两次反向观测后才切换到新侧。目标丢失、ID切换、瞄准状态或分辨率变化仍立即清空全部状态。自动开火继续使用真实目标框判定，本阶段不要求预测点位于框内。

## 诊断

流水线CSV新增目标框、框外身位配置、预测偏移、自身视角位移、方向锁定和框外生效字段。移动分析器输出 `PredictionOutsideAppliedPct`、`P50OutsideLeadPx`、`P95OutsideLeadPx` 和 `PredictionSideFlipCount`，用于区分“框外提前未生效”“提前距离不足”和“预测侧额外振荡”。控制器行为修订号升级为6。

## 验证方案

首轮保持10/75/2000不变，只设置框外提前0.50，采集jump、持续左移、持续右移、高频反转各2000行，静止至少9次，并执行目标飞出、目标死亡和角色死亡各10次。静止框外生效比例必须低于1%；单向场景预测侧翻转不超过1次；反转场景预测侧翻转不得明显多于脚本实际反转次数；目标丢失后不得继续输出。

只有0.50无额外左右振荡但视觉提前仍不足时，才把该单一参数提高到1.00并仅复测jump与reverse。若0.50已发生多余翻向，应优先排查视角补偿和方向确认，不得通过加大平滑掩盖问题。
