# 持续运动预测复位与门控锁存修复

## 背景

用户使用 `DML / 3cd5e7f8376d / r45`、正式补偿 12 ms 中心和 24 ms 线性宽度完成 left/right 各三轮复测，观察到预测提前量归零后重新建立。CSV 自身确认 `ViewMotionCompensationDelayMs=12`、`ViewMotionCompensationResponseMs=24`。

## r45 结果

| 指标 | left | right |
|---|---:|---:|
| X真实稳态P95 | 12.17 px | 12.33 px |
| X输出换向 | 2 | 0 |
| 同方向步进P95 | 7.33 counts | 7.67 counts |
| 同方向脉冲频率 | 35.06 Hz | 33.84 Hz |
| 预测生效率 | 99.2% | 96.1% |
| 自运动抑制 | 0.3% | 0.9% |
| 预热后预测中断 | 1 | 2 |
| 停止/高速/往返抑制 | 0% | 0% |
| 预测方向翻转 | 0 | 0 |

r45 的 24 ms 连续响应没有降低 r44 的 7.33/7.00 counts 步进和 36.66/34.05 Hz 脉冲，也没有改善稳态误差，按预先约定停止修改自运动响应核，正式默认回退 `aim_motion_compensation_response_ms=0`。线性模型和配置入口保留为实验能力，不再作为生产默认。

## 复位分类

逐帧检查共找到五段“非零提前量归零后重新建立”：

- left 第三轮 1 段，持续 112 ms；
- right 第一轮 2 段，持续 312 ms 和 134 ms；
- right 第二轮 1 段，持续 99 ms；
- right 第三轮 1 段，持续 132 ms。

其中四段在归零首帧明确命中 `PredictionSelfMotionSuppressed=1`。门控保持四帧后调用 `discardMotionEvidence()`，方向需要重新确认并经过七帧提前量渐进释放，因此形成约 99~134 ms 的可见复位。

right 第一轮 312 ms 段发生在试次起始收敛：raw pivot 先从 184.5 px 快速回到 172 px，随后目标才持续向右；预测器短暂建立旧方向后进入真实正方向证据确认。该段停止、自运动、高速和往返字段均为0，不是成熟预测被门控打断，不能强行保留旧方向提前量。

## 根因

`applySelfMotionSuppression()` 的设计注释要求：真实 left/right 一旦连续预测成熟，正常闭环收敛不得再被自运动门控打断。实现却直接检查当前 `continuousPredictionFrames_`。同一帧中，短暂回归退化会先把该计数清零，随后自运动判定看不到此前已经成熟的状态，于是重新开放门控并丢弃整段运动证据。

这是状态生命周期错误，不是门控几何阈值过严。继续调整对齐阈值会同时削弱 static 保护。

## r46 设计

1. 新增 `sustainedMotionConfirmed_` 锁存。连续有效预测达到三帧后建立；短暂回归退化只撤销当帧提前量，不重新开放自运动门控。
2. 只有目标重置、确认停止达到30 ms、连续五帧运动证据不可靠、真实方向切换或自运动保持完成并主动丢弃证据时清除锁存。
3. static 的1~2帧伪预测无法建立锁存；确认停止仍在30 ms立即撤销提前量，安全路径不放宽。
4. 新增 `PredictionMotionEvidenceSuppressed` CSV 字段和 `PredictionMotionEvidenceSuppressedPct` 分析指标，独立标记初次建向、回归退化和反向确认，不再把“所有门控均为0”的零提前量当成未知复位。
5. 正式补偿响应宽度回退0，控制器身份提升到 r46。

## 测试与验收

单元测试新增“持续运动成熟后连续四帧二维交替回归退化”的复现：先确认当帧提前量确实归零，再验证历史持续运动锁存仍阻止自运动伪迹触发保持。原有 static 自运动保持、四帧尾迹、停止后清除豁免、重新武装和目标重置测试全部保留。

r46 使用相同 DML/NDI left/right 各三轮：

- CSV 必须为 `ControllerRevision=46`、补偿 `12/0 ms`；
- 单向成熟段 `PredictionSelfMotionSuppressedPct=0`，预热后预测中断为0；
- 初始建向允许 `PredictionMotionEvidenceSuppressed=1`，但不得在长时间成熟预测后反复出现；
- 停止、高速、往返抑制和预测方向翻转继续为0；
- X/Y输出换向不高于r45的2/0和0/0。

同方向幅值脉冲在r45未改善，r46不以该指标作为本次门控修复收益；预测复位稳定后再转查请求counts、整数余数和最终命令之间的二帧极限环。
