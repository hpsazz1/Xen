# P0-5低速反向确认复验

## 背景

静止角度回差把跨域矩阵从613/845提升到708/845，但仍有52组static因候选输出反向次数增加而失败。逐帧检查显示，剩余问题不是持续的大幅跟踪错误，而是静止回差释放后，近中心的低速二维修正方向被观测噪声短暂翻转。继续放宽静止角速率会吞掉真实运动，因此本轮只处理控制器输出语义，不修改Kalman、P增益、前馈、限速和既有0.08°/1.2°/s回差。

## 设计

`LosAimController`新增真实时间域的低速反向确认：

- 只在误差不超过静止退出边界0.12°、相对LOS速率低于1.2°/s时工作；误差或速率越界当帧放行，避免给jump、reverse和正常左右跟随增加固定延迟。
- 二维候选与最近一次在静止回差区内接受的输出方向点积为负时，必须保持同一方向80 ms才放行；方向再次变化会重新计时。
- 静止锁存清除待确认候选，但保留最近接受方向，便于释放后识别孤立反向脉冲；控制器reset清除全部历史。
- 配置`aim_shadow_reverse_confirm_ms`范围0~250 ms，默认80 ms，0关闭。流水线逐帧CSV记录抑制标记和已确认时长，跨域汇总记录抑制占比。

## 参数排查

先以未限定静止回差区的方案复验：80 ms为706/845，40 ms为704/845，说明全局低速确认会损伤真实reverse。将速率旁路边界收紧到1.2°/s后恢复到708/845，但仍无净收益。最终把确认同时限定在0.12°误差回差区内，才把噪声修正与真实运动分开。

最终命令在既有P0-5命令后增加：

```powershell
--settle-error-deg 0.08 --settle-rate-dps 1.2 --reverse-confirm-ms 80
```
