# P0-6C输出暂停连续影子观测

## 背景与根因

r55长暂停验收已证明，legacy在松开期间不消费检测观测，重新按住后首次可靠预测P95达到249.5 ms。r58随后修复检测发布时钟并通过jump及四普通场景回归，legacy已冻结。P0-6C只解决新影子链的数据连续性，不修改legacy滤波、预测、控制、设备速度或发送语义。

调用链复核发现两个断点：鼠标主循环只有`aimingNow=true`时才调用同时包含shadow观测的`moveMousePivot()`；松开时`MouseThread::suspendAimingOutput()`还会重置`AimPipelineRuntime`。因此检测线程和MOT虽继续维护目标ID，shadow的Kalman、观测序列和控制诊断仍完全断流。

## 设计与实现

1. `MouseThread::observeAimPipelineOnly()`成为输出暂停专用入口。主循环只在未瞄准、同一跟踪目标有效且本帧有真实检测时调用；SOT滑行帧不伪造观测。
2. 公共`processAimPipelineObservation()`统一正式帧和暂停帧的观测、完整时间戳、角度换算、应用相机历史、Kalman和候选控制计算，避免两条shadow数据流产生口径差异。
3. `AimPipelineRuntime::suspendOutput()`把“暂停不是目标丢失”固化为生命周期契约：shadow保留重置代次和观测序列；legacy维持原空状态重置。真实目标丢失、ID切换和配置变化仍走`resetTracking()`完整重置。
4. 暂停入口不调用legacy滤波、预测、PI、条件速度预算、被动标定或`queueMove()`，也不访问设备发送接口。松开时既有`clearQueuedMoves()`和取消代次仍先执行。
5. `AimObservation::outputPaused`贯通CSV字段`AimPipelineOutputPaused`。暂停帧的legacy请求、最终输出和命令生命周期字段保持默认0，`QueuedMoveCount`记录实际队列长度；shadow控制与轨迹可产生候选诊断，但`AimPipelineCommandSuppressed`和`AimPipelineTrajectoryCommandSuppressed`始终为1。
6. 控制器修订号升级为r59，使UI、CSV和构建产物可明确区分新增暂停观测行为。

## 自动验收

算法测试覆盖shadow输出暂停前后保持同一重置代次、观测序列1→2→3连续、暂停标志正确和双重命令抑制；目标丢失的完整reset测试保持不变。CSV测试覆盖新增列及表头/数据列数一致。

`tools/analyze_shadow_pipeline.ps1`新增暂停安全验收：可指定预期控制器修订号；统计暂停观测；检查同目标同代次的暂停边界序列连续；要求暂停帧`FinalMx/FinalMy`、命令入队、发送、请求、应用及队列长度均为0。`-RequirePausedObservations`和`-MinPausedObservations`用于正式实机门禁。脚本测试包含有效暂停、暂停命令越界和完全缺少暂停观测三类样本。

预提交验证使用仓库正式构建脚本导入Visual Studio工具链后完成：DML与CUDA均执行干净Release构建，两个构建目录的CTest均为9/9通过，影子CSV脚本的有效、越界和缺失暂停样本也按预期通过或拒绝。直接在未导入Visual Studio环境的普通PowerShell中调用`cmake --build`会因系统头文件路径缺失而失败；这不是源码回归，后续仍应统一使用正式脚本，避免把工具链环境问题误判为编译问题。

## 实机门禁

构建完成后使用DML/NDI和`aim_pipeline_mode=shadow`采集static、left、right、reverse、jump各1份，再额外采集left/right各至少6个短暂停和6个0.35～2秒长暂停。轨迹模式保持off，shadow前馈、积分和提前保持0。正式命令为：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools/analyze_shadow_pipeline.ps1 `
  -DataRoot "C:\Users\16143\Desktop\Xen\DML\ndi" `
  -ExpectedBuildRevision <r59构建提交短号> `
  -ExpectedControllerRevision 59 `
  -MinEstimateSamples 30 `
  -RequireStandardScenarios `
  -RequirePausedObservations `
  -MinPausedObservations 30
```

必须确认模式/时间/诊断/阶段越界、暂停命令和暂停连续性违例全部为0。随后再比较暂停期间shadow速率方向与重新按住时legacy首次提前时间。DML数据通过前禁止active，也不进入CUDA shadow。
