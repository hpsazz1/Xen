# jump条件速度预算实现

## 背景与问题

r56/3200五场景复测证明left、right、reverse和static均已满足安全回归要求，但jump四轮真实水平误差P95均值仍为66.88 px，最新长试次末1秒P95达到110.5 px。全局3200/4000 A/B又表明，4000 counts/s只在3000～5000 px/s高速层达到15.7%收益，三轮总体P95仅改善1.1%，同时放大低速输出步进和最大换向幅度。因此不能把全局上限直接提高到4000，也不能继续缩短已经在大误差区达到30 ms的响应时间。

最新r56/3200 jump中，`PredictionHighSpeedSuppressed=1`且水平误差至少60 px的样本只有17帧；这些样本预测水平速度中位数/95分位为1772/1898 px/s，观测水平速度为1899/3543 px/s，误差中位数/95分位为73.8/83.1 px。它们对应的是可验证的高速严重落后尾部，适合作为独立额外速度预算的进入证据。

## 设计

新增`ConditionalSpeedBudget`状态机，基础设备上限仍由`move_max_speed_cps`控制，默认3200 counts/s；新增`move_catch_up_max_speed_cps`作为条件追赶绝对上限，默认4000 counts/s，设置为基础上限即可完全关闭。

320检测裁剪的进入条件必须同时成立：

- `PredictionHighSpeedSuppressed=1`，即预测器已识别到高速高加速度超出常速度可信范围；
- 预测方向已经锁定，且不处于自运动、振荡或静止安全抑制；
- 水平误差绝对值至少60 px；
- 可靠回归水平速度绝对值至少1280 px/s；
- 速度与误差同号，即目标仍沿落后方向远离准星。

非320裁剪时，误差与速度阈值按检测宽度等比例放大。进入后不要求高速瞬态标志持续存在，但使用32 px和640 px/s退出阈值形成滞回；误差恢复、速度下降、方向丢失、速度转为朝向准星、自运动/振荡/静止安全门控、120 ms最长持续时间中的任一条件成立即退出。退出后冷却200 ms，防止检测尖峰反复打开高预算。目标丢失、身份切换、松开瞄准和配置变更均重置状态机。

额外预算只改变基础控制器最终二维counts速度上限，不修改滤波、预测、积分、响应时间、稳定回差或影子链。流水线CSV新增`ConditionalSpeedBudgetActive`，并继续用动态`MaxCountsPerSecond`记录实际采用的3200或4000；控制器修订号升级为r57。

## 配置与界面

配置新增：

```ini
move_max_speed_cps = 3200
move_catch_up_max_speed_cps = 4000
```

条件上限保存和读取时被限制在基础上限至4000 counts/s。基础移动界面新增“条件追赶速度”滑块；修改基础上限时同步保证条件上限不低于基础上限。

## 自动验证

单元测试覆盖以下边界：误差不足、速度朝向准星、安全门控时禁止进入；满足全部证据时进入4000；高速标志消失后仍按退出滞回保持；误差恢复时退出；冷却期禁止重入；冷却结束允许重入；120 ms超时强制退出；条件上限等于基础上限时完全关闭。配置测试覆盖自动推导不得覆盖该参数、默认值、低于基础上限回夹和超过4000回夹；CSV表头顺序与数据列数继续由算法测试校验。

## 实机复测与阶段门禁

本次只解决已被数据定位的jump高速严重落后尾部，不能在代码测试通过后直接宣称现场问题解决。下一轮先使用DML/NDI、320裁剪、120 ms响应、500 ms积分、基础3200和条件4000，分别采集jump三个独立试次；每轮采集前清空流水线缓冲，避免2000帧环形缓冲把多轮合并。

验收时除真实X P95/P99、框内率、限速率和末1秒P95外，还必须检查`ConditionalSpeedBudgetActive`占比、单次连续时长、进入时误差/速度、退出原因附近的输出换向和最大步进。通过条件为jump三轮P95相对r56/3200四轮均值66.88 px至少改善15%，且P99、末段拖尾和框内率同向改善；随后必须复测left、right、reverse、static各一轮，要求普通场景条件预算不误触，预测中断、输出换向和静止停发不回退。

在上述实机门禁完成前，r57仍属于legacy条件候选，跨游戏新链继续`HOLD_SHADOW`，禁止active接管。若jump收益不足15%或普通场景误触，将`move_catch_up_max_speed_cps`设为3200即可立即回退，并基于CSV决定否决该候选，而不是继续叠加局部门槛。
