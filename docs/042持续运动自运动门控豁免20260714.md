# 持续运动自运动门控豁免

## 背景与版本

2026-07-14使用 `BuildRevision=b7bde5e33681`、`ControllerRevision=14`、DML+NDI、3200 counts/s和预测50/50/1.0完成left/right各4轮复测。CSV身份与正式DML构建一致，全程未使用手动鼠标辅助，NDI接收约237~238 FPS。

## 数据结论

r14在static九段已经通过，但移动场景明显不稳定。left/right X轴平均P95误差为26.28/24.23 px，高于r10基线22.26/19.97 px；预测生效率只有63.9%/83.8%，自运动抑制却高达23.6%/10.6%。left第2轮抑制42.1%、第3轮29.3%，与用户观察到的反复震荡一致。

对照轮次证明瓶颈不是15 px提前量本身。right第3轮预测生效率98.2%、自运动抑制0.9%，X轴P95为全组最佳18.78 px；门控较少时连续预测能够达到既有验收线。抑制频繁的轮次误差普遍升至25~31 px。

## 根因

r11自运动门控使用“目标屏幕位置向中心收敛、程序视角与误差同向、补偿后预测速度向误差外侧”的联合特征。这个特征可以识别主动瞄准静止目标的初始收敛，但在程序持续跟随真实横移目标时同样天然成立，因此不能无限期应用。

逐帧统计显示，多次误抑制发生前预测已经连续稳定23、29、48、68、184、214甚至302帧，显然不是静止目标初始残差。误抑制后预测器执行四帧保持、丢弃证据和8 px重新武装，再次建立预测后又可能被门控命中，最终形成周期性开关。

static r14正式数据中，唯一有效预测段只有2帧，随后即被门控捕获；因此可以用短时间连续性区分初始伪迹和真实持续移动，而不需要改变几何阈值、预测强度或控制速度。

## r15设计

1. 同一方向连续有效预测达到5帧后，标记为持续真实运动；此后忽略自运动伪迹判定，不再撤销提前量。
2. 五帧约对应当前DML推理下50 ms。static伪预测只有1~2帧，仍会在获得豁免前被原门控捕获。
3. 停止导致预测撤销、窗口持续退化、方向解锁或真实反向确认时，连续计数归零；新方向必须重新连续5帧才能获得豁免。
4. 自运动门控真正命中时仍执行原有四帧保持、运动证据清除和8 px重新武装，static安全链路不变。
5. 不修改3200 counts/s、50/50/1.0预测参数、透视换算、基础控制器或目标丢失策略。

控制器修订号升级为15。单元测试覆盖连续五帧后忽略自运动门控，以及目标停止后撤销豁免并恢复门控能力。

## 下一轮测试

- 保持参数不变，先复测left/right各4轮；每轮约4~5秒更容易暴露周期性误抑制，轮间间隔至少300 ms。
- left/right预测生效率至少95%，自运动抑制低于5%，每场景预测中断不超过5次；X轴P95分别不高于22.26/19.97 px。
- left/right通过后立即补static九段，要求X/Y预测生效率仍不高于0.5%，确认五帧豁免没有放过静止伪迹。
- 然后再进入jump、reverse和 `Target_static_Role_reverse` 各20秒，以及三类目标丢失各10次。
