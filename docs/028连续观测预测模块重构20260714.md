# 连续观测预测模块重构

## 背景

2026-07-14完成DML FP32/FP16各6000行性能数据复核。OBS NDI采集始终为320×320，但模型审计确认FP32模型输入为320×320、FP16模型输入为640×640。FP16虽然把推理均值从9.395 ms降至7.069 ms、发布FPS从94.2提高到108.5，但持续右移稳态P95回退约16%，静止水平P95回退约40%，不能视为同输入精度下的有效替换。按用户决定保持FP32并结束性能阶段，转入预测模块。

旧预测实现由多个参数和失效的未来点可视化拼接而成，主控制链已在基础移动重构时将其禁用。重新启用旧实现会同时引入预测间隔、点数、延迟补偿开关和独立丢失超时，且可能绕过现有目标生命周期，存在目标飞出检测框、目标死亡或角色死亡后继续外推的风险。

## 调用链与约束

鼠标主循环只在当前锁定目标有真实检测观测时调用 `moveMousePivot()`。跟踪器滑行帧可以暂时维持内部锁定，但不会发送鼠标移动。以下事件统一调用 `resetTracking()`：无锁定目标、最近目标模式无目标、目标超时、跟踪ID切换、开镜状态变化、检测分辨率变化以及跟踪器开关变化。

因此预测器不自行维护丢失超时，也不接受跟踪器滑行位置。调用方每次提供真实观测；任何丢失或身份变化立即通过同一个重置入口清空基础滤波、预测速度、PI控制器和待发送队列。重置后的第一个观测只建立位置，第二个连续观测才允许预测，禁止使用旧目标速度跨身份外推。

## 设计

新增无外部依赖的 `TargetPredictor`，接在 `BasicTargetFilter` 之后、`BasicAimController` 之前。速度由相邻过滤位置和真实检测时间戳计算，再使用帧率无关的一阶低通更新。控制位置为：

`预测位置 = 过滤位置 + 平滑速度 × (观测年龄 + 固定额外前瞻)`

对外只保留三个设置：

- `prediction_enabled`：总开关；关闭时清空预测状态并精确返回基础过滤位置。
- `prediction_lead_ms`：观测年龄之外的额外前瞻，默认20 ms。
- `prediction_velocity_tau_ms`：速度平滑时间常数，默认35 ms。

检测延迟由观测时间戳自动计算，不再提供重复开关。丢失重置由目标生命周期负责，不再提供预测专用超时。旧 `predictionInterval` 和 `prediction_tau` 只在读取历史配置时迁移，保存后改写为新毫秒制键；旧预测点数和两套无输出的预测点绘制配置被移除。

内部安全边界不作为调参项：观测年龄最多计100 ms，总前瞻最多120 ms，速度最多为检测宽度的6倍每秒，单帧预测距离最多为检测宽度的10%。这些限制只拦截错误关联、异常时间戳或瞬时检测跳变，不替代目标身份判断。

## 诊断与验证

流水线CSV新增 `PredictionApplied`、`PredictionVelocityX/Y`、`PredictionLeadMs` 和 `PredictedX/Y`，控制器行为修订号由4升级为5。单元测试覆盖首帧预热、连续匀速前瞻、静止不漂移、不规则DML时间戳、观测年龄补偿、丢失重置、异常跳变限制和关闭预测后的精确旁路。DML模型输入尺寸审计同时补入CSV，后续测试可直接确认采集尺寸与模型内部尺寸不是同一字段。

## 下一步现场测试

保持DML NDI、FP32 320×320模型、80 ms响应、1440 cps和320 ms积分不变。按 `关闭 → 20 ms → 35 ms → 关闭` 顺序，每轮预热30秒后分别采集静止、持续左移、持续右移和高频反转各2000行，速度平滑固定35 ms。首尾关闭组用于排除测试时段漂移。

20 ms候选需使持续移动稳态P95至少改善15%；高频反转P95回退不超过10%、恢复P95回退不超过10 ms；静止双轴P95回退不超过0.5 px且稳定率保持100%。还需检查目标主动离开、目标死亡和角色死亡各至少10次，丢失后不得出现新的控制CSV行或鼠标移动。若20 ms收益不足10%或安全指标越界，关闭预测并直接结束本阶段；只有20 ms通过且持续移动仍保留明显同向误差，才比较35 ms。
